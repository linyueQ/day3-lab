## 1. 背景与问题

> 描述用户遇到的问题，至少写 2~3 条痛点。参考：信息获取、历史记录、可追溯性、系统可用性。

| 痛点 | 来源 |
|------|------|
| 信息分散：研报、新闻、数据分布在多个系统，无统一问答入口 | 用户访谈 |
| 阅读耗时：投研人员需耗费大量时间阅读长篇研报以提取核心观点（如评级、目标价），人工提效工具缺失 | 投研部门需求收集会 |
| 研报对比难：多份异构研报的核心观点与数据难以直观横向对比，依赖人工摘录至 Excel，效率低下且易出错 | 投研助手产品需求讨论会 |
| 可信度挑战：AI 生成内容存在幻觉风险，若无法快速核对原文，分析师不敢直接采用问答结果 | 用户访谈 |

## 2. 提案概要

> 用 3~5 句话概括要做什么。参考格式："在 XXX 应用内交付 YYY 能力"。要素：会话式问答 · 能力降级 · 可追溯

本项目旨在投研工作台应用内交付"智能问答助手"能力，以显著提升投研人员的信息获取效率。核心提供基于研报文档的会话式问答与信息自动摘要功能，并扩展为三大业务模块：① 研报问答（RAG 检索 + 三级降级）；② 研报对比（多维度横向对比评级、目标价、财务指标）；③ 舆情监控（RSS 多源聚合 + 优先级自动评分）。系统设计了完善的能力降级预案，在外部服务异常时仍能保障基础的查询响应或 Demo 演示。同时，问答生成的结果均强制附带原文引用路径，确保投研决策信息的完全可追溯与合规可信。

## 3. 范围边界

> 明确"做什么"和"不做什么"，避免范围蔓延。各写 3~4 条。

| 做（范围内） | 不做（范围外） |
|-------------|---------------|
| 会话管理：创建、列表、删除 | 多租户/行级数据权限 |
| 研报问答：单篇研报核心观点摘要提取与问答交互 | 跨库比对：不支持与外部资讯库(如Wind)的实时联网查询比对 |
| 研报对比：支持 2~5 份研报多维度横向对比（估值概览/财务对比/深度分析） | 自动化交易指令生成或执行 |
| 舆情监控：RSS 多源新闻聚合、优先级自动评分、关键词搜索筛选 | 自研爬虫或深度舆情分析（NLP 情感分析） |

## 4. 成功标准（可度量）

> 写出 3~4 条可量化的验收标准。句式：用户在 X 分钟内完成 XXX 全链路 / 未配置外部模型时返回 XXX

| 标准 | 度量方式 |
|------|----------|
| 主路径可走通 | 用户在 3 分钟内完成"新建会话 → 提问 → 查看历史"全链路 |
| 问答响应效率及格线 | 用户提交问答后，首字响应时间<3秒，单次完整问答结果返回<15秒 |
| 核心信息溯源有效性 | 随机抽查10次数据问答，生成的原文引用链接点击后，必须100%准确跳转定位到源文档段落 |
| 研报对比可演示 | 选择 2~5 份研报后，系统在 5 秒内返回包含评级、目标价、财务指标的多维对比结果 |
| 舆情监控可演示 | 舆情页面加载后 10 秒内展示分级新闻列表，支持按优先级筛选与关键词搜索 |

## 5. 风险（简表）

> 列出 2~3 个主要风险。常见方向：外部依赖不可用、数据存储并发、模块间契约延期。

| 风险 | 影响 | 应对 |
|------|------|------|
| 外部 LLM 不可用 | 问答功能降级 | 三级降级：CoPaw → 百炼 → Demo 模式兜底 |
| 非标格式研报解析错乱 | 提取失败或问答质量直线下降 | 一期预先定义支持的标准研报排版格式，遇非标格式前端提示"解析置信度受限" |
| RSS 数据源不可用 | 舆情监控无法获取实时新闻 | 自动降级到 Mock 数据，保证页面可用 |
