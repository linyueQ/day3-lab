## 字段说明 — 每条 UserStory 必须包含以下字段

| 字段 | 填写说明 | 数据来源 ← | 流向下游 → |
|------|----------|------------|-----------|
| **需求编号** | 唯一编号，如 REQ-M1QA-001 | ← `02` 采集编号 | → `14` 追踪矩阵首列 |
| **故事** | 作为【角色】，我希望【行为】，以便【价值】 | ← `04` PRD 场景拆解 | → `06` FSD 功能点 |
| **优先级** | P0（必须）/ P1（应该）/ P2（可选） | ← `03` §4 成功标准 | → `12` 里程碑排期 |
| **来源** | 对应 03/04 哪一条需求 | ← `03` 范围 · `04` 场景 | |
| **前置条件** | 触发该故事的前提状态 | ← `04` 用户流程 | → `13` 测试前置 |
| **功能对齐** | 对应功能规格 + 非功能约束 | | → `06` FSD · `07` NFR |
| **设计对齐** | 对应架构、接口、数据、安全 | | → `08` 架构 · `09` API · `10` Data · `11` 安全 |
| **验证对齐** | 对应测试用例 + 追踪矩阵行 | | → `13` TC · `14` 追踪矩阵行 |
| **验收标准** | AC，每条可独立验证 | ← 故事 + `07` NFR | → `13` 测试用例依据 |

## US-001（P0）会话管理

| 字段 | 内容 |
|------|------|
| **需求编号** | REQ-M1QA-001 |
| **故事** | 作为 **投研分析师**，我希望 **能够创建、列表查看并删除问答会话**，以便 **按不同的研究主题（如特定股票）管理和追溯我的研究记录** |
| **优先级** | P0 |
| **来源** | ← 03 §3 / 04 §会话节点 |
| **前置条件** | 【用户已登录系统并具备访问权限】 |
| **功能对齐** | → 06 §会话列表管理 · 07 §列表加载时间<1s |
| **设计对齐** | → 08 §前端缓存联动 · 09 GET /sessions API · 10 session_info 表 · 11 §仅本人可见授权 |
| **验证对齐** | → 13 TC-001 · 14 第 1 行 |

| 验收标准 | 描述（每条可独立测试） |
|----------|----------------------|
| AC-001-01 | 【新建重置】点击"新建会话"时，主对话区域立即清空当前记录，重置为欢迎语状态。 |
| AC-001-02 | 【倒序展示】左侧边栏按时间倒序展示历史会话列表，点击任意会话，主区域能在1秒内无缝加载。 |
| AC-001-03 | 【二次确认删除】点击会话的删除图标需弹出二次确认框，确认后列表即时刷新，被删除项消失。 |

## US-002（P0）问答提交

| 字段 | 内容 |
|------|------|
| **需求编号** | REQ-M1QA-002 |
| **故事** | 作为 **投研分析师**，我希望 **在对话框输入研报相关问题并提交**，以便 **快速获取大模型提炼后的结构化观点，免除手动翻阅** |
| **优先级** | P0 |
| **来源** | ← 03 §3 / 04 §问答交互节点 |
| **前置条件** | 【已创建会话并打开对话框】 |
| **功能对齐** | → 06 §流式问答输出 · 07 §首字响应<3s，防抖控制 |
| **设计对齐** | → 08 §SSE流式通信 · 09 POST /chat API · 10 message_log 表 · 11 §防XSS注入 |
| **验证对齐** | → 13 TC-002 · 14 第 2 行 |

| 验收标准 | 描述（每条可独立测试） |
|----------|----------------------|
| AC-002-01 | 【防空提交】输入框为空白或全空格时，发送按钮禁用，回车不触发网络请求。 |
| AC-002-02 | 【状态反馈】提交问题后，界面展示"思考中"动画，同时发送按钮进入禁用状态防重复提交。 |
| AC-002-03 | 【流式渲染】接收到后端回包后，需以打字机流式效果平滑展示文本，并正确解析Markdown格式排版。 |

## US-003（P1）问答结果溯源

| 字段 | 内容 |
|------|------|
| **需求编号** | REQ-M1QA-003 |
| **故事** | 作为 **投研分析师**，我希望 **在查看 AI 回答时能点击引用标记直接查看研报原文**，以便 **快速核实数据的准确性，确保决策合规性** |
| **优先级** | P1 |
| **来源** | ← 03 §3 / 04 §结果核验节点 |
| **前置条件** | 【AI 已返回包含引用出处的回答，且源文件在系统中可访问】 |
| **功能对齐** | → 06 §引用标记与原文联动 · 07 §原文定位加载时间<1s |
| **设计对齐** | → 08 §PDF预览组件 · 09 GET /doc/chunks API · 10 doc_chunks 表 · 11 §防越权访问文档 |
| **验证对齐** | → 13 TC-003 · 14 第 3 行 |

| 验收标准 | 描述（每条可独立测试） |
|----------|----------------------|
| AC-003-01 | 【引用标记】当回复基于原文提取时，必须在关键句后以醒目的数字上标（如[1]）展示出处。 |
| AC-003-02 | 【跳转定位】点击引用标记，右侧预览区应自动加载源 PDF 并滚动定位至对应的原文段落。 |
| AC-003-03 | 【视觉高亮】定位后的原文段落需应用背景色高亮，协助分析师一眼锁定信息源。 |

## US-006（P1）研报对比

| 字段 | 内容 |
|------|------|
| **需求编号** | REQ-M1QA-006 |
| **故事** | 作为 **投研分析师**，我希望 **能够选择多份研报进行横向对比，查看评级、目标价、财务预测等多维度数据**，以便 **快速发现各券商观点差异，辅助投资决策** |
| **优先级** | P1 |
| **来源** | ← 03 §3 / 04 §SC-06 研报横向对比 |
| **前置条件** | 【系统中已有至少 2 份研报数据】 |
| **功能对齐** | → 06 §研报对比视图 · 07 §对比响应时间<5s |
| **设计对齐** | → 08 §研报解析架构 · 09 GET /reports + POST /reports/compare API · 10 Report 实体 |
| **验证对齐** | → 13 TC-006 · 14 第 6 行 |

| 验收标准 | 描述（每条可独立测试） |
|----------|----------------------|
| AC-006-01 | 【研报列表】切换至研报对比视图时，系统展示所有可用研报的卡片列表，每张卡片显示公司名、股票代码、评级、发布机构、目标价。 |
| AC-006-02 | 【多选对比】用户可选择 2~5 份研报进行对比，选择不足 2 份时"开始对比"按钮禁用；支持全选/取消全选操作。 |
| AC-006-03 | 【多维展示】对比结果分"估值概览""财务对比""深度分析"三个标签页呈现；财务对比表格中最优值以高亮背景标注。 |
