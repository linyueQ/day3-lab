## S-02 邮件沟通 — 关于投研助手的需求建议

| 项 | 内容 |
|---|---|
| 发件人 | 李总监 <li.d@company.com> |
| 收件人 | 王经理 <wang.m@company.com> |
| 时间 | 2026-04-10 09:30 |

**问题1 — 研报阅读效率低：** 每天需阅读10+份研报，平均每份30-50页，耗时2-3小时。希望能快速提取关键信息（评级、目标价、核心观点）。

**问题2 — 研报对比困难：** 对比不同券商对同一公司研报需手动复制粘贴，效率很低，希望能自动生成对比表。

**问题3 — 可信度挑战：** AI生成内容存在幻觉风险，若无法快速核对原文，分析师不敢直接采用问答结果。必须支持原文溯源。

## S-03 用户访谈 — 高级分析师

| 项 | 内容 |
|---|---|
| 访谈对象 | 陈研究员（高级分析师） |
| 访谈人 | 张产品 |
| 时间 | 2026-04-12 10:00-11:00 |
| 方式 | 线上访谈 |

| 问题 | 回答要点 |
|------|----------|
| Q1：平时工作中最耗时的环节？ | 读研报最耗时。每天扫一遍行业研报找重点，电子版反而不知从哪看起。 |
| Q2：最希望在研报中获取什么信息？ | ①评级和目标价 → ②核心观点（分析师主要逻辑）→ ③数据预测（营收增速、PE等） |
| Q3：同时看多份研报如何对比？ | 各家格式不同，需手动整理到Excel。希望能自动生成对比表。 |
| Q4：对AI的了解或期待？ | ChatGPT用得多，但涉及内部数据不敢传。**必须内部部署，数据安全第一。** |
| Q5：如何验证AI回答的准确性？ | 必须能看到原文出处，点击就能跳转到研报对应位置，不然不敢用。 |

**总结：** ①快速提取+原文溯源为核心诉求 ②必须本地部署，数据不出域 ③AI接受度高但要求可验证

## S-04 竞品分析 — 现有投研工具调研

| 工具 | 优势 | 劣势 | 借鉴点 |
|------|------|------|--------|
| Wind 金融终端 | 数据全面、权威 | 界面复杂、学习成本高、无自然语言问答 | 数据标准化展示 |
| 同花顺 iFinD | 研报库丰富 | 检索依赖关键词、无AI摘要 | 研报分类体系 |
| ChatPDF / 通用RAG | 自然语言交互 | 数据外泄风险、无金融专业优化 | 对话式交互体验 |

**差异化机会**：本地化部署 + 金融场景优化 + 强制原文溯源
