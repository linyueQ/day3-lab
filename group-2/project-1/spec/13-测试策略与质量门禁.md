## 一、测试分层

> 共 4 层，写测试前先搞清楚每层验什么。

| 层级 | 工具 | 覆盖对象 | 说明 |
|------|------|----------|------|
| **L1 Unit** | pytest（Flask）/ Jest（Express） | 单个函数/类的逻辑 | 不依赖网络，最快 |
| **L2 Integration** | pytest + Flask test_client / supertest（Express） | API 端点 + 存储 | 模拟 HTTP 请求 |
| **L3 Contract** | pytest + schema 断言 / JSON Schema 校验 | 响应体结构与 `09` 一致 | 验字段名、类型 |
| **L4 E2E** | Playwright | 完整用户旅程（含三视图切换） | 浏览器级操作 |

## 二、用例拆分原则

> **核心理念：数"可独立失败的点"** — 每条 TC 只回答一个问题，失败时不看代码就知道哪里坏了。

| 来源 | 计数方法 | 示例 |
|------|----------|------|
| 正常路径 | 每个端点的 happy path = 1 条 | `POST /ask → 200` = TC-001 |
| 错误码 | 每个 distinct error code = 1 条 | `EMPTY_QUERY` = TC-002 |
| 分支路径 | 每个 if/else 降级分支 = 1 条 | 无 API Key → demo = TC-004 |
| 类型约束 | Spec `09` 约定的字段类型 = 1 条 | `llm_used` 必须 bool = TC-006 |

## 四、质量门禁

| Gate-ID | 条件 | 阻塞合并 |
|---------|------|----------|
| **G-LINT** | `black --check` + `flake8`（Flask）/ `eslint`（Express/前端）通过 | **是** |
| **G-UNIT** | Unit 测试全绿（L1） | **是** |
| **G-INT** | Integration 测试全绿（L2） | **是** |
| **G-AC** | P0 验收标准全覆盖（TC-M01-041~048, TC-M01-061~063, TC-M01-071~074） | **是** |
| **G-PERF** | `response_time_ms < 5000` | 否（warning） |
| **G-COV** | 代码覆盖率 ≥ 80% | 否（warning） |

### 执行命令

```bash
# Flask 后端测试
cd code/backend
python -m pytest tests/ -q

# Flask 后端测试 + 覆盖率
python -m pytest tests/ -q --cov=. --cov-report=term-missing

# Express 后端启动（手动验证）
cd code/backend
node server.js

# E2E 测试（需先启动前后端服务）
cd code/frontend
npx playwright test
```

## 五、TDD 适用场景

| 场景 | TDD 价值 | 建议 |
|------|----------|------|
| 简单 CRUD（列表、删除） | 低 | 写完补测试 |
| 参数校验 / 错误码 | 中 | 建议 TDD |
| 多级降级 / 状态机 | **高** | **强烈建议 TDD** |
| 会话级联删除 | **高** | **建议 TDD** |
