## 1. 页面总体布局

```
┌──────────────────────────────────────────────────────────────────┐
│  header: "投研问答助手" + [问答|研报对比|舆情监控] + [能力芯片] + [健康灯] │
├────────────┬──────────────────────────────────┬──────────────────┤
│  sidebar   │  main-content（按 activeView 切换）│ doc-preview      │
│  历史会话   │  ┌ chat: 三态渲染                 │（默认隐藏）       │
│  [+新建]   │  │ A) 空状态：欢迎语              │  PDF预览          │
│  session1  │  │ B) 常见问题网格               │  高亮定位          │
│  session2  │  │ C) 对话历史列表               │                  │
│  session3  │  ├ compare: 研报对比视图          │                  │
│   ...      │  └ sentiment: 舆情监控视图        │                  │
│            │  input-area:（仅 chat 视图）      │                  │
│            │  textarea + 发送/清空按钮         │                  │
└────────────┴──────────────────────────────────┴──────────────────┘
```

**布局说明**：

| 区块 | 宽度占比 | 功能 | 对齐 US |
|------|----------|------|---------|
| Sidebar | 约 20% | 会话列表管理 | US-001 |
| Main-content | 约 55%（无预览时 80%） | 问答交互 + 历史对话 | US-002 / US-004 |
| Doc-preview | 约 25%（按需展开） | PDF 原文预览与高亮定位 | US-003 |
| Header | 100% | 品牌标题 + 能力状态 + 健康指示 | US-005 |

## 2. Header 区域

> 对齐 `05` US-005 健康检查、US-006 研报对比、US-007 舆情监控。Header 包含品牌标题、视图导航、能力芯片和健康指示灯。

### 2.0 视图导航按钮

| 按钮 | `activeView` 值 | 行为 |
|------|-----------------|------|
| 问答 | `chat` | 切换至问答视图，侧边栏 + 聊天区域可见 |
| 研报对比 | `compare` | 切换至研报对比视图（ReportCompare 组件），对齐 US-006 |
| 舆情监控 | `sentiment` | 切换至舆情监控视图（SentimentMonitor 组件），对齐 US-007 |

> 当前激活的按钮高亮显示。切换视图不影响会话状态和聊天记录。

### 2.1 能力状态芯片（Capability Chips）

| 条件 | 芯片显示 | 说明 |
|------|----------|------|
| `caps.copaw_configured = true` | CoPaw 桥接（绿色） | CoPaw 已配置且可用 |
| `caps.bailian_configured = true` | 百炼 · {model}（蓝色） | 百炼已配置 + 模型名 |
| `caps.copaw = false && caps.bailian = false` | 离线演示（灰色） | 降级到离线模式 |

### 2.2 健康指示灯

| 条件 | 指示灯 | 附加行为 |
|------|--------|----------|
| `GET /health` → `status: "UP"` | 绿色圆点 | 无额外提示 |
| `GET /health` → `status: "DEGRADED"` | 黄色圆点 + 顶部横幅 | 横幅文案"当前为 Demo 模式/服务受限"，对齐 AC-005-02 |
| `GET /health` 请求失败 | 红色圆点 + 顶部横幅 | 横幅文案"后端服务不可用，请稍后重试" |

> 健康检查频率：页面加载时触发一次，之后每 30 秒轮询一次 `GET /health`。

## 3. Sidebar 会话管理

> 对齐 `05` US-001。描述会话列表的增删查交互。

| 功能 | 触发 | API 调用 | 行为 |
|------|------|----------|------|
| 加载列表 | 页面首次渲染 | `GET /sessions` | 渲染会话列表（按 `created_at` 倒序）+ 默认选中第一个，对齐 AC-001-02 |
| 新建会话 | 点击"+ 新建"按钮 | `POST /sessions` | 插入列表头部 + 自动选中 + 主区域切换至状态 B（常见问题），对齐 AC-001-01 |
| 删除会话 | 点击会话右侧 × 图标 | `DELETE /sessions/<id>` | 弹出二次确认框"确认删除该会话？"→ 确认后从列表移除 + 若删除的是当前会话则切换至空状态，对齐 AC-001-03 |
| 选中会话 | 点击会话列表项 | `GET /sessions/<id>/records` | 高亮选中项 + 主区域加载该会话的历史记录（状态 C）+ 若有记录则滚动至底部 |

**会话列表项显示规则**：

| 元素 | 内容 | 说明 |
|------|------|------|
| 标题 | `session.title` | 首次问答后自动重命名（对齐 10 §6 首次问答自动命名） |
| 副标题 | 最后一轮 query 的前 20 字 | 无记录时显示"暂无问答" |
| 问答数 | `session.query_count` 条 | 右侧小字展示 |

## 4. Main 内容区（三态渲染）

> 对齐 `05` US-002（问答提交）、US-004（历史记录）。描述三种条件下显示什么。

| 状态 | 条件 | 显示内容 |
|------|------|----------|
| **A 空状态** | `currentSession === null` | 居中显示欢迎语"欢迎使用投研问答助手"+ 提示"请在左侧创建或选择一个会话开始" |
| **B 常见问题** | 有会话 && 无记录 | 居中显示会话标题 + 2×2 常见问题网格卡片（如"该股票最新评级是什么？""目标价区间是多少？""与同行业对比如何？""核心风险因素有哪些？"）；点击任一卡片自动填入输入框并发送，对齐 03 §2 研报问答场景 |
| **C 对话历史** | 有记录 | 按时间正序展示对话气泡列表；用户提问靠右、AI 回答靠左，对齐 AC-004-01 |

### 4.1 对话历史记录卡片元素

> 状态 C 下，每条 AI 回答气泡包含以下元素。

| 元素 | 位置 | 说明 |
|------|------|------|
| 答案文本 | 气泡主体 | Markdown 渲染（`react-markdown`），对齐 AC-002-03 |
| 引用标记 | 答案文本内嵌 | 数字上标 `[1]`、`[2]`，蓝色可点击，对齐 AC-003-01 |
| 来源标签 | 气泡底部右侧 | 根据 `answer_source` 显示，规则见 §4.2 |
| 时间戳 | 气泡底部左侧 | `timestamp` 格式化为 HH:mm |

### 4.2 记录卡片中的来源标签逻辑

> 根据 `answer_source` 和 `llm_used` 组合显示不同标签。

| `answer_source` | `llm_used` | 显示文本 | 样式 |
|-----------------|------------|----------|------|
| `copaw` | `true` | CoPaw | 绿色标签 |
| `bailian` | `true` | 百炼 | 蓝色标签 |
| `demo` | `false` | 离线演示 | 灰色标签 |

### 4.3 Doc-preview 原文预览区

> 对齐 `05` US-003 问答结果溯源。当用户点击引用标记 `[n]` 时展开。

| 触发 | API 调用 | 行为 |
|------|----------|------|
| 点击引用标记 `[n]` | `GET /doc/chunks?chunk_id={chunk_id}` | 右侧 Doc-preview 区域展开，加载源 PDF 并滚动至 `page` 页，对齐 AC-003-02 |
| PDF 定位完成 | — | 目标段落应用黄色背景高亮，对齐 AC-003-03 |
| 点击 Doc-preview 关闭按钮 | — | 收起预览区，Main 区域恢复全宽 |

> Doc-preview 默认隐藏（宽度 0），点击引用标记后滑出占据约 25% 宽度，关闭时收回。

## 5. 输入区域

> 对齐 `05` US-002 AC（query 非空且长度 1~500）。

| 元素 | 类型 | 行为 |
|------|------|------|
| query-input | `<textarea>` | 3 行，placeholder "请输入您的问题..."；仅当 `currentSession !== null` 时启用 |
| send-btn | `<button>` | loading 时 disabled + 文案变为"发送中…"；输入为空/全空格时 disabled，对齐 AC-002-01 |
| clear-btn | `<button>` | 清空输入框，重置焦点 |
| char-counter | `<span>` | 显示"已输入 {len}/500 字"，超过 450 字变为橙色警告，达到 500 字时输入框阻止继续输入 |

**输入交互规则**：

| 交互 | 行为 | 对齐 AC |
|------|------|---------|
| 空白/全空格输入 | send-btn 禁用 + 回车不触发请求 | AC-002-01 |
| 点击发送/回车 | 立即显示"思考中"动画 + send-btn 禁用，防止重复提交 | AC-002-02 |
| 收到 SSE 流式响应 | 打字机效果逐字渲染 + Markdown 实时解析 | AC-002-03 |
| 响应完成 | "思考中"动画消失 + send-btn 恢复可用 + 输入框清空 | — |

## 6. 错误处理

| 后端 `error.code` | HTTP | 前端展示 |
|--------------------|------|----------|
| `EMPTY_QUERY` | 400 | 输入框下方红色提示"请输入问题" |
| `INVALID_QUERY` | 400 | 输入框下方红色提示"问题过长，请控制在 500 字以内" |
| `MISSING_SESSION_ID` | 400 | 顶部 Toast 提示"会话参数缺失" |
| `SESSION_NOT_FOUND` | 404 | 顶部 Toast 提示"会话不存在或已被删除" + 刷新侧边栏列表 |
| `DOC_CHUNK_NOT_FOUND` | 404 | Doc-preview 区域提示"原文引用未找到" |
| `LLM_SERVICE_ERROR` | 500 | 对话气泡中显示"服务暂时不可用，请稍后重试" + 来源标签显示"离线演示" |
| `SERVICE_DEGRADED` | 503 | 顶部横幅"后端服务异常" + 对话气泡显示降级提示 |

## 8. 研报对比视图（ReportCompare）

> 对齐 `05` US-006 研报对比。`activeView === "compare"` 时渲染此组件。

### 8.1 选择阶段

| 元素 | 说明 |
|------|------|
| 研报卡片列表 | 调用 `GET /reports` 获取可用研报，每张卡片展示公司名、股票代码、评级（彩色背景）、发布机构、目标价、当前价、上涨空间 |
| 多选复选框 | 每张卡片左上角，最少选 2 份、最多 5 份 |
| 全选/取消全选 | 快捷操作按钮 |
| "开始对比"按钮 | 选中 < 2 时禁用；点击后调用 `POST /reports/compare`，对齐 AC-006-02 |

### 8.2 对比结果（三标签页）

| 标签页 | 内容 | 对齐 AC |
|--------|------|---------|
| **估值概览** | 每份研报的卡片（评级/目标价/当前价/上涨空间）+ 上涨空间条形图对比 | AC-006-03 |
| **财务对比** | 表格模式，行 = 指标（营收/增速/净利润/毛利率等），列 = 年份，最优值高亮背景 | AC-006-03 |
| **深度分析** | 核心观点对比、风险提示对比、研发实力表格（研发人员/专利数/资产负债率/现金余额） | AC-006-03 |

## 9. 舆情监控视图（SentimentMonitor）

> 对齐 `05` US-007 舆情监控。`activeView === "sentiment"` 时渲染此组件。

### 9.1 重点预警面板

| 元素 | 说明 |
|------|------|
| 预警卡片 | 最新的 urgent 和 high 优先级新闻（最多 3 条），突出显示 |

### 9.2 新闻列表区

| 元素 | 说明 |
|------|------|
| 优先级标签页 | 全部 / 紧急预警 / 重要关注 / 值得关注 / 一般资讯，点击切换筛选，对齐 AC-007-02 |
| 搜索框 | 支持按标题、摘要、标签关键词搜索过滤，对齐 AC-007-02 |
| 新闻卡片 | 包含：优先级图标+徽章、来源、发布时间、标题、摘要、热度条（颜色渐变绿→蓝→黄→红）、标签列表，对齐 AC-007-01 |
| 数据源状态栏 | 底部展示各 RSS 源连接状态 |

### 9.3 刷新机制

| 行为 | 说明 | 对齐 AC |
|------|------|---------|
| 自动轮询 | 每 5 分钟调用 `GET /news` 更新数据 | AC-007-03 |
| 手动刷新 | 点击刷新按钮，传 `forceRefresh=true` 参数强制拉取 | AC-007-03 |
| 降级兜底 | 所有 RSS 源不可用时展示 Mock 数据 | AC-007-04 |
