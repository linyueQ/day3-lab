## 1. 性能需求

### 1.1 延迟 SLO

| 指标 | 目标值 | 度量方式 | 来源 |
|------|--------|----------|------|
| **问答端点 P95 延迟** | < **5000ms** | `response_time_ms` 字段 | `03` §4 |
| 会话管理端点延迟 | < 500ms | 服务端计时 | JSON I/O < 10ms |
| 首字响应时间 | < **3000ms** | 从用户提交到首字节返回时间 | `03` §4 / `05` AC-002-02 |
| 单次完整问答返回 | < **15000ms** | 流式响应完整结束时间 | `03` §4 |
| 会话列表加载延迟 | < **1000ms** | 从点击会话到主区域渲染完成 | `05` AC-001-02 |
| 原文定位加载延迟 | < **1000ms** | 点击引用标记到 PDF 段落高亮完成 | `05` AC-003-02 / `05` 功能对齐 |
| 页面初始加载时间 | < **3000ms** | 首屏渲染完成（FCP）| `05` 功能对齐 |
| JSON 文件读写延迟 | < **50ms** | 单文件 < 1MB 时的读或写操作 | `08` §5.1 JSON 文件存储 |
| 研报对比响应延迟 | < **5000ms** | 选择 2~5 份研报后返回对比结果 | `05` AC-006-03 |
| 舆情新闻加载延迟 | < **10000ms** | 首次加载或强制刷新 RSS 聚合结果 | `05` AC-007-03 |
| 舆情缓存命中延迟 | < **200ms** | 5 分钟内缓存命中时的响应时间 | `04` R-12 |

### 1.2 降级策略

**示例 — 三级降级链**：

```
CoPawAgent.ask(query, session_id)
  ├─[1] CoPaw 桥接 → answer_source='copaw', llm_used=True
  ├─[2] 百炼 DashScope → answer_source='bailian', llm_used=True
  └─[3] 离线演示兜底 → answer_source='demo', llm_used=False
```

| 约束 | 说明 | 来源 |
|------|------|------|
| 三级降级**不可跳级** | 必须按 CoPaw → 百炼 → Demo 顺序尝试 | `03` §5 |
| 降级**静默执行** | 不抛异常，通过 `answer_source` 标识 | `03` §5 |
| Demo 模式**始终可用** | 无外部依赖，纯字符串拼接 | `03` §5 |
| 降级超时阈值 | CoPaw 20s / 百炼 120s，超时即触发降级 | `08` §4 三级降级编排 |

## 2. 可用性需求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 服务可用性 | **99%**（教学环境） | 单实例 Flask，不设 HA |
| MTTR | < 5 分钟 | 重启 Flask 进程即可恢复 |

### 2.1 故障模式与恢复方式

| 故障模式 | 触发条件 | 系统行为 | 恢复方式 | 来源 |
|----------|----------|----------|----------|------|
| CoPaw 服务不可用 | `IRA_COPAW_*_URL` 配置缺失或请求超时 20s | 静默降级至百炼，标识 `answer_source='bailian'` | 恢复 CoPaw 网络/配置后自动恢复 | `03` §5 |
| 百炼服务不可用 | `DASHSCOPE_API_KEY` 配置缺失或请求异常 | 静默降级至 Demo，标识 `answer_source='demo'` | 恢复百炼 API Key/网络后自动恢复 | `03` §5 |
| 三级全失败 | CoPaw + 百炼 + Demo 均异常 | 返回 500 `LLM_SERVICE_ERROR` | 检查 Demo 模式实现与本地存储状态 | `09` §2 错误码清单 |
| JSON 文件损坏 | 存储文件被意外修改或删除 | 启动时检测，返回 503 `SERVICE_DEGRADED` | 从备份恢复或重建数据文件 | `08` §5.1 |
| 存储目录不可写 | `./data/` 目录权限不足或磁盘满 | 写操作失败，抛异常由 Route 层映射 503 | 修复目录权限或清理磁盘空间 | `03` §6 演示数据目录可读写 |
| 会话不存在 | 请求携带已删除的 `session_id` | 返回 404 `SESSION_NOT_FOUND` | 用户刷新会话列表后重试 | `04` R-01 / `09` §2 |
| RSS 数据源全部不可用 | 所有 RSS 源请求超时或返回错误 | 自动降级到 Mock 数据展示示例新闻 | RSS 源恢复后下次轮询自动恢复 | `04` R-13 |

## 3. 可观测性需求

### 3.1 链路追踪

| 项 | 规格 |
|----|------|
| traceId 格式 | `tr_{uuid.hex}`（32 位十六进制） |
| 注入位置 | 所有成功响应顶层 + 错误响应 `error.traceId` |
| 上游透传 | 优先复用请求头 `X-Trace-Id`，缺省时本地生成 |

### 3.2 错误码体系

| HTTP | `error.code` | 触发条件 | 来源 |
|------|--------------|----------|------|
| 400 | `EMPTY_QUERY` | query 为空/空白/null | `04` R-02 |
| 400 | `INVALID_QUERY` | query >500 字符 或 缺少 session_id | `04` R-02 |
| 400 | `MISSING_SESSION_ID` | session_id 缺失或为空 | `04` R-01 / `09` §2 |
| 404 | `SESSION_NOT_FOUND` | 指定 session_id 不存在 | `04` R-01 / `09` §2 |
| 404 | `DOC_CHUNK_NOT_FOUND` | 指定引用文档块不存在 | `04` R-07 / `09` §2 |
| 500 | `LLM_SERVICE_ERROR` | 三级降级均失败 | `04` R-04 / `09` §2 |
| 500 | `UPSTREAM_ERROR` | Agent 编排内部异常 | 模板 |
| 503 | `SERVICE_DEGRADED` | 后端核心服务异常 | `09` §2 |

## 4. 安全需求

| 项 | 当前实现 | 说明 |
|----|----------|------|
| API 认证 | 无（开放访问） | Workshop 教学版 |
| CORS | 全开放 | 生产需白名单 |
| 速率限制 | 无 | 教学场景不限速 |

### 4.1 敏感数据保护

| 数据类型 | 保护方式 | 来源 |
|----------|----------|------|
| LLM API Key | 环境变量存储（`.env`），禁止硬编码 | `08` §8 安全架构 |
| 用户查询 | 仅存储在本地 JSON，不传输到第三方（LLM 除外） | `05` US-002 设计对齐 |
| 会话历史数据 | 本地文件系统存储，不经过网络传输，定期备份 `./data/` 目录 | `08` §5.2 |
| 研报源文件 | 静态文件目录只读挂载，禁止上传接口修改 | `03` §3 "不做：交易指令" |
| traceId 日志 | 仅记录链路追踪 ID，不记录用户身份敏感信息 | `09` §2 统一响应规范 |

## 5. 数据约束

### 5.1 输入约束

| 约束 | 规格 | 来源 |
|------|------|------|
| `query` 最小长度 | 1 字符 | `04` R-02 |
| `query` 最大长度 | 500 字符 | `04` R-02 |
| `session_id` 必填 | 非空字符串，UUID 格式 | `04` R-01 |
| `title` 默认值 | "新会话"，首次问答后自动重命名为 query 前 20 字 | `04` R-05 |
| `title` 最大长度 | 100 字符 | `09` §4 POST /sessions |
| `references` 非空校验 | 当 `answer_source` 为 `copaw`/`bailian` 且基于原文提取时，`references` 不得为空数组 | `04` R-07 |

### 5.2 类型约束

| 字段 | 类型 | 约束 | 来源 |
|------|------|------|------|
| `llm_used` | `boolean` | 必须为 True/False，不得为字符串 | `04` R-03 |
| `answer_source` | `string` | 枚举：copaw \| bailian \| demo | `04` R-03 / `04` R-04 |
| `session_id` | `string` | UUID v4 格式（8-4-4-4-12） | `09` §4/§5/§6/§7 |
| `record_id` | `string` | 格式 `rec_{timestamp}`，timestamp 为毫秒级 Unix 时间戳 | `10` §4 QARecord |
| `chunk_id` | `string` | 格式 `chk_{hash}`，hash 为文档内容短哈希 | `10` §4b DocChunk |
| `query_count` | `integer` | ≥ 0，首次问答时从 0 递增至 1 触发自动重命名 | `04` R-05 / `10` §6 |
| `response_time_ms` | `integer` | ≥ 0，单位毫秒 | `04` R-03 |
| `page` | `integer` | ≥ 1，PDF 页码从 1 开始计数 | `09` §9 / `10` §4b |
| `created_at` / `updated_at` / `timestamp` | `string` | ISO 8601 格式，UTC 时区，带 Z 后缀 | `09` §4/§7 |
| `references` | `array` | 元素为对象，包含 chunk_id/doc_title/page/highlight_text | `09` §3 / `10` §4 |
| `bbox` | `object\|null` | 若存在，必须包含 x/y/width/height 四个 number 字段 | `09` §9 / `10` §4b |
