## 1. 安全目标与合规要求

> 明确本模块需遵循的安全目标和外部合规标准。投研场景尤其关注数据隐私与内部信息防泄露。

| 安全目标 | 说明 | 对应合规 |
|----------|------|----------|
| 数据不出域 | 所有用户数据、研报内容、问答记录禁止传输至外部服务（LLM API 除外，但需经审批） | 内部信息安全规范 |
| 最小权限原则 | 每个组件只拥有完成任务所需的最低权限 | 安全设计最佳实践 |
| 可追溯可审计 | 所有数据访问和操作行为可记录、可追溯 | 金融行业合规要求 |

| 项目 | 规格 |
|------|------|

## 3. 数据安全

> 定义数据分类分级、加密策略、存储安全规范。**投研场景核心关注点**。

### 3.1 数据分类与分级

| 数据类型 | 安全等级 | 示例 | 保护措施 |
|----------|----------|------|----------|
| 研报原文 | **机密** | PDF 研报文件 | 加密存储，访问审计 |
| 问答记录 | **内部** | 用户提问与AI回答 | 存储加密，定期清理 |
| 用户信息 | **内部** | 用户名、角色、操作日志 | 脱敏展示，最小采集 |
| 系统配置 | 一般 | 模型参数、界面配置 | 版本控制 |
| LLM API Key | **机密** | CoPaw/百炼密钥 | 环境变量，禁止硬编码 |

### 3.2 加密策略

| 场景 | 加密方式 | 说明 |
|------|----------|------|
| 传输加密 | TLS 1.2+ | 所有 API 通信强制 HTTPS（生产环境） |
| 存储加密 | AES-256 | 研报文件与问答记录落盘加密（生产环境） |
| 密钥管理 | 环境变量 / Vault | 密钥不硬编码，定期轮转 |

## 4. 接口安全

> 与 `09` API 规格配合，定义 API 层面的安全防护措施。

| 防护措施 | 实现方式 | 说明 |
|----------|----------|------|
| 输入校验 | 服务端白名单校验 | 所有用户输入进行类型、长度、格式校验，对齐 `09` §10 |
| SQL 注入防护 | N/A（JSON 文件存储，无 SQL） | 08 §5.1 存储演进至 SQLite/PostgreSQL 时需启用 ORM 参数化查询 |
| XSS 防护 | 输出转义 + CSP | 问答内容渲染时防止脚本注入，对齐 `05` US-002 §防XSS注入 |
| CSRF 防护 | 无（教学版）；Token 校验（生产版） | 状态变更接口需 CSRF Token |
| 速率限制 | 无（教学版）；100 req/min/user（生产版） | 防止接口滥用，对齐 `05` US-005 §接口访问限流 |
| 文件上传校验 | 类型白名单 + 大小限制 | 仅允许 PDF，限制单文件 50MB |
| 研报端点校验 | 文件名白名单 | `GET /reports` 和 `POST /reports/compare` 仅返回 `./data/reports/` 目录下的合法 Markdown 文件，防止路径穿越 |
| RSS 聚合安全 | 输出转义 | `GET /news` 返回的新闻标题/摘要经 HTML 转义，防止 XSS |

## 5. LLM / AI 安全

> 针对大模型集成的专项安全措施。投研场景数据敏感，需严格管控数据流向。

| 风险点 | 防护措施 | 验收标准 |
|--------|----------|----------|
| 数据外泄至第三方模型 | 本地部署模型 / 内网百炼服务，禁止调用外网 API | 网络出口审计零外传记录 |
| Prompt 注入攻击 | 系统 Prompt 隔离，用户输入预处理过滤 | 恶意 Prompt 不触发非预期行为 |
| 模型幻觉 / 错误信息 | 回答标注来源引用，标识置信度 | 每条回答携带 `references` 溯源 |
| 敏感信息泄露（模型回复） | 输出过滤层，正则匹配敏感信息并脱敏 | 手机号、身份证号等不脱敏不出现在回复中 |

## 6. 审计与日志

> 定义安全审计日志的记录范围、保留策略。

| 事件类型 | 记录内容 | 保留期限 | 告警 |
|----------|----------|----------|------|
| 登录 / 登出 | 用户ID、时间、IP、结果 | 180天 | 连续失败 ≥5 次告警 |
| 研报访问 | 用户ID、文档ID、操作类型 | 365天 | 异常批量下载告警 |
| 问答交互 | 会话ID、query 摘要、answer_source | 180天 | 高频调用告警 |
| 系统配置变更 | 变更人、变更前后值 | 永久 | 关键配置变更即时通知 |

## 7. 安全威胁清单（STRIDE 简表）

> 基于 STRIDE 模型，列出主要安全威胁与应对方案。

| 类型 | 威胁 | 风险等级 | 应对措施 | 负责人 |
|------|------|----------|----------|--------|
| S | 伪造用户身份访问研报 | **高** | 强认证 + Token 校验 | 后端开发 |
| T | 研报内容在传输中被篡改 | 中 | TLS + 文件哈希校验 | 运维 |
| R | 内部数据通过AI模型泄露 | **高** | 本地部署 + 输出过滤 | 架构师 |
| I | 未授权访问其他用户会话 | **高** | 会话级权限校验，仅本人可见 | 后端开发 |
| D | 大量请求导致服务不可用 | 中 | 速率限制 + 降级策略 | 后端开发 |
| E | 日志文件泄露敏感信息 | 中 | 日志脱敏，定期归档 | 运维 |

## 8. 部署安全

> 定义部署架构中的安全要求。

- [ ] 生产/开发环境**严格隔离**，数据不互通
- [ ] 数据目录访问仅限应用服务器用户，禁止其他用户读取 `./data/`
- [ ] 敏感配置（API Key、密码）通过环境变量注入，不入代码仓库
- [ ] 依赖包定期安全扫描（`pip audit` / `npm audit`）
- [ ] 容器镜像使用非 root 用户运行
- [ ] 日志文件设置 640 权限，仅所有者可读写
