## 1. 架构总览

### 1.1 系统上下文

```
┌─────────────────────────────────┐
│       用户浏览器 React SPA       │
│       http://localhost:5173     │
└──────────────┬──────────────────┘
               │ HTTP  /api/v1/agent/*
               │       /reports, /news
               ▼
┌─────────────────────────────────────┐
│   Express Backend (server.js)       │
│   http://localhost:5000             │
│  ┌────────────────────────────────┐ │
│  │  11 个 API 端点 + Swagger UI  │ │
│  │  会话管理 · 问答(SSE) · 溯源  │ │
│  │  研报对比 · 舆情监控 · 健康    │ │
│  └───┬──────────┬──────────┬─────┘ │
│      ▼          ▼          ▼       │
│  ┌────────┐ ┌──────────┐ ┌──────┐ │
│  │ LLM 层 │ │ 文件系统  │ │ RSS  │ │
│  │CoPaw   │ │ ./data/  │ │聚合器│ │
│  │百炼    │ │ *.json   │ │      │ │
│  │Demo    │ │ reports/ │ └──────┘ │
│  └────────┘ └──────────┘          │
├───────────────────────────────────┤
│   Flask Backend (wsgi.py) 备选    │
│   agent_bp.py · agent.py          │
│   storage.py · copaw/bailian      │
└───────────────────────────────────┘
```

### 1.2 架构风格

> 选择架构风格时思考：教学场景需要什么样的复杂度？分层带来什么好处？

| 决策 | 选型 | 理由 |
|------|------|------|
| **前后端分离** | React SPA + Flask REST API | 教学清晰度：前端独立开发服务器，职责边界明确 |
| **单体部署** | 前后端各一个进程 | 不引入微服务复杂度，适合投研助手当前体量 |
| **分层架构** | Route → Agent → Storage | 替换存储引擎时上层零改动，职责隔离清晰 |
| **流式通信** | Server-Sent Events（SSE） | 问答为服务端单向推流场景（对齐 US-002 AC-002-03），SSE 比 WebSocket 轻量：无需握手升级、浏览器 `EventSource` 原生支持、Flask `Response(generator)` 即可实现；不需要客户端→服务端的反向实时通道 |

## 2. 后端分层结构

> 每层写清"允许做什么、禁止做什么"。Express 为主入口，Flask 作为 Python 备选后端。

### 2.1 Express 主后端（server.js）

| 层 | 职责 | 说明 |
|----|------|------|
| **路由层** | 11 个 API 端点注册、参数校验、SSE 流式响应、错误码映射 | 包含会话/问答/溯源/研报对比/舆情/健康全部端点 |
| **RAG 检索** | 关键词加权匹配 doc_chunks.json 提取相关文档块 | TF 加权：精确匹配(10) + 分词模糊(3) + 字符级(1) |
| **百炼集成** | 调用 DashScope OpenAI 兼容 API | qwen-turbo 模型，超时 120s |
| **研报解析** | 从 Markdown 研报中提取多维度数据 | parseReport 函数：评级/目标价/财务/观点/风险 |
| **RSS 聚合** | rss-parser 聚合多个财经 RSS 源 | 5 分钟缓存 + 优先级自动评分 |
| **Swagger** | swagger-ui-express 提供 `/api-docs` | API 文档自动生成 |

### 2.2 Flask 备选后端

| 层 | 文件 | 允许做 | 禁止做 |
|----|------|--------|--------|
| **Route** | `agent_bp.py` | 参数校验、HTTP 状态码、traceId 生成、SSE 流封装 | 直接读写文件、调用 LLM |
| **Agent** | `agent.py` | 三级降级编排、结果组装、引用出处拼接 | 感知 HTTP 请求/响应 |
| **Provider** | `copaw_bridge.py` / `bailian_qa.py` | HTTP 调用外部 LLM API、超时控制 | 操作本地存储 |
| **Storage** | `storage.py` | 文件 CRUD、数据校验、JSON 序列化/反序列化 | 调用外部 API |

## 3. 前端架构

| 项 | 选型 | 理由 |
|----|------|------|
| UI 框架 | React 19（Hooks）| 最新稳定版，并发特性支持 |
| 构建工具 | Vite 8 | 极快 HMR，零配置启动 |
| 状态管理 | `useState` + `useEffect` | 9 个 state 变量足够（含 activeView） |
| HTTP 客户端 | `fetch` API | 浏览器原生，不引入 axios |
| Markdown 渲染 | react-markdown | 问答结果 Markdown 格式化展示 |
| 样式方案 | 自定义 CSS 变量体系 | 南方基金配色（主色 #0844BE），CSS 变量定义颜色/阴影/圆角/动画全套设计令牌；无需引入 Tailwind 等第三方框架，保持样式一致性与品牌还原度 |

## 5. 数据存储架构

### 5.1 存储演进路线

| 阶段 | 存储 | 变更范围 | 说明 |
|------|------|----------|------|
| **P0（当前）** | JSON 文件 | — | 适合教学，零外部依赖，`./data/*.json` 目录即数据库 |
| P1 | SQLite（`ira.db` 单文件） | 仅重写 `storage.py`，上层 Agent/Route 零改动 | 获得事务与并发读写能力，仍为单文件部署，无需运维数据库实例 |
| P2 | PostgreSQL | `storage.py` 改用 SQLAlchemy ORM + 新增数据库连接配置与迁移脚本 | 生产级方案：支持多实例水平扩展、行级锁、全文检索；需配套 Alembic 迁移管理 |

### 5.2 研报数据存储

| 项 | 说明 |
|----|------|
| 存储格式 | Markdown 文件，存放于 `./data/reports/` 目录 |
| 解析方式 | `parseReport()` 函数从 Markdown 中提取结构化数据（评级/目标价/财务预测等） |
| 解析结果 | 不持久化，每次请求实时解析（研报数量少，性能可接受） |

## 5b. RSS 聚合架构

| 项 | 说明 |
|----|------|
| 聚合引擎 | rss-parser npm 包 |
| 数据源 | 路透社商业/科技、CNBC、MarketWatch 等国际财经 RSS 源 |

## 7. ADR（架构决策记录）

> 每条 ADR 记录一个"为什么选 A 而不选 B"的决策。至少写 2 条。

| ADR | 决策 | 核心理由 |
|-----|------|----------|
| ADR-001 | Flask 而非 FastAPI | 学习曲线低，无需 async/await，学员 Python 基础即可上手 |
| ADR-002 | P0 选 JSON 文件存储而非 SQLite | 零依赖启动：无需安装数据库驱动，`json.load/dump` 即完成读写，教学调试时可直接用文本编辑器查看数据内容；分层架构已预留 `storage.py` 抽象层，后续迁移至 SQLite/PostgreSQL 时上层代码零改动 |
| ADR-003 | 前端不引入全局状态库（Redux / Zustand）| 投研助手全局状态仅 `sessions`、`currentSession`、`messages`、`isLoading`、`activeView` 等约 9 个变量，组件树深度 ≤ 3 层，`useState` + props 透传完全可控；引入 Redux 反而增加 boilerplate 和学习负担，违背教学简洁性原则 |
| ADR-004 | Express (Node.js) 替代 Flask 作为主后端 | Express 5.x 天然支持 SSE 流式响应和异步 I/O，RSS 聚合等 I/O 密集操作性能更优；npm 生态直接集成 swagger-ui-express、rss-parser 等包；Flask 保留为 Python 备选后端，供教学对比和 Agent 编排演示 |
| ADR-005 | 自定义 CSS 变量替代 Tailwind CSS | 项目需严格还原南方基金配色体系（主色 #0844BE），CSS 变量定义完整的设计令牌（颜色梯度/阴影/圆角/动画/渐变），品牌一致性要求高于开发速度；全局 index.css 一处定义、全局复用，维护成本可控 |

## 8. 安全架构

> 对齐 `07` 4 安全需求。当前教学版与生产需求的差距。

| 层 | 当前措施 | 生产改进 |
|----|----------|----------|
| 认证 | 无（开放访问） | JWT |
| CORS | 全开放 | 白名单 |
| 密钥 | 环境变量 `.env` | Vault / Secrets Manager |
| 授权 | 会话按用户维度隔离（对齐 US-001 §仅本人可见授权） | RBAC 行级数据权限 |
| 输入过滤 | `query` 长度 ≤500 + 空白检测（对齐 US-002 AC-002-01 防空提交） | 完整 XSS / SQL 注入过滤 + HTML 转义（对齐 09 §防XSS注入） |
| 文档越权 | `chunk_id` 存在性校验 | 文档级 ACL 鉴权，仅授权用户可访问对应研报（对齐 US-003 §防越权访问文档） |
| 接口限流 | 无 | IP / 用户级令牌桶限流，防止高频刷取问答额度（对齐 US-005 §接口访问限流） |
| 日志审计 | `traceId` 链路追踪写入控制台 | 全量请求日志持久化 + 异常操作告警 |
