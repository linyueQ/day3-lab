## 二、排版与字体

### 2.1 字体栈

```css
font-family: "Helvetica Neue", Arial, "PingFang SC", "Hiragino Sans GB",
             "Microsoft YaHei", "WenQuanYi Micro Hei", sans-serif;
```

### 2.2 字号层级

| 层级 | 字号（近似） | 使用场景 |
|------|-------------|---------|
| 超大标题 | 32–36px | 营销大字（"专属基金方案"） |
| 大数字 | 28–32px | 资产总额（81,282.36） |
| 市场指数 | 24–28px | 上证指数数值（4045.46） |
| 区块标题 | 18–20px | "热门话题"等区块标题 |
| 正文/导航 | 14–16px | Tab文字、正文内容、链接 |
| 辅助说明 | 12–13px | 涨跌幅百分比、日期、状态提示 |

### 2.3 文字风格

- **金融数字**：等宽字体渲染，保持数据列对齐
- **涨跌标识**：绿色正值前加 `+`，红色负值前加 `-`
- **标题**：加粗（`font-weight: 600–700`），无衬线
- **链接**：品牌蓝，hover 时加下划线或变色

┌────────────────────────────────────────────────────┐

## 五、视觉风格特征

### 5.1 整体风格

| 特征 | 描述 |
|------|------|
| 设计风格 | 简洁专业的金融社区风格，信息密度适中 |
| 视觉基调 | 白底蓝调，干净清晰 |
| 信息层次 | 通过字号、颜色、间距创建明确的视觉层级 |
| 装饰元素 | 极少，仅在 Hero 区域有浅蓝渐变和抽象插画 |

### 5.2 金融数据展示规范

| 规范 | 说明 |
|------|------|
| 涨跌颜色 | **绿涨红跌**（A股国内习惯，雪球使用：绿色=正收益，红色=负收益） |
| 数字格式 | 千位逗号分隔，保留 2 位小数 |
| 百分比 | 前缀 `+/-` 号，保留 2 位小数 |
| 货币 | "元"或"¥"标注，大额数据加粗 |

### 5.3 交互风格

| 交互 | 描述 |
|------|------|
| Tab 切换 | 平滑下划线过渡 |
| 链接 Hover | 颜色加深或出现下划线 |
| 按钮 Hover | 背景色轻微加深 |
| 输入框 Focus | 底部线条变为品牌蓝 |
| 卡片 Hover | 轻微阴影加深（部分场景） |

## 七、关键设计要点总结

1. **蓝白基调**：以品牌蓝 + 白色为主，营造专业可信的金融社区氛围
2. **绿涨红跌**：与国内A股行情色系一致（注意：与部分国际平台的红涨绿跌相反）
3. **卡片化布局**：内容以白色圆角卡片承载，层次分明
4. **信息密度适中**：金融数据展示紧凑但不拥挤，关键数字放大加粗
5. **简洁交互**：Tab下划线、轻阴影悬停等微交互，不喧宾夺主
6. **响应式结构**：固定最大宽度 ~1200px，主内容 + 侧栏双栏布局
