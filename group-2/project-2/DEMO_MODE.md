# 演示模式使用说明

## 功能说明

当 URL 带有 `?demo=1` 参数时,前端将使用 mock 数据进行演示,无需启动后端服务。

**注意**: 基金智投助手(聊天功能)仍然使用真实 API,不使用 mock 数据。

## 使用方式

### 1. 正常模式(使用真实后端)
```
http://localhost:5173/
```

### 2. 演示模式(使用 mock 数据)
```
http://localhost:5173/?demo=1
```

## Mock 数据覆盖范围

### ✅ 使用 mock 数据的功能
- **基金行情**: 大盘指数、基金列表
- **基金诊断**: 基金基本信息、收益数据、持仓资讯
- **基金视点**: 资讯文章列表、热门标签

### 🔄 本身已使用Mock的功能
- **我的自选**: 通过 `watchlist.json` 文件持久化，不受 demo=1 影响

### ❌ 不使用 mock 数据的功能
- **基金智投助手**: 仍然调用真实的后端聊天 API

## Mock 数据说明

### 基金行情
- 4 个大盘指数(上证指数、深证成指、创业板指、上证50)
- 8 只基金(混合型、指数型)

### 自选列表
- 通过 `backend/data/watchlist.json` 持久化存储
- 默认包含基金代码，实际数据从行情接口获取

### 基金诊断
- 综合评级: 4星(优秀)
- 5个维度评分: 收益能力85、风控能力78、稳定性82、择时能力75、选股能力88
- 历史收益数据、前十大持仓、投资建议

### 基金视点
- 5篇资讯文章,涵盖市场展望、行业动态、投资策略、产品分析
- 14个热门标签

## 演示模式提示

进入演示模式后,页面顶部会显示蓝色提示条:
```
🧪 演示模式 (基金智投助手使用真实数据,其他功能使用模拟数据)
```

## 注意事项

1. **自选功能**: 本身已使用 `watchlist.json` 文件存储，添加/移除操作会持久化到文件，与 demo=1 无关
2. **基金诊断**: 演示模式下所有基金代码都会返回相同的 mock 诊断数据
3. **后端依赖**: 如果需要使用基金智投助手，仍然需要启动后端服务

## 启动步骤

### 仅前端演示(不需要后端)
```bash
cd frontend
pnpm install
pnpm dev
```
然后访问: `http://localhost:5173/?demo=1`

### 完整功能(需要后端)
```bash
# 启动后端
cd backend
source venv/bin/activate  # 或 venv\Scripts\activate (Windows)
python app.py

# 启动前端
cd frontend
pnpm dev
```
然后访问: `http://localhost:5173/` 或 `http://localhost:5173/?demo=1`
