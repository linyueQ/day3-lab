## 1. 概念模型

```
Fund（基金信息）1 ── * FundHolding（持仓股）
     │                      │
     │  fund_code (PK)      │  stock_code (PK)
     │  fund_name            │  stock_name / industry / weight
     │  fund_type            │
     │  manager / company    │
     │  nav / scores         │
     └                      └ ── * HoldingNews（持仓股资讯）
                                   │
                                   │  id (PK)
                                   │  sentiment / title / source
                                   └  published_at

ChatSession（问答会话）1 ── * ChatMessage（消息记录）
     │                           │
     │  session_id (PK)          │  id (PK)
     │  created_at               │  session_id (FK)
     │  message_count            │  role / content
     └  updated_at               └  timestamp

Watchlist（自选基金）
     │
     │  fund_code (PK)
     │  fund_name / fund_type
     │  cost / shares
     └  added_at
```

## 2. 存储引擎

### 2.1 P0 阶段（当前）

| 项 | 说明 |
|----|------|
| 引擎 | 前端 Mock 数据（组件内硬编码） |
| 编码 | TypeScript 对象字面量 |
| 持久化 | 无（刷新即重置） |
| 并发安全 | 不涉及（纯前端） |

### 2.2 P1 阶段（真实数据）

| 项 | 说明 |
|----|------|
| 主数据库 | PostgreSQL 15+ |
| 缓存 | Redis 7+（行情数据、热门资讯） |
| 文件存储 | 对象存储（资讯封面图等） |
| 编码 | UTF-8 |

## 3. 实体定义 — Fund（基金信息）

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `fund_code` | `string` | 是 | **PK**，6位数字 | 基金代码 |
| `fund_name` | `string` | 是 | ≤50 字符 | 基金名称 |
| `fund_type` | `string` | 是 | 枚举 | 混合型/指数型/股票型/债券型 |
| `manager` | `string` | 是 | ≤20 字符 | 基金经理 |
| `company` | `string` | 是 | ≤30 字符 | 基金公司 |
| `scale` | `string` | 是 | — | 基金规模（如"126.80亿"） |
| `nav` | `number` | 是 | > 0 | 最新净值 |
| `nav_change` | `number` | 是 | — | 日涨跌幅（%） |
| `rating` | `integer` | 是 | 1-5 | 综合评级 |
| `rating_label` | `string` | 是 | 枚举 | 优秀/良好/一般 |
| `scores` | `object` | 是 | — | 5维评分 |
| `scores.returns` | `integer` | 是 | 0-100 | 收益能力 |
| `scores.risk_control` | `integer` | 是 | 0-100 | 风控能力 |
| `scores.stability` | `integer` | 是 | 0-100 | 稳定性 |
| `scores.timing` | `integer` | 是 | 0-100 | 择时能力 |
| `scores.stock_picking` | `integer` | 是 | 0-100 | 选股能力 |

## 12. 关键业务逻辑

| 逻辑 | 触发条件 | 行为 |
|------|----------|------|
| 自选基金去重 | POST /watchlist 时 fund_code 已存在 | 返回 409 DUPLICATE_WATCHLIST |
| 会话自动清理 | 会话超过 30 天未更新 | 定时任务清理过期会话及消息 |
| 情感分析缓存 | 同一新闻 24 小时内重复请求 | 返回缓存结果，不重复分析 |
| 持仓汇总计算 | GET /watchlist | 实时计算总市值 = Σ(nav × shares) |

## 13. API 响应 DTO 概要

| API 端点 | 核心 DTO 字段 | 来源实体 |
|----------|---------------|----------|
| `GET /fund/diagnosis` | fund_code, fund_name, rating, scores | Fund |
| `GET /fund/returns` | returns, holdings, advice | FundReturns + FundHolding |
| `GET /fund/holdings-news` | news[] | HoldingNews |
| `GET /market/indices` | indices[] | MarketIndex |
| `GET /market/funds` | funds[] | Fund（列表视图） |
| `GET /insights/articles` | articles[], hot_articles, hot_tags | Article |
| `POST /chat/ask` | answer, session_id, llm_used | ChatSession + ChatMessage |
| `GET /watchlist` | summary, funds[] | Watchlist + Fund |
| `POST /watchlist` | fund_code, message | Watchlist |
| `DELETE /watchlist/{code}` | fund_code, message | Watchlist |
