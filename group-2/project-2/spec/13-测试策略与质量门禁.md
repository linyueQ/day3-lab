## 一、测试分层

| 层级 | 工具 | 覆盖对象 | 说明 |
|------|------|----------|------|
| **L1 Unit** | Vitest | 工具函数、数据处理逻辑 | 不依赖 DOM |
| **L2 Component** | Vitest + React Testing Library | React 组件渲染与交互 | 模拟用户操作 |
| **L3 Integration** | Vitest + MSW（Mock Service Worker） | 前端 + Mock API 联调 | 模拟 HTTP 请求/响应 |
| **L4 E2E** | Playwright | 完整用户旅程 | 浏览器级操作 |

## 二、用例拆分原则

> **核心理念：数"可独立失败的点"** — 每条 TC 只回答一个问题，失败时不看代码就知道哪里坏了。

| 来源 | 计数方法 | 示例 |
|------|----------|------|
| 正常路径 | 每个页面/端点的 happy path = 1 条 | 基金诊断搜索成功 = TC-001 |
| 错误码 | 每个 distinct error code = 1 条 | INVALID_FUND_CODE = TC-009 |
| 分支路径 | 每个 if/else 降级分支 = 1 条 | LLM 降级到兜底 = TC-014 |
| 类型约束 | Spec `09` 约定的字段类型 = 1 条 | sentiment 必须为枚举 = TC-015 |

## 四、质量门禁

| Gate-ID | 条件 | 阻塞合并 |
|---------|------|----------|
| **G-LINT** | ESLint + Prettier 通过 | **是** |
| **G-TYPE** | TypeScript 编译零错误 | **是** |
| **G-UNIT** | L1 + L2 测试全绿 | **是** |
| **G-INT** | L3 集成测试全绿 | **是** |
| **G-AC** | P0 验收标准全覆盖 | **是** |
| **G-PERF** | 首屏加载 ≤ 2000ms，问答响应 ≤ 1500ms | 否（warning） |
| **G-AUDIT** | pnpm audit 无 high/critical 漏洞 | **是** |

### 执行命令

```bash
# 前端测试
cd frontend
pnpm test              # Vitest 单元 + 组件测试
pnpm test:e2e          # Playwright E2E 测试
pnpm lint              # ESLint 检查
pnpm type-check        # TypeScript 编译检查
pnpm audit             # 依赖安全扫描
```

## 五、TDD 适用场景

| 场景 | TDD 价值 | 建议 |
|------|----------|------|
| 简单 CRUD（自选增删） | 低 | 写完补测试 |
| 参数校验 / 错误码 | 中 | 建议 TDD |
| LLM 降级 / 多轮上下文 | **高** | **强烈建议 TDD** |
| NLP 情感分析准确率 | **极高** | **必须 TDD** |
| 涨跌幅配色逻辑 | 中 | 建议 TDD |
