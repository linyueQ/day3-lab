## 1. 架构总览

### 1.1 系统上下文

               │ HTTP  /api/*

┌─────────────────────────────────────┐
│       后端 API 服务                  │

│  │  Fund API   — 基金诊断/行情     │ │
│  │  News API   — 持仓股资讯/NLP    │ │

### 1.2 架构风格

| 决策 | 选型 | 理由 |
|------|------|------|
| **前后端分离** | React SPA + REST API | 前端独立开发、热更新，后端独立扩展 |
| **单体后端** | 单进程 API 服务 | 首期不引入微服务复杂度 |
| **分层架构** | Route → Service → Repository | 替换存储引擎时上层零改动 |
| **NLP 服务集成** | 新闻爬虫 + 情感分析 Pipeline | 实时爬取 + NLP 标注情感倾向 |
| **LLM 集成** | 真实大模型 + 多轮上下文记忆 | 提供通俗投资建议，支持连续追问 |

## 2. 后端分层结构

| 层 | 职责 | 允许做 | 禁止做 |
|----|------|--------|--------|
| **Route** | 路由注册、参数校验、HTTP 状态码 | 解析请求、返回响应 | 直接读写数据库、调用 LLM |
| **Service** | 业务逻辑编排、降级策略 | 调用 Repository 和外部服务 | 感知 HTTP 请求/响应 |
| **Repository** | 数据 CRUD、数据校验 | 操作数据库/文件 | 调用外部 API |
| **Provider** | 外部服务集成（LLM/NLP/基金API） | HTTP 调用外部 API | 操作本地存储 |

## 3. 前端架构

| 项 | 选型 | 理由 |
|----|------|------|
| UI 框架 | React 19（Hooks） | 社区生态最完善，组件复用性强 |
| 组件库 | Ant Design 6 | 企业级 UI 组件，金融场景适配度高 |
| 构建工具 | Vite 8 | 极快 HMR，零配置启动 |
| 语言 | TypeScript | 静态类型检查，减少运行时错误 |
| 路由 | react-router-dom v7 | 声明式路由，嵌套布局支持 |
| 状态管理 | `useState` + `useEffect` | 当前状态量级不需要 Redux |
| HTTP 客户端 | `fetch` API | 浏览器原生，不引入额外依赖 |
| 样式方案 | Ant Design Token + CSS-in-JS | 全局主题统一，组件级样式隔离 |
| 国际化 | ConfigProvider zhCN | 中文语言包，开箱即用 |
| 包管理器 | pnpm | 严格依赖提升，安装速度快 |

## 4. LLM 集成架构

```
POST /api/chat/ask
  ├─[1] 真实 LLM 大模型（首选）
  │     - 多轮上下文记忆（session_id 关联历史对话）
  │     - 基金领域 Prompt 模板
  │     - 超时 30s
  ├─[2] 备用 LLM 提供商
  │     - 超时 20s
  └─[3] 兜底规则引擎
        - 预设回复模板
        - 始终可用
```

| 能力 | 实现方式 | 说明 |
|------|----------|------|
| 多轮上下文 | session_id → 历史对话列表注入 Prompt | 保留最近 10 轮对话 |
| 基金知识增强 | 基金诊断数据 + 持仓资讯作为 Context | RAG 式增强回答质量 |
| 回复安全 | 输出过滤层，投资建议免责声明 | 每条回复附带风险提示 |

## 5. NLP 情感分析架构

| 步骤 | 说明 |
|------|------|
| 1. 新闻爬取 | 定时爬取持仓股相关新闻（财经网站/公告） |
| 2. 文本预处理 | 去噪、分词、关键词提取 |
| 3. 情感分析 | NLP 模型标注 positive / negative / neutral |
| 4. 关联存储 | 将情感标签 + 新闻绑定到基金持仓股 |
| 5. API 输出 | GET /api/fund/holdings-news 返回标注结果 |

## 6. 数据存储架构

### 6.1 存储演进路线

| 阶段 | 存储 | 变更范围 | 说明 |
|------|------|----------|------|
| **P0（当前）** | Mock 数据 + JSON 文件 | — | 前端原型验证交互 |

## 8. ADR（架构决策记录）

| ADR | 决策 | 核心理由 |
|-----|------|----------|
| ADR-001 | React 而非 Vue | 团队 React 经验更丰富，Ant Design React 版生态更完善 |
| ADR-002 | Ant Design 而非 Material UI | 中文场景支持更好，金融表格/图表组件丰富 |
| ADR-003 | pnpm 而非 npm/yarn | 严格依赖管理，磁盘占用少，安装速度更快 |
| ADR-004 | 前端 Mock 优先 | 先用 Mock 数据验证交互设计，再对接真实 API，降低返工成本 |
| ADR-005 | TypeScript 而非 JavaScript | 静态类型减少运行时 Bug，IDE 自动补全提升开发效率 |
| ADR-006 | 真实 LLM 而非纯规则引擎 | 用户期望自然语言交互，规则引擎覆盖场景有限 |

## 9. 安全架构

| 层 | 当前措施 | 生产改进 |
|----|----------|----------|
| 认证 | 无（开放访问） | JWT + OAuth2 |
| CORS | 全开放 | 域名白名单 |
| 密钥 | 环境变量 `.env` | Vault / Secrets Manager |
| 输入校验 | 前端 + 后端双重校验 | WAF + 后端校验 |
| LLM 安全 | 输出过滤 + 免责声明 | Prompt 注入防护 + 输出审核 |
| 移动端 | 响应式布局适配 | CSP + HSTS |
