## 1. 安全目标与合规要求

| 安全目标 | 说明 | 对应合规 |
|----------|------|----------|
| 投资建议免责 | 所有 AI 回复附带免责声明，不构成投资建议 | 金融信息合规 |
| 数据不外泄 | 用户自选、问答记录不传输至无关第三方 | 数据隐私保护 |
| 最小权限原则 | 每个组件只拥有完成任务所需的最低权限 | 安全设计最佳实践 |
| 输入安全 | 所有用户输入经过校验和过滤 | OWASP Top 10 |

## 2. 认证与授权

### 2.1 认证机制

| 项目 | 规格 |
|------|------|
| 认证方式 | P0 阶段：无认证（开放访问）；P1 阶段：JWT Token |
| Token 有效期 | P1：Access Token 30min，Refresh Token 7d |
| 密码策略 | P1：最少 8 位，含大小写 + 数字 |
| 多因素认证 | P2：敏感操作（大额交易建议）触发 |

### 2.2 角色与权限矩阵

| 角色 | 基金诊断 | 行情浏览 | 资讯阅读 | AI 问答 | 自选管理 | 系统管理 |
|------|----------|----------|----------|---------|----------|----------|
| 游客（P0） | ✓ | ✓ | ✓ | ✓ | ✓ | ✗ |
| 注册用户（P1） | ✓ | ✓ | ✓ | ✓ | ✓ | ✗ |
| 管理员（P1） | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |

## 3. 数据安全

### 3.1 数据分类与分级

| 数据类型 | 安全等级 | 示例 | 保护措施 |
|----------|----------|------|----------|
| 用户问答记录 | **内部** | 用户提问与 AI 回答 | 服务端存储，定期清理 |
| 自选基金数据 | **内部** | 持仓代码、成本、份额 | 用户隔离，不可跨用户访问 |
| 基金诊断结果 | 一般 | 评级、评分数据 | 附免责声明 |
| LLM API Key | **机密** | 大模型服务密钥 | 环境变量存储，禁止硬编码 |
| 系统配置 | 一般 | 主题、路由配置 | 版本控制 |

### 3.2 加密策略

| 场景 | 加密方式 | 说明 |
|------|----------|------|
| 传输加密 | TLS 1.2+（P1） | 所有 API 通信强制 HTTPS |
| 存储加密 | AES-256（P1） | 用户敏感数据落盘加密 |
| 密钥管理 | 环境变量（P0）→ KMS（P2） | 密钥不硬编码，定期轮转 |

## 4. 接口安全

| 防护措施 | 实现方式 | 说明 |
|----------|----------|------|
| 输入校验 | 服务端白名单校验 | fund_code 6位数字、query ≤500 字符 |
| XSS 防护 | React 自动转义 + CSP | JSX 自动 escape，资讯内容渲染防注入 |
| CSRF 防护 | SameSite Cookie（P1） | 状态变更接口需校验 |
| 速率限制 | 100 req/min/IP（P1） | 防止接口滥用 |
| SQL 注入防护 | 参数化查询 / ORM（P1） | 禁止字符串拼接 SQL |

## 5. LLM / AI 安全

| 风险点 | 防护措施 | 验收标准 |
|--------|----------|----------|
| 数据外泄至第三方模型 | 仅传输用户 query + 基金公开数据，不传输自选持仓 | 审计日志无敏感数据外传 |
| Prompt 注入攻击 | 系统 Prompt 隔离，用户输入预处理过滤 | 注入测试用例通过率 100% |
| 模型幻觉 / 错误信息 | 回答附带免责声明 + "仅供参考" | 每条 AI 回复包含免责文字 |
| 不当投资建议 | 输出过滤层，禁止明确买卖指令 | AI 回复不含"一定要买/卖"等断言 |

## 6. 审计与日志

| 事件类型 | 记录内容 | 保留期限 | 告警 |
|----------|----------|----------|------|
| API 请求 | 端点、参数摘要、响应码、traceId | 30 天 | 5xx 错误率 > 5% 告警 |
| AI 问答 | 用户 query 摘要、llm_used、模型、耗时 | 90 天 | 连续失败 ≥ 3 次告警 |
| 自选基金变更 | 操作类型（添加/删除）、fund_code | 90 天 | — |
| LLM 降级 | 降级原因、从哪级降到哪级 | 30 天 | 降级频率 > 50% 告警 |

## 7. 安全威胁清单（STRIDE 简表）

| 类型 | 威胁 | 风险等级 | 应对措施 |
|------|------|----------|----------|
| **S（仿冒）** | 伪造用户访问自选基金数据 | 中 | P1 引入 JWT 认证 |
| **T（篡改）** | API 请求参数被中间人篡改 | 中 | TLS 加密传输 |
| **I（泄露）** | 用户问答数据通过 LLM 外泄 | **高** | 限制传输内容 + 输出过滤 |
| **D（拒绝服务）** | 大量请求导致服务不可用 | 中 | 速率限制 + 降级策略 |
| **E（提权）** | 普通用户访问管理功能 | 低 | P1 角色权限控制 |
| **R（抵赖）** | 用户否认操作行为 | 低 | 审计日志记录 |
