## 1. 性能需求

### 1.1 延迟 SLO

| 指标 | 目标值 | 度量方式 | 来源 |
|------|--------|----------|------|
| **首屏加载时间** | ≤ **2000ms** | 浏览器 Performance API | `03` §4 成功标准5 |
| **基金诊断端点 P95 延迟** | < **1000ms** | `response_time_ms` | `03` §4 成功标准1 |
| **AI 问答端点 P95 延迟** | < **1500ms** | `response_time_ms` | `03` §4 成功标准3 |
| **行情列表端点延迟** | < **500ms** | 服务端计时 | `03` §4 成功标准2 |
| **自选基金管理端点延迟** | < **300ms** | 服务端计时 | `03` §4 成功标准4 |
| **NLP 情感分析延迟** | < **2000ms** | Pipeline 计时 | `03` §3 范围2 |

### 1.2 前端性能

| 指标 | 目标值 | 说明 |
|------|--------|------|
| Vite HMR 刷新 | < 200ms | 开发体验 |
| 页面切换（路由） | < 100ms | React SPA 无刷新 |
| 列表渲染（100条） | < 500ms | 虚拟滚动备选 |

### 1.3 LLM 降级策略

```
POST /api/chat/ask
  ├─[1] 真实 LLM（首选） → llm_used=true
  ├─[2] 备用 LLM 提供商 → llm_used=true
  └─[3] 兜底规则引擎    → llm_used=false
```

| 约束 | 说明 |
|------|------|
| 三级降级**不可跳级** | 必须按优先级顺序尝试 |
| 降级**静默执行** | 不向用户暴露技术细节 |
| 兜底模式**始终可用** | 无外部依赖 |

## 2. 可用性需求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 服务可用性 | **99%**（开发环境） | 允许计划内维护停机 |
| MTTR | < 5 分钟 | 重启服务即可恢复 |
| 前端容错 | API 失败时展示友好错误提示 | 不白屏、不卡死 |
| 移动端适配 | 375px 最小宽度 | 侧边栏自动折叠 |

### 2.1 故障模式

| 故障 | 影响 | 恢复方式 |
|------|------|----------|
| 后端 API 不可用 | 前端数据无法加载 | Mock 数据兜底 + 错误提示 |
| LLM 服务超时 | 问答无法回复 | 降级到兜底模式 |
| 基金数据 API 异常 | 诊断结果为空 | 展示"暂无数据"提示 |
| 新闻爬虫异常 | 持仓资讯为空 | 展示缓存数据或空状态 |

## 3. 可观测性需求

### 3.1 链路追踪

| 项 | 规格 |
|----|------|
| traceId 格式 | `tr_{uuid.hex}`（32 位十六进制） |
| 注入位置 | 所有成功/错误响应顶层 |
| 前端日志 | `console.warn` 输出 API 异常 + traceId |

### 3.2 错误码体系

| HTTP | `error.code` | 触发条件 |
|------|--------------|----------|
| 400 | `EMPTY_QUERY` | query 为空/空白/null |
| 400 | `INVALID_QUERY` | query > 500 字符 |
| 400 | `INVALID_FUND_CODE` | 基金代码格式不合法 |
| 404 | `FUND_NOT_FOUND` | 基金代码不存在 |
| 409 | `DUPLICATE_WATCHLIST` | 自选基金重复添加 |
| 500 | `LLM_ERROR` | LLM 服务异常 |
| 500 | `UPSTREAM_ERROR` | 上游数据服务异常 |

## 4. 安全需求

| 项 | 当前实现 | 说明 |
|----|----------|------|
| API 认证 | 无（开放访问） | 首期开发版 |
| CORS | 全开放 | 生产需限制域名 |
| 速率限制 | 无 | 生产需限流 |
| 输入校验 | 前端 + 后端双重校验 | 防注入 |

### 4.1 敏感数据保护

| 数据类型 | 保护方式 |
|----------|----------|
| LLM API Key | 环境变量存储，禁止硬编码 |
| 用户自选数据 | 仅存储在本地/服务端，不传输第三方 |
| 问答记录 | 不向外部泄露用户提问内容 |
| 基金诊断结果 | 附加免责声明，不构成投资建议 |

## 5. 数据约束

### 5.1 输入约束

| 约束 | 规格 | 来源 |
|------|------|------|
| `query` 最小长度 | 1 字符 | `04` R-04 |
| `query` 最大长度 | 500 字符 | `04` R-04 |
| `fund_code` 格式 | 6位数字 | `09` §13 |
| `page_size` 最大值 | 100 | `09` §7 |
| 会话标题默认值 | "新会话" | `10` §7 |

### 5.2 类型约束

| 字段 | 类型 | 约束 |
|------|------|------|
| `llm_used` | `boolean` | 必须为 true/false |
| `sentiment` | `string` | 枚举：positive / negative / neutral |
| `rating` | `integer` | 1-5 |
| `scores.*` | `integer` | 0-100 |
| `nav` | `number` | > 0 |

### 5.3 展示约束

| 规则 | 说明 | 来源 |
|------|------|------|
| 红涨绿跌 | 所有涨跌幅正值红色、负值绿色、零值灰色 | `04` R-07 |
| 百分比格式 | 正值带"+"号前缀 | `05` AC-002-02 |
| 时间格式 | 相对时间（如"2小时前"）或 YYYY-MM-DD | 用户友好 |
