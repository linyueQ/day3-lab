## 1. 背景与问题

> 个人投资者和研究人员在基金投资过程中面临信息分散、决策支持不足的困境。

| 痛点 | 来源 |
|------|------|
| 基金健康状况缺乏一站式诊断入口，投资者需在多个平台手动查阅评级、收益、持仓等多维数据 | S-01 会议纪要、S-03 用户访谈 |
| 基金持仓股的利好/利空新闻分散在不同平台，无法与具体基金快速关联，影响投资决策时效性 | S-02 邮件沟通、S-03 用户访谈 |
| 基金行情数据分散，分类筛选和涨跌排序操作繁琐，无法高效找到目标基金 | S-01 会议纪要 |
| 行业资讯阅读体验差，缺乏分类导航和个性化推荐，信息获取效率低 | S-01 会议纪要 |
| 缺少便捷的智能问答入口，普通投资者难以从专业数据中获取直观的买卖建议 | S-02 邮件沟通、S-03 用户访谈 |
| 自选基金管理功能缺失，无法在网页端统一跟踪持仓收益变化 | S-01 会议纪要、S-03 用户访谈 |

## 2. 提案概要

在「基金管家」Web 应用内交付一站式基金投资辅助平台。核心能力包括：基金智能诊断（多维评分 + 持仓股资讯聚合）、基金行情实时浏览（大盘指数 + 基金列表筛选排序）、基金视点资讯阅读（分类浏览 + 搜索 + 互动）、AI 基金问答（全局悬浮对话入口，提供通俗买卖建议）、以及用户自选基金管理（添加/删除 + 持仓收益统计）。首期以前端 Mock 数据验证交互体验，后续对接真实数据 API。

## 3. 范围边界

| 做（范围内） | 不做（范围外） |
|-------------|---------------|
| 基金诊断：输入基金代码/名称，展示综合评级、5维度评分、历史收益、前十大持仓；真实基金数据API对接 |  |
| 持仓股资讯：在诊断结果中聚合展示持仓股利好/利空/中性新闻；实时新闻爬取与NLP情感分析 |  |
| 基金行情：大盘指数卡片 + 基金列表（分类筛选、搜索、排序、收藏） | 实时行情WebSocket推送 |
| 基金视点：资讯列表（分类、搜索、点赞）+ 热门排行 + 标签云 | CMS后台内容管理系统 |
| 基金问答：全局右下角悬浮聊天窗口，快捷提问 + 自由输入，模拟AI回复；接入真实LLM大模型 / 多轮上下文记忆 |  |
| 自选基金：添加/删除自选、持仓汇总（市值/成本/收益率） | 用户注册登录 / 服务端持久化存储 |
| 响应式侧边栏布局，可折叠菜单; 移动端原生适配 |  |

## 4. 成功标准（可度量）

| 标准 | 度量方式 |
|------|----------|
| 基金诊断主路径可走通 | 用户在 30 秒内完成"输入基金代码 → 查看诊断结果 → 浏览持仓资讯"全链路 |
| 基金行情筛选可用 | 用户可通过类型筛选 + 关键词搜索在 10 秒内定位到目标基金 |
| 基金问答交互流畅 | 用户点击悬浮按钮 → 输入问题 → 1.5 秒内收到真实回复 |
| 自选基金管理完整 | 用户可完成"添加自选 → 查看持仓汇总 → 删除自选"全流程 |
| 页面加载性能达标 | 首屏加载时间 ≤ 2 秒（开发环境） |
| 所有页面无控制台报错 | 切换全部5个页面 + 执行核心操作后，浏览器控制台无 Error 级别日志 |

## 5. 风险（简表）

| 风险 | 影响 | 应对 |
|------|------|------|
| Ant Design 组件库升级导致 breaking changes | 页面样式/交互异常 | 锁定 antd@6.x 版本，升级前进行全量回归测试 |
| 基金问答模拟回复覆盖场景有限 | 用户提问超出预设范围时体验下降 | 设置兜底回复，引导用户使用基金诊断功能获取详细分析 |

## 6. 立项前置条件（Go / No-Go）

- ☑ Node.js ≥ 18 环境就绪
- ☑ pnpm 包管理器已安装
- ☑ React + Vite + TypeScript 项目脚手架已初始化
- ☑ Ant Design 6 + @ant-design/icons 已安装
- ☑ react-router-dom 路由已配置
- ☑ 开发服务器可正常启动（localhost:5173）
