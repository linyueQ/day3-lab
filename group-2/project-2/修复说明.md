# 基金诊断模块修复说明

## 修复的问题

### 1. 持仓股近期资讯没有显示数据

**问题原因：**
- 后端 `HOLDINGS_NEWS_DB` 中只有基金代码 `"005827"` 有对应的资讯数据
- 当查询其他基金时，会返回默认的"暂无该基金持仓股相关资讯"提示

**修复方案：**
- 在 `backend/repositories/mock_data.py` 中为更多基金添加了资讯数据：
  - `"003095"` (中欧医疗健康混合A) - 医疗相关持仓股资讯
  - `"161725"` (招商中证白酒指数) - 白酒相关持仓股资讯
- 前端添加了空状态处理，当没有资讯时显示友好提示

### 2. 前十大持仓表格中没有显示行业

**问题原因：**
- AkShare 的 `fund_portfolio_hold_em` 接口不提供行业信息
- 原代码中行业字段被设置为空字符串：`"industry": ""`

**修复方案：**
- 在 `backend/providers/akshare_provider.py` 中添加了 `_get_stock_industry()` 函数
- 该函数使用 `ak.stock_individual_info_em()` 接口获取股票的行业信息
- 修改了 `get_fund_holdings()` 函数，在获取持仓时自动补充行业信息
- 添加了缓存机制，避免重复请求相同股票的行业信息

## 技术细节

### 行业信息获取
```python
def _get_stock_industry(stock_code: str) -> str:
    """获取股票所属行业（带缓存）"""
    if not stock_code:
        return ""
    
    def fetch():
        try:
            # 使用东财个股资料接口获取行业信息
            df = ak.stock_individual_info_em(symbol=stock_code)
            if df is not None and not df.empty:
                # 查找行业相关字段
                industry_row = df[df["item"].str.contains("行业", na=False)]
                if not industry_row.empty:
                    return industry_row.iloc[0]["value"]
            return ""
        except Exception as e:
            print(f"[akshare_provider] get stock industry error for {stock_code}: {e}")
            return ""
    
    return _get_cached(f"stock_industry_{stock_code}", fetch) or ""
```

### 缓存机制
- 行业信息请求带有5分钟缓存，避免频繁请求
- 缓存键格式：`stock_industry_{stock_code}`
- 使用 TTL（Time To Live）机制，超时后自动刷新

## 注意事项

1. 行业信息获取依赖 AkShare 的 `stock_individual_info_em` 接口，需要确保网络连接正常
2. 如果接口调用失败，会返回空字符串，不会影响整体功能
3. 资讯数据目前仍使用 Mock 数据，后续可考虑接入真实资讯 API
4. 缓存机制可能导致行业信息更新延迟，但在基金诊断场景下可接受
