## 一、测试分层

| 层级 | 工具 | 覆盖对象 | 说明 |
|------|------|----------|------|
| **L1 Unit** | pytest | 单个函数/类的逻辑 | 不依赖网络，最快 |
| **L2 Integration** | pytest + Flask test_client | API 端点 + 存储 | 模拟 HTTP 请求 |
| **L3 Contract** | pytest + schema 断言 | 响应体结构与 `09` 一致 | 验字段名、类型 |
| **L4 E2E** | Playwright / Cypress | 完整用户旅程 | 浏览器级操作 |

## 二、用例拆分原则

> **核心理念：数"可独立失败的点"** — 每条 TC 只回答一个问题，失败时不看代码就知道哪里坏了。

| 来源 | 计数方法 | 示例 |
|------|----------|------|
| 正常路径 | 每个端点的 happy path = 1 条 | `POST /reports/upload → 201` = TC-001 |
| 错误码 | 每个 distinct error code = 1 条 | `INVALID_FILE_TYPE` = TC-002 |
| 分支路径 | 每个 if/else 降级分支 = 1 条 | AKShare 不可用 → 降级 = TC-062 |
| 类型约束 | Spec `09` 约定的字段类型 = 1 条 | `rating` 必须为枚举值 = TC-013 |

## 五、TDD 适用场景

| 场景 | TDD 价值 | 建议 |
|------|----------|------|
| 简单 CRUD（研报列表、删除） | 低 | 写完补测试 |
| 参数校验 / 错误码 | 中 | 建议 TDD |
| LLM 解析结果结构校验 | **高** | **强烈建议 TDD** |
| 研报比对逻辑 | **高** | **强烈建议 TDD** |
