### 1.2 架构风格

| 决策 | 选型 | 理由 |
|------|------|------|
| **前后端分离** | React SPA + Flask REST API | 前端独立开发服务器，职责清晰 |
| **单体部署** | 前后端各一个进程 | 不引入微服务复杂度 |
| **分层架构** | Route → Engine → Storage | 替换存储引擎或 LLM 提供商时上层零改动 |
| **数据存储** | 内存数据库 + JSON 文件持久化 + 本地文件系统 | 启动快、查询快、教学环境零外部依赖 |

## 2. 后端分层结构

| 层 | 文件 | 允许做 | 禁止做 |
|----|------|--------|--------|
| **Route** | `report_bp.py`, `kb_bp.py`, `compare_bp.py` | 参数校验、HTTP 状态码、traceId | 直接读写文件、调用 LLM |
| **Parser** | `parser.py` | PDF 文本提取、LLM 结构化解析 | 感知 HTTP 请求/响应 |
| **Comparator** | `comparator.py` | 研报比对逻辑、相似合并、差异计算 | 操作本地存储 |
| **Stock Data** | `stock_data.py` | AKShare API 调用、数据缓存 | 操作本地存储 |
| **Knowledge Base** | `knowledge_base.py` | 按股票聚合数据、观点汇总 | 调用外部 API |
| **Storage** | `storage.py` | 内存 CRUD、JSON 持久化、文件管理 | 调用外部 API |

## 3. 前端架构

| 项 | 选型 | 理由 |
|----|------|------|
| UI 框架 | React（Hooks）>= 18 | 教学通用性最高 |
| 构建工具 | Vite >= 5 | 极快 HMR，零配置启动 |
| 路由 | Tab 导航（无 React Router） | 3 个页面足够，降低复杂度 |
| 状态管理 | `useState` + `useEffect` | 12 个 state 变量可控 |
| HTTP 客户端 | `fetch` API | 浏览器原生，不引入 axios |
| 样式方案 | 原生 CSS | 教学场景无需 CSS 框架 |

## 4. 核心引擎设计

### 4.1 研报解析引擎（Parser）

```
upload(pdf_file)
  ├─[1] PDF 文本提取 (PyPDF2 / pdfplumber)
  ├─[2] LLM 结构化解析 (Prompt → JSON)
  │     → 提取: title, rating, target_price,
  │              key_points, stock_code, stock_name
  ├─[3] 自动打标签 (industry 从 LLM 推断)
  └─[4] 存入知识库 (按 stock_code 聚合)
```

| 组件 | 文件 | 配置 | 错误处理 |
|------|------|------|----------|
| PDF 提取 | `parser.py` | — | 提取失败返回 PARSE_FAILED |
| LLM 解析 | `parser.py` | `LLM_API_KEY` + `LLM_BASE_URL` | LLM 不可用返回 LLM_ERROR |

### 4.2 研报比对引擎（Comparator）

```
compare(report_ids[])
  ├─[1] 从知识库获取各研报解析结果
  ├─[2] 逐字段对比 (rating, target_price, key_points)
  ├─[3] 核心观点相似度计算 (LLM 语义比对)
  ├─[4] 生成合并视图 (相似合并)
  └─[5] 生成差异标注 (不同处高亮)
```

## 5. 数据存储架构

### 5.1 存储方案

| 数据类型 | 存储方式 | 说明 |
|----------|----------|------|
| 研报 PDF 文件 | 本地文件系统 `{DATA_DIR}/reports/` | 按 report_id 命名 |
| 研报元数据 | 内存 dict + JSON 持久化 | `reports.json` |
| 解析结果 | 内存 dict + JSON 持久化 | `parsed_reports.json` |
| 知识库索引 | 内存 dict + JSON 持久化 | `knowledge_base.json` |
| 行情数据缓存 | 内存 dict（不持久化） | TTL 缓存，过期重新拉取 |

### 5.2 存储演进路线

| 阶段 | 存储 | 变更范围 | 说明 |
|------|------|----------|------|
| **P0（当前）** | 内存 + JSON 文件 | — | 启动快，教学适合 |
| P1 | SQLite | 仅替换 Storage 层实现 | 支持并发读、事务 |
| P2 | PostgreSQL + 向量数据库 | Storage 层 + 部署配置 | 生产级 + 语义检索 |

## 6. 外部依赖

| 依赖 | 用途 | 配置 | 降级 |
|------|------|------|------|
| **LLM API** (DashScope/OpenAI) | 研报解析、比对分析 | `LLM_API_KEY` + `LLM_BASE_URL` | 返回 LLM_ERROR |
| **AKShare** | 股票行情数据 | `pip install akshare` | 显示"暂无数据" |
| **PyPDF2 / pdfplumber** | PDF 文本提取 | `pip install` | 返回 PARSE_FAILED |

## 8. ADR（架构决策记录）

| ADR | 决策 | 核心理由 |
|-----|------|----------|
| ADR-001 | 内存数据库 + JSON 持久化而非 SQLite | 启动零延迟，查询极快，JSON 文件可直接查看调试 |
| ADR-002 | 按股票代码聚合知识库而非按研报 | 投研分析以股票为核心维度，符合用户心智模型 |
| ADR-003 | LLM 解析而非规则引擎 | 券商研报格式多样，规则引擎无法覆盖，LLM 泛化能力强 |
| ADR-004 | AKShare 而非付费行情 API | 免费开源，覆盖 A 股数据，教学环境足够 |
| ADR-005 | Tab 导航而非 React Router | 3 个页面，复杂度不值得引入路由库 |

## 9. 安全架构

| 层 | 当前措施 | 生产改进 |
|----|----------|----------|
| 认证 | 无（开放访问） | JWT + SSO |
| CORS | 全开放 | 白名单 |
| 密钥 | 环境变量 `.env` | Vault / Secrets Manager |
| 文件上传 | 类型白名单(PDF) + 大小限制(50MB) | + 病毒扫描 |
| LLM 安全 | Prompt 隔离 | + 输出过滤 |
