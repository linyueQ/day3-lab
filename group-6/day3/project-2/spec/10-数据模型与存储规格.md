## 1. 概念模型

```
Report（研报）1 ── 1 ParsedReport（解析结果）
   │                      │
   │  report_id (PK)      │  report_id (PK/FK → Report)
   │  filename             │  title / rating / target_price
   │  file_path            │  key_points / stock_code / stock_name
   │  upload_time          │  industry / parse_time_ms
   │  parse_status         └  parsed_at
   │
   └──── * ──── Stock（股票知识库，逻辑聚合）
                   │
                   │  stock_code (PK)
                   │  stock_name / industry
                   │  report_ids[] (关联研报)
                   └  recent_summary
```

> **核心关系**：
> - 一个 Report 对应一个 ParsedReport（1:1）
> - 一个 Stock 聚合多个 ParsedReport（1:N，通过 stock_code 关联）

## 2. 存储引擎

| 项 | 说明 |
|----|------|
| 引擎 | 内存 dict + JSON 文件持久化（`json` 标准库） |
| 编码 | UTF-8，缩进 2 空格 |
| 读写模式 | 启动时全量加载到内存 → 内存操作 → 变更时写回 JSON |
| 并发安全 | 单进程内存操作，无锁 |
| 文件布局 | `{DATA_DIR}/reports.json` + `parsed_reports.json` + `knowledge_base.json` |
| PDF 存储 | `{DATA_DIR}/reports/{report_id}.pdf` |

## 3. 实体定义 — Report（研报元数据）

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `report_id` | `string` (UUID) | 是 | **PK** | 研报唯一标识 |
| `filename` | `string` | 是 | 原始文件名 | 用户上传时的文件名 |
| `file_path` | `string` | 是 | 服务端路径 | PDF 存储路径 |
| `parse_status` | `string` | 是 | 枚举：pending/parsing/completed/failed | 解析状态 |
| `upload_time` | `string` (ISO-8601) | 是 | UTC | 上传时间 |

## 4. 实体定义 — ParsedReport（解析结果）

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `report_id` | `string` | 是 | **PK/FK** → Report | 关联研报 ID |
| `title` | `string` | 是 | — | 研报标题 |
| `rating` | `string` | 是 | 枚举：买入/增持/中性/减持/卖出/未提及 | 投资评级 |
| `target_price` | `number\|null` | 是 | 正数或 null | 目标价（元） |
| `key_points` | `string` | 是 | — | 核心观点摘要 |
| `stock_code` | `string` | 是 | 6 位数字 | 股票代码 |
| `stock_name` | `string` | 是 | — | 股票名称 |
| `industry` | `string` | 是 | — | 行业分类 |
| `raw_text` | `string` | 是 | — | PDF 提取的原始文本（用于比对分析） |
| `parse_time_ms` | `integer` | 是 | >= 0 | 解析耗时（毫秒） |
| `parsed_at` | `string` (ISO-8601) | 是 | UTC | 解析完成时间 |

## 5. 实体定义 — Stock（股票知识库，逻辑聚合）

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `stock_code` | `string` | 是 | **PK**，6 位数字 | 股票代码 |
| `stock_name` | `string` | 是 | — | 股票名称 |
| `industry` | `string` | 是 | — | 行业分类 |
| `report_ids` | `string[]` | 是 | — | 关联的研报 ID 列表 |
| `recent_summary` | `string` | 是 | — | 最近观点自动汇总（LLM 生成） |
| `updated_at` | `string` (ISO-8601) | 是 | UTC | 最后更新时间 |

## 6. Storage 方法清单

### 6.1 研报管理

| 方法 | 签名 | 行为 | 关联 TC |
|------|------|------|---------|
| `save_report` | `(report_id, filename, file_path) → dict` | 保存研报元数据，parse_status 初始为 pending | TC-M02-001 |
| `get_reports` | `(filters?) → list` | 返回全部研报，支持按 stock_code/industry/date 筛选 | TC-M02-050 |
| `get_report` | `(report_id) → dict\|None` | 返回单个研报详情 | TC-M02-051 |
| `delete_report` | `(report_id) → None` | 删除研报 + 级联删除解析结果 + 更新知识库 + 删除 PDF | TC-M02-053 |
| `update_report_status` | `(report_id, status) → dict` | 更新解析状态 | TC-M02-010 |

### 6.2 解析结果管理

| 方法 | 签名 | 行为 | 关联 TC |
|------|------|------|---------|
| `save_parsed_report` | `(report_id, parsed_data) → dict` | 保存解析结果 + 更新 parse_status 为 completed + 更新知识库 | TC-M02-011 |
| `get_parsed_report` | `(report_id) → dict\|None` | 返回解析结果 | TC-M02-012 |

### 6.3 知识库管理

| 方法 | 签名 | 行为 | 关联 TC |
|------|------|------|---------|
| `get_stocks` | `() → list` | 返回知识库股票列表 | TC-M02-020 |
| `get_stock_detail` | `(stock_code) → dict\|None` | 返回股票详情（含关联研报 + 观点汇总） | TC-M02-021 |
| `update_stock_summary` | `(stock_code) → dict` | 重新生成该股票的观点汇总 | TC-M02-022 |
| `add_report_to_stock` | `(stock_code, stock_name, industry, report_id) → dict` | 将研报添加到对应股票的知识库 | TC-M02-023 |
| `remove_report_from_stock` | `(stock_code, report_id) → None` | 从知识库移除研报，若无关联研报则删除股票条目 | TC-M02-024 |

## 7. 关键业务逻辑

| 逻辑 | 触发条件 | 行为 |
|------|----------|------|
| 解析后自动入库 | save_parsed_report 成功 | 调用 add_report_to_stock 将研报关联到对应股票知识库 |
| 知识库观点汇总 | 新增/删除研报后 | 重新聚合该股票的所有研报核心观点，生成 recent_summary |
| 级联删除 | 删除 Report | 同步删除 ParsedReport + 更新 Stock.report_ids + 删除 PDF 文件 |
| JSON 持久化 | 每次写操作后 | 将内存数据全量写回对应 JSON 文件 |
