## 1. 安全目标与合规要求

| 安全目标 | 说明 | 对应合规 |
|----------|------|----------|
| 数据不出域 | 研报 PDF、解析结果禁止传输至未授权的外部服务 | 内部信息安全规范 |
| 最小权限原则 | 每个组件只拥有完成任务所需的最低权限 | 安全设计最佳实践 |
| 审计可追溯 | 所有关键操作形成完整链路日志，支持事后审计 | 金融行业审计要求 |
| 研报数据保密 | 券商研报属于内部机密数据，禁止未授权访问或泄露 | 金融信息安全规范 |

## 2. 认证与授权

### 2.1 认证机制

| 项目 | 规格 |
|------|------|
| 认证方式 | 无认证（开放访问）—— 教学版；生产建议对接内部 SSO + JWT Token |
| Token 有效期 | 教学版不适用；生产建议 Access Token 30min，Refresh Token 7d |
| 密码策略 | 教学版不适用；生产建议最少 8 位，含大小写+数字 |

### 2.2 角色与权限矩阵

| 角色 | 研报上传 | 研报阅读 | 研报解析 | 研报比对 | 知识库浏览 | 研报删除 |
|------|----------|----------|----------|----------|-----------|----------|
| 投研分析师 | ✓ | ✓ | ✓ | ✓ | ✓ | ✗ |
| 高级研究员 | ✓ | ✓ | ✓ | ✓ | ✓ | ✗ |
| 研究总监 | ✓ | ✓ | ✓ | ✓ | ✓ | ✗ |
| 系统管理员 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |

## 3. 数据安全

### 3.1 数据分类与分级

| 数据类型 | 安全等级 | 示例 | 保护措施 |
|----------|----------|------|----------|
| 研报原文 PDF | **机密** | 券商研究报告 | 本地存储，访问审计 |
| 解析结果 | **内部** | 评级、目标价、核心观点 | 内存+JSON 存储，不外传 |
| 股票行情数据 | 一般 | PE、PB、市值（公开数据） | 缓存，正常处理 |
| 系统配置 | 一般 | LLM 参数、界面配置 | 版本控制 |

### 3.2 加密策略

| 场景 | 加密方式 | 说明 |
|------|----------|------|
| 传输加密 | TLS 1.2+ | 所有 API 通信强制 HTTPS（教学版可走 HTTP） |
| 存储加密 | 教学版明文 JSON；生产建议 AES-256 | 研报文件与解析结果落盘加密 |
| 密钥管理 | 环境变量（`.env`） | 密钥不硬编码，生产建议使用 KMS |

## 4. 接口安全

| 防护措施 | 实现方式 | 说明 |
|----------|----------|------|
| 输入校验 | 服务端白名单校验 | 所有用户输入进行类型、长度、格式校验 |
| XSS 防护 | 输出转义 + CSP | 研报内容渲染时防止脚本注入 |
| CSRF 防护 | 前后端分离 REST API 无状态 | 教学版不设 CSRF Token |
| 文件上传校验 | 类型白名单 + 大小限制 | 仅允许 PDF，≤ 50MB |
| 路径穿越防护 | 文件名过滤 + 固定存储目录 | 禁止 `../` 路径穿越 |

## 5. LLM / AI 安全

| 风险点 | 防护措施 | 验收标准 |
|--------|----------|----------|
| 数据外泄至第三方模型 | 仅调用指定 LLM API，不传输无关数据 | 网络出口审计零未授权外传 |
| Prompt 注入攻击 | 系统 Prompt 与用户输入隔离，结构化 Prompt 模板 | 注入测试用例通过 |
| 模型幻觉 / 错误信息 | 解析结果提供原文对照，llm_used 标识 | 解析结果可与原文核对 |
| 研报内容通过 LLM 泄露 | 仅传输必要的文本片段，不传输完整 PDF | Prompt 中研报内容截断处理 |

## 6. 审计与日志

| 事件类型 | 记录内容 | 保留期限 |
|----------|----------|----------|
| 研报上传 | traceId、文件名、上传时间 | 365天 |
| 研报解析 | traceId、report_id、解析结果摘要 | 365天 |
| 研报删除 | traceId、report_id、操作时间 | 365天 |
| 研报比对 | traceId、report_ids、比对时间 | 180天 |
| 系统错误 | traceId、error.code、错误详情 | 90天 |

## 7. 安全威胁清单（STRIDE 简表）

| 类型 | 威胁 | 风险等级 | 应对措施 |
|------|------|----------|----------|
| **S**（仿冒） | 伪造用户身份访问研报 | **高** | 生产环境: 强认证 + Token 校验 |
| **T**（篡改） | 研报文件在传输/存储中被篡改 | 中 | TLS + 文件哈希校验 |
| **I**（泄露） | 研报内容通过 LLM API 泄露 | **高** | 指定 LLM 服务 + 最小数据传输 |
| **D**（拒绝服务） | 大量文件上传导致磁盘耗尽 | 中 | 文件大小限制 + 磁盘监控 |
| **E**（抵赖） | 用户操作后否认行为 | 中 | traceId 全链路追踪 + 审计日志 |
| **R**（提权） | 普通用户删除研报 | 中 | RBAC 权限矩阵（生产环境） |

## 8. 部署安全

- [ ] 生产/开发环境**严格隔离**，数据不互通
- [ ] 敏感配置（LLM API Key）通过环境变量注入，不入代码仓库
- [ ] PDF 存储目录设置合适的文件系统权限
- [ ] 依赖包定期安全扫描（`pip audit` / `npm audit`）
- [ ] 研报文件上传路径做目录隔离，防止路径穿越
