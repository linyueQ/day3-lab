## 5. 依赖与风险

**依赖链**：

**S1 研报上传解析** → **S2 知识库管理**（必须先有解析数据才能聚合）→ **S3 研报比对**（依赖知识库中的解析结果）→ **S4 行情数据与收口**

| 风险 | 影响 | 应对 |
|------|------|------|
| LLM API 不可用 | 解析/比对全部降级 | 返回 LLM_ERROR，不影响研报管理和知识库浏览 |
| PDF 格式多样导致文本提取不佳 | 解析结果不准确 | pdfplumber 兜底 + LLM 容错 Prompt |
| AKShare API 限流或不可用 | 行情数据无法展示 | 本地缓存 + 降级显示"暂无数据" |
| 内存数据量过大 | 性能下降 | 初期足够，后续迁移 SQLite |
