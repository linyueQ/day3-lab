### 需求记录

1. **研报管理**：支持 PDF 格式研报上传，自动打标签（股票代码、行业），按标签分类管理
2. **研报解析**：用 LLM 自动提取标题、评级、目标价、核心观点，存入以股票为维度的知识库
3. **研报比对**：输入同一上市公司的多份研报，相似信息合并，差异高亮展示
4. **股票行情数据**：通过 AKShare API 获取股票基本资料、行情、财务数据（PE、PB、市值等）

## S-02 邮件沟通 — 关于投研分析平台的需求建议

| 项 | 内容 |
|---|---|
| 发件人 | 李总监 <li.d@company.com> |
| 收件人 | 王经理 <wang.m@company.com> |
| 时间 | 2026-04-11 09:30 |

**问题1 — 研报阅读效率低：** 每天需阅读 10+ 份研报，平均每份 30-50 页，耗时 2-3 小时。希望系统能自动提取关键信息（标题、评级、目标价、核心观点、股票代码、股票名称），一目了然。

**问题2 — 研报对比困难：** 对比不同券商对同一公司的研报需手动复制粘贴，效率极低。希望系统能自动识别相似观点并合并，差异部分高亮标注，生成对比视图。

**问题3 — 知识沉淀缺失：** 已读过的研报内容无法高效检索和复用。希望以股票为维度建立知识库，自动汇总各券商最新观点。

**问题4 — 股票数据分散：** 查看股票数据需要在多个系统间切换。希望在研报阅读界面直接获取对应股票的 PE、PB、市值等基本面指标。

## S-03 用户访谈 — 高级分析师

| 项 | 内容 |

| 访谈对象 | 陈研究员（高级分析师） |
| 访谈人 | 张产品 |
| 时间 | 2026-04-12 10:00-11:00 |
| 方式 | 线上访谈 |

| 问题 | 回答要点 |
|------|----------|
| Q1：平时工作中最耗时的环节？ | 读研报最耗时。每天扫一遍行业研报找重点，电子版反而不知从哪看起。希望系统自动提取关键信息。 |
| Q2：最希望在研报中获取什么信息？ | ①评级和目标价 → ②核心观点（分析师主要逻辑）→ ③股票代码和名称 → ④数据预测（营收增速、PE等） |
| Q3：同时看多份研报如何对比？ | 各家券商格式不同，需手动整理到 Excel。最期待系统能自动合并相同观点、高亮差异。 |
| Q4：对知识库有什么期望？ | 按股票维度组织。一个股票页面能看到所有相关研报的最新观点汇总，加上行情数据。 |

**总结：** ①自动解析 + 智能比对为核心诉求 ②知识库按股票聚合，结合行情数据 ③数据安全本地存储
