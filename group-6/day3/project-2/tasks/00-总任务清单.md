## 1. 任务总览

本项目按 4 个交付阶段（S1~S4）推进，拆分为 **1 份前端任务 + 3 份后端任务**，每份任务内部包含前后端交互约定。

| 任务文档 | 覆盖范围 | 交付阶段 | 涉及 WBS |
|----------|----------|----------|----------|
| `01-后端Task1-研报上传与解析.md` | 研报上传、PDF 文本提取、LLM 解析、存储层基础 | **S1** | W1(部分), W2, W3 |
| `02-后端Task2-知识库与研报管理.md` | 知识库聚合、观点汇总、研报列表/筛选/删除 | **S2** | W1(部分), W2(部分), W5 |
| `03-后端Task3-研报比对与行情数据.md` | 研报比对引擎、行情数据集成 | **S3 + S4** | W1(部分), W4, W6 |
| `04-前端Task-全部页面与交互.md` | 3 个页面组件 + API 客户端 + 状态管理 | **S1~S4** | W7 |

## 2. 阶段依赖链

```
S1 研报上传与解析
  │  后端Task1: upload + parse + storage 基础
  │  前端Task: ReportManager 上传区 + 解析结果展示
  │
  ▼
S2 知识库与研报管理
  │  后端Task2: knowledge_base + 研报 CRUD + 观点汇总
  │  前端Task: ReportManager 列表区 + KnowledgeBase 页面
  │  依赖: S1 的解析数据作为知识库聚合的数据源
  │
  ▼
S3 研报比对
  │  后端Task3(前半): comparator 引擎
  │  前端Task: ReportCompare 页面
  │  依赖: S2 的知识库中需有同一公司 ≥2 份已解析研报
  │
  ▼
S4 行情数据与收口
     后端Task3(后半): stock_data + AKShare 集成
     前端Task: KnowledgeBase 行情数据区域
     依赖: S2 知识库页面已可用
```

### 3.2 统一响应规范（前后端契约）

**成功响应**：
```json

```

**错误响应**：
```json
{ "error": { "code": "ERROR_CODE", "message": "用户可读消息", "details": {}, "traceId": "tr_..." } }
```

| HTTP | error.code | 前端展示 | 归属后端任务 |

| 400 | `FILE_TOO_LARGE` | "文件大小不能超过 50MB" | Task1 |
| 400 | `COMPARE_MIN_REPORTS` | "至少选择 2 份研报进行比对" | Task3 |

## 4. 关键数据流

### 4.1 研报上传解析流（S1 — Task1 + 前端）

```
[前端] 选择 PDF → POST /reports/upload
    → [后端 Task1] 校验文件类型/大小 → 存储 PDF → 返回 report_id (parse_status=pending)
[前端] 收到 report_id → POST /reports/{id}/parse
    → [后端 Task1] PDF 文本提取 → LLM 结构化解析 → 6 字段入库 → 自动入知识库
    → 返回解析结果 (parse_status=completed)
[前端] 刷新研报列表
```

### 4.2 知识库浏览流（S2 — Task2 + 前端）

```
[前端] 进入知识库页 → GET /kb/stocks
    → [后端 Task2] 返回按股票聚合的列表
[前端] 点击某只股票 → GET /kb/stocks/{code}
    → [后端 Task2] 返回详情 + 关联研报 + 观点汇总
[前端] 展示行情数据 → GET /stocks/{code}/market-data
    → [后端 Task3] AKShare 查询 → 返回 PE/PB/市值（或降级）
```

### 4.3 研报比对流（S3 — Task3 + 前端）

```
[前端] 选择股票 → GET /kb/stocks（获取股票列表）
[前端] 勾选 ≥2 份研报 → POST /reports/compare
    → [后端 Task3] 校验同一公司 → 逐字段对比 → LLM 语义比对
    → 返回 similarities[] + differences[]
[前端] 渲染比对结果视图
```

### 4.4 研报管理流（S2 — Task2 + 前端）

```
[前端] 进入研报管理页 → GET /reports
    → [后端 Task2] 返回研报列表（支持筛选参数）
[前端] 点击删除 → 确认弹窗 → DELETE /reports/{id}
    → [后端 Task2] 级联删除: 解析结果 + 知识库引用 + PDF 文件
[前端] 下载原文 → GET /reports/{id}/file
    → [后端 Task2] 返回 PDF 文件流
```

| 层级 | 工具 | 后端任务覆盖 | 门禁 |

| L1 Unit | pytest | Storage 层 7 条 (Task1+Task2) | G-UNIT 全绿 |

## 9. 风险与降级策略

| 风险 | 影响范围 | 降级方式 | 涉及任务 |
|------|----------|----------|----------|
| LLM API 不可用 | 解析 + 比对功能 | 主模型 Qwen3.5-Plus 异常时自动降级到 GLM-4-Flash，两者均失败时返回 `LLM_ERROR` | Task1, Task3 |
| AKShare API 不可用 | 行情数据展示 | 显示"暂无数据"，source="unavailable" | Task3 |
| PDF 格式特殊 | 解析准确率 | pdfplumber 兜底 + LLM 容错 Prompt | Task1 |
| 内存数据量过大 | 查询性能 | 后续迁移 SQLite（仅替换 Storage 层） | Task1, Task2 |
