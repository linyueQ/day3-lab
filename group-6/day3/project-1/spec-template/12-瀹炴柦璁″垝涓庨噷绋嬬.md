## 1. 里程碑（S1～S4）

**S1 研报管理**：上传/解析/列表/详情 API + 前端研报列表和阅读页 → 满足 US-001, US-002 → 验证 TC-M01-001~012

## 5. 依赖与风险

**示例 — 依赖链**：

**S1 研报管理** → **S2 智能问答**（必须先有研报解析结果才能进行问答）→ **S3 对比与联动**（问答需研报 chunks 向量化，对比需多篇研报存在）→ **S4 增值功能**（收藏/标注/历史依附于研报和问答功能）

| 风险 | 影响 | 应对 |
|------|------|------|
| 上游 LLM（CoPaw/百炼）不可用 | 问答降级 | Demo 模式兜底，三级降级自动切换 |
| PDF 解析失败或提取信息不完整 | 研报无法正常使用 | 显示错误原因，支持重新解析，手动补录关键信息 |
| 股票数据源不可用 | 股票面板/详情页无数据 | 返回缓存数据并提示"数据延迟"，股票快照优先本地缓存 |
| 多研报对比耗时超过1分钟 | 用户体验下降 | 异步生成对比，轮询获取结果，前端显示加载进度 |
| 前端状态管理复杂（上传/解析/会话/收藏） | 状态不一致导致 UI 异常 | 统一状态机设计，关键状态持久化到 Storage 层 |
| 并发上传导致文件存储冲突 | 数据覆盖或丢失 | 文件名加 UUID 前缀，写入前校验文件唯一性 |
