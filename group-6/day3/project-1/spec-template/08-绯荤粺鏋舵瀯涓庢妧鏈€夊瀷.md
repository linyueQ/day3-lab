### 1.2 架构风格

> 选择架构风格时思考：教学场景需要什么样的复杂度？分层带来什么好处？

| 决策 | 选型 | 理由 |
|------|------|------|
| **前后端分离** | React SPA + Flask REST API | 教学清晰度：前端独立开发服务器 |
| **单体部署** | 前后端各一个进程 | 不引入微服务复杂度 |
| **分层架构** | Route → Service → Storage | 替换存储引擎时上层零改动 |
| **RAG 架构** | 向量数据库 + LLM 生成 | 支持语义检索和智能问答 |
| **股票双接口设计** | `/stock/{code}`（完整详情）+ `/stock/{code}/snapshot`（轻量快照） | US-006 研报联动仅需关键指标（快照减少传输量）；US-007 独立查询需完整行情+财务数据，两者职责不同，拆分避免过度加载 |

## 2. 后端分层结构

> 每层写清"允许做什么、禁止做什么"。

| 层 | 文件 | 允许做 | 禁止做 |
|----|------|--------|--------|
| **Route** | `agent_bp.py` | 参数校验、HTTP 状态码、traceId | 直接读写文件、调用 LLM |
| **Agent** | `agent.py` | 降级编排、结果组装 | 感知 HTTP 请求/响应 |
| **Provider** | `copaw_bridge.py` / `bailian_qa.py` | HTTP 调用外部 API | 操作本地存储 |
| **Storage** | `storage.py` | 文件 CRUD、数据校验 | 调用外部 API |

## 3. 前端架构

| 项 | 选型 | 理由 |
|----|------|------|
| UI 框架 | React（Hooks）≥ 18 | 教学通用性最高 |
| 构建工具 | Vite ≥ 5 | 极快 HMR，零配置启动 |
| 状态管理 | `useState` + `useEffect` | 8 个 state 变量足够 |
| HTTP 客户端 | `fetch` API | 浏览器原生，不引入 axios |
| 样式方案 | Tailwind CSS v3 + CSS Modules（局部隔离） | Tailwind 覆盖通用布局与间距；涉及研报阅读页、对比表等复杂组件时，CSS Modules 做局部样式隔离，防止全局污染 |

## 5. 数据存储架构

### 5.1 存储演进路线

| 阶段 | 存储 | 变更范围 | 说明 |
|------|------|----------|------|
| **P0（当前）** | JSON 文件 | — | 适合教学，零依赖直接运行 |
| **P1** | SQLite + sqlite-vec | `storage.py` 替换读写层；新增向量表；API 层零改动 | 支持多用户并发查询；sqlite-vec 扩展提供轻量向量检索，无需外部服务，保留教学可运行性 |
| **P2** | PostgreSQL 16 + pgvector | Storage 层 + 连接池配置；向量检索接口迁移至 pgvector；新增迁移脚本 | 生产级关系型数据库，ACID 完整，pgvector 支持 HNSW 索引，百万级向量检索 <50ms |

## 7. ADR（架构决策记录）

> 每条 ADR 记录一个"为什么选 A 而不选 B"的决策。至少写 2 条。

**示例 — ADR-001**：为什么选 Flask 而非 FastAPI？Flask 学习曲线低，无需 async/await，学员 Python 基础即可上手。

| ADR | 决策 | 核心理由 |
|-----|------|----------|
| ADR-001 | Flask 而非 FastAPI | 学习曲线低，教学更适合 |
| ADR-002 | 股票接口拆分为详情（`/stock/{code}`）与快照（`/stock/{code}/snapshot`）两个端点，而非单一通用接口 | US-006 研报侧边栏仅需价格+涨跌幅+PE+市値+关联研报数（快照），全量返回会浪费带宽、拖慢渲染；US-007 独立详情页需完整行情、财务指标及关联研报列表。单一接口若返回全量则研报联动性能差，若返回精简则详情页数据不足。拆分后各端点职责单一，前端按场景按需调用。 |
| ADR-003 | 文档分块（Chunk）存储于 JSON/SQLite，通过向量嵌入做语义检索，而非基于关键词的全文检索（如 Elasticsearch） | US-003/US-004 要求答案有原文引用并精确定位页码段落。全文检索对“核心竞争优势是什么”这类自然语言问题召回率低；向量检索将问题与文档 chunk 在语义空间对齐，召回更准，且 chunk 结构天然携带 page/chunk_id，直接支持 AC-004-2 原文跳转，无需额外索引。 |

## 8. 安全架构

> 对齐 `07` 4 安全需求。当前教学版与生产需求的差距。

| 层 | 当前措施 | 生产改进 |
|----|----------|----------|
| 认证 | 无（开放访问） | JWT |
| CORS | 全开放 | 白名单 |
| 密钥 | 环境变量 `.env` | Vault / Secrets Manager |
| 输入校验 | 基础长度校验 | 完整参数校验 + 防注入 |
| 文件上传 | 格式/大小校验（PDF/HTML，≤50MB） | 病毒扫描 + 内容安全检测 |
| 请求频率 | 无限制（单机教学版） | API 限流（Rate Limit）：问答接口 10 req/min/user，上传接口 5 req/min/user |
| 数据传输 | HTTP 明文（开发环境） | HTTPS（TLS 1.3）全链路加密，禁用 TLS 1.0/1.1 |
| 日志审计 | 无 | 敏感操作（上传/删除/问答）写入审计日志，保留 90 天 |
