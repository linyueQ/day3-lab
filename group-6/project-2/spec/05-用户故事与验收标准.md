> **本文只做一件事**：按用户能理解的子场景拆成多条 **User Story**，并为每条写清 **REQ-ID**、**验收标准（AC）**。

## 字段说明 — 每条 UserStory 必须包含以下字段

| 字段 | 填写说明 | 数据来源 ← | 流向下游 → |
|------|----------|------------|-----------|
| **需求编号** | 唯一编号，如 REQ-M2IRA-001 | ← `02` 采集编号 | → `14` 追踪矩阵首列 |
| **故事** | 作为【角色】，我希望【行为】，以便【价值】 | ← `04` PRD 场景拆解 | → `06` FSD 功能点 |
| **优先级** | P0（必须）/ P1（应该）/ P2（可选） | ← `03` §4 成功标准 | → `12` 里程碑排期 |
| **来源** | 对应 03/04 哪一条需求 | ← `03` 范围 · `04` 场景 | |
| **前置条件** | 触发该故事的前提状态 | ← `04` 用户流程 | → `13` 测试前置 |
| **功能对齐** | 对应功能规格 + 非功能约束 | | → `06` FSD · `07` NFR |
| **设计对齐** | 对应架构、接口、数据、安全 | | → `08` 架构 · `09` API · `10` Data · `11` 安全 |
| **验证对齐** | 对应测试用例 + 追踪矩阵行 | | → `13` TC · `14` 追踪矩阵行 |
| **验收标准** | AC，每条可独立验证 | ← 故事 + `07` NFR | → `13` 测试用例依据 |

## US-001（P0）研报上传

| 字段 | 内容 |
|------|------|
| **需求编号** | REQ-M2IRA-001 |
| **故事** | 作为投研分析师，我希望上传 PDF 格式的研报文件，以便系统能自动解析并存入知识库 |
| **优先级** | P0 |
| **来源** | ← 03 §3 研报管理 / 04 SC-01 |
| **前置条件** | 用户已进入研报管理页面 |
| **功能对齐** | → 06 §3 研报上传区 · 07 §5 文件约束 |
| **设计对齐** | → 08 §2 后端分层 · 09 POST /reports/upload · 10 Report 表 · 11 §4 文件上传校验 |
| **验证对齐** | → 13 TC-M02-001~003 · 14 第1行 |

| 验收标准 | 描述（每条可独立测试） |
|----------|----------------------|
| AC-001-01 | 上传 PDF 文件（≤ 50MB）成功，返回 report_id 和文件存储路径 |
| AC-001-02 | 上传非 PDF 文件被拒绝，返回明确错误信息 |
| AC-001-03 | 上传超过 50MB 的文件被拒绝，返回明确错误信息 |

## US-002（P0）研报自动解析

| 字段 | 内容 |
|------|------|
| **需求编号** | REQ-M2IRA-002 |
| **故事** | 作为投研分析师，我希望上传研报后系统自动提取标题、评级、目标价、核心观点、股票代码、股票名称，以便我无需手动阅读全文即可获取关键信息 |
| **优先级** | P0 |
| **来源** | ← 03 §4 成功标准 / 04 SC-01、R-02 |
| **前置条件** | 研报文件已上传成功 |
| **功能对齐** | → 06 §4 解析结果展示 · 07 §1 延迟SLO |
| **设计对齐** | → 08 §2 Parser层 · 09 POST /reports/parse · 10 ParsedReport 表 · 11 §5 LLM安全 |
| **验证对齐** | → 13 TC-M02-010~015 · 14 第2行 |

| 验收标准 | 描述（每条可独立测试） |
|----------|----------------------|
| AC-002-01 | 系统成功从 PDF 中提取文本内容 |
| AC-002-02 | LLM 解析成功提取标题字段，非空字符串 |
| AC-002-03 | LLM 解析成功提取评级字段（如"买入""增持""中性""减持""卖出"） |
| AC-002-04 | LLM 解析成功提取目标价字段（数值或"未提及"） |
| AC-002-05 | LLM 解析成功提取核心观点字段（文本摘要） |
| AC-002-06 | LLM 解析成功提取股票代码和股票名称 |
| AC-002-07 | 解析结果保存到内存数据库并持久化 |

| **优先级** | P0 |
