## 1. 性能需求

### 1.1 延迟 SLO

| 指标 | 目标值 | 度量方式 | 来源 |
|------|--------|----------|------|
| **研报上传端点** | < **3000ms**（不含 LLM 解析） | 文件接收到返回确认 | 文件 I/O |
| **LLM 解析端点 P95** | < **30000ms** | `parse_time_ms` 字段 | LLM 调用耗时 |
| **研报比对端点 P95** | < **15000ms** | `compare_time_ms` 字段 | LLM 比对分析耗时 |
| **知识库查询端点** | < **500ms** | 服务端计时 | 内存查询 |
| **行情数据端点** | < **3000ms** | 服务端计时 | AKShare API 调用 |

### 1.2 降级策略

```
LLM 调用降级链:
  ├─[1] 主 LLM (DashScope/OpenAI) → 正常解析/比对
  └─[2] 降级模式 → 返回"LLM 服务不可用"提示，不影响其他功能

AKShare 降级链:
  ├─[1] AKShare API 正常 → 返回最新行情数据
  ├─[2] 本地缓存 → 返回缓存数据 + 标注缓存时间
  └─[3] 降级 → 显示"暂无数据"
```

| 约束 | 说明 |
|------|------|
| LLM 不可用**不影响**研报管理和知识库浏览 | 仅解析/比对功能降级 |
| AKShare 不可用**不影响**核心功能 | 仅行情数据区域降级 |
| 降级**静默执行** | 不抛异常，通过前端提示用户 |

## 2. 可用性需求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 服务可用性 | **99%**（教学环境） | 单实例 Flask，不设 HA |
| MTTR | < 5 分钟 | 重启 Flask 进程即可恢复 |
| 数据持久化 | 内存数据库 + JSON 文件持久化 | 服务重启后从 JSON 恢复数据 |
| 故障恢复 | JSON 文件损坏时自动初始化空数据 | 研报 PDF 文件不受影响 |

## 3. 可观测性需求

### 3.1 链路追踪

| 项 | 规格 |
|----|------|
| traceId 格式 | `tr_{uuid.hex}`（32 位十六进制） |
| 注入位置 | 所有成功响应顶层 + 错误响应 `error.traceId` |
| 上游透传 | 优先复用请求头 `X-Trace-Id`，缺省时本地生成 |

### 3.2 错误码体系

| HTTP | `error.code` | 触发条件 |
|------|--------------|----------|
| 400 | `INVALID_FILE_TYPE` | 上传非 PDF 文件 |
| 400 | `FILE_TOO_LARGE` | 文件超过 50MB |
| 400 | `COMPARE_MIN_REPORTS` | 比对研报数 < 2 |
| 404 | `REPORT_NOT_FOUND` | 研报不存在 |
| 404 | `STOCK_NOT_FOUND` | 股票知识库不存在 |
| 500 | `PARSE_FAILED` | PDF 文本提取或 LLM 解析失败 |
| 500 | `LLM_ERROR` | LLM API 调用异常 |
| 500 | `AKSHARE_ERROR` | AKShare API 调用异常 |

## 4. 安全需求

| 项 | 当前实现 | 说明 |
|----|----------|------|
| API 认证 | 无（开放访问） | 教学版 |
| CORS | 全开放 | 生产需白名单 |
| 速率限制 | 无 | 教学场景不限速 |

### 4.1 敏感数据保护

| 数据类型 | 保护方式 |
|----------|----------|
| LLM API Key | 环境变量存储（`.env`），禁止硬编码 |
| 研报 PDF 文件 | 本地文件系统存储，不传输到外部 |
| 解析结果 | 内存数据库 + 本地 JSON 持久化 |

## 5. 数据约束

### 5.1 文件约束

| 约束 | 规格 | 来源 |
|------|------|------|
| 文件格式 | 仅 PDF（`application/pdf`） | `04` R-01 |
| 文件大小上限 | 50MB | `04` R-01 |
| 文件存储路径 | `{DATA_DIR}/reports/` | 本地存储 |

### 5.2 输入约束

| 约束 | 规格 | 来源 |
|------|------|------|
| 比对研报最小数量 | 2 份 | `04` R-04 |
| 比对研报 MUST 属于同一公司 | 股票代码一致 | `04` R-04 |

### 5.3 类型约束

| 字段 | 类型 | 约束 |
|------|------|------|
| `report_id` | `string` (UUID) | 唯一标识 |
| `stock_code` | `string` | 6 位数字（如"600519"） |
| `stock_name` | `string` | 股票简称 |
| `rating` | `string` | 枚举：买入 \| 增持 \| 中性 \| 减持 \| 卖出 \| 未提及 |
| `target_price` | `number\|null` | 正数或 null（未提及） |
| `key_points` | `string` | 核心观点文本 |
| `parse_status` | `string` | 枚举：pending \| parsing \| completed \| failed |
