## 1. 安全目标与合规要求

> 明确本模块需遵循的安全目标和外部合规标准。投研场景尤其关注数据隐私与内部信息防泄露。

| 安全目标 | 说明 | 对应合规 |
|----------|------|----------|
| 数据不出域 | 所有用户数据、研报内容、问答记录禁止传输至外部服务 | 内部信息安全规范 / `07` 4.1 |
| 最小权限原则 | 每个组件只拥有完成任务所需的最低权限 | 安全设计最佳实践 |
| 审计可追溯 | 所有用户操作、数据访问记录完整审计日志 | 内部合规要求 |
| 外部模型调用安全 | LLM API Key 环境变量存储，用户查询本地化处理 | `07` 4.1 / 外部模型调用约束 |

## 2. 认证与授权

> 定义用户身份验证方式和权限控制模型。写出认证流程和角色划分。

### 2.1 认证机制

| 项目 | 教学版实现 | 生产环境建议 |
|------|------------|--------------|
| 认证方式 | **无认证，开放访问** | JWT Token + OAuth2 |
| Token 有效期 | N/A | Access Token 30min, Refresh Token 7d |
| 密码策略 | N/A | 最少8位，含大小写+数字+特殊字符，90天强制更换 |
| 多因素认证 | N/A | 敏感操作（数据导出）需二次验证 |
| CORS 策略 | 全开放 | 白名单限制 |
| 速率限制 | 无限制 | 100 req/min/user |

> **来源**: `07` 4 安全需求 - 教学版开放访问，生产需加强认证

### 2.2 角色与权限矩阵

| 角色 | 研报上传 | 研报阅读 | 问答交互 | 对比分析 | 系统管理 | 收藏标注 | 数据导出 |
|------|:--------:|:--------:|:--------:|:--------:|:--------:|:--------:|:--------:|
| 分析师 | ✓ | ✓ | ✓ | ✓ | ✗ | ✓ | ✓ |
| 管理员 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| 访客（只读） | ✗ | ✓ | ✗ | ✗ | ✗ | ✗ | ✗ |

**权限说明**（基于 `09` API 端点）：
- **研报上传**: `POST /reports/upload` - 需要文件写入权限
- **研报阅读**: `GET /reports`, `GET /reports/{id}`, `GET /reports/{id}/file` - 基础查看权限
- **问答交互**: `POST /chat/sessions`, `POST /chat/sessions/{id}/messages` - 创建会话和发送消息
- **对比分析**: `POST /compare`, `GET /compare/{id}` - 创建和查看对比
- **系统管理**: 用户管理、配置管理、审计日志查看
- **收藏标注**: `POST /favorites`, `POST /annotations` - 收藏研报和添加标注
- **数据导出**: 对比结果导出、报表导出

## 3. 数据安全

> 定义数据分类分级、加密策略、存储安全规范。**投研场景核心关注点**。

### 3.1 数据分类与分级

| 数据类型 | 安全等级 | 示例 | 保护措施 |
|----------|----------|------|----------|
| 研报原文 | **机密** | PDF/HTML 研报文件 | 加密存储，访问审计 |
| 问答记录 | **内部** | 用户提问与AI回答 | 存储加密，定期清理 |
| 用户信息 | **内部** | 用户名、角色、操作日志 | 脱敏展示，最小采集 |
| 系统配置 | 一般 | 模型参数、界面配置 | 版本控制 |

### 3.2 加密策略

| 场景 | 教学版实现 | 生产环境建议 | 来源 |
|------|------------|--------------|------|
| 传输加密 | HTTP（开发环境） | TLS 1.2+ 强制 HTTPS | `09` 1.1 |
| 存储加密 | 本地明文存储 | AES-256 研报文件与问答记录落盘加密 | `07` 4.1 |
| 密钥管理 | 环境变量 `.env` | 环境变量 + Vault，密钥定期轮转 | `07` 4.1 |
| API Key 存储 | 环境变量，禁止硬编码 | KMS 加密存储，动态获取 | `07` 4.1 |
| 向量索引 | 本地存储 | 加密存储向量数据 | `07` 4.1 |
| 用户标注数据 | 本地存储 | 加密存储，访问控制 | `07` 4.1 |

## 4. 接口安全

> 与 `09` API 规格配合，定义 API 层面的安全防护措施。

| 防护措施 | 实现方式 | 说明 |
|----------|----------|------|
| 输入校验 | 服务端白名单校验 | 所有用户输入进行类型、长度、格式校验 |
| SQL 注入防护 | 参数化查询 / ORM | 禁止字符串拼接 SQL |
| XSS 防护 | 输出转义 + CSP | 研报内容渲染时防止脚本注入 |
| CSRF 防护 | **无（教学版无状态）** / SameSite Cookie | 教学版：无状态 API，无需 CSRF 防护<br>生产环境：状态变更接口需 CSRF Token |
| 速率限制 | **教学版无限制**（`07` 第4章）；生产建议 100 req/min/user | 防止接口滥用 |
| 文件上传校验 | 类型白名单 + 大小限制 | 仅允许 PDF/HTML，限制单文件大小 |

## 5. LLM / AI 安全

> 针对大模型集成的专项安全措施。投研场景数据敏感，需严格管控数据流向。

| 风险点 | 防护措施 | 验收标准 |
|--------|----------|----------|
| 数据外泄至第三方模型 | 本地部署模型 / 内网百炼服务，禁止调用外网 API | 网络出口审计零外传记录；`07` 4.1 要求用户查询仅本地存储 |
| Prompt 注入攻击 | 系统 Prompt 隔离，用户输入预处理过滤 | 输入长度限制 500 字符（`07` 5.1）；过滤危险字符和指令 |
| 模型幻觉 / 错误信息 | 回答标注来源引用，标识置信度 | 所有AI回答必须包含引用来源（`05` AC-004）；无来源时提示"基于综合分析" |
| 敏感信息泄露（模型回复） | 输出过滤层，正则匹配敏感信息并脱敏 | 手机号、身份证号、内部代码等敏感信息正则过滤；通过安全测试用例验证 |
| 越权访问其他用户会话 | 会话 ID 绑定用户身份校验 | 用户只能访问自己的会话（`09` 3.x 端点）；返回 403 禁止访问 |
| 批量请求滥用 | 速率限制 + 异常检测 | 单用户限制 100 req/min；连续异常请求触发告警 |

## 6. 审计与日志

> 定义安全审计日志的记录范围、保留策略。

| 事件类型 | 记录内容 | 保留期限 | 告警 |
|----------|----------|----------|------|
| 登录 / 登出 | 用户ID、时间、IP、结果 | 180天 | 连续失败 ≥5 次告警 |
| 研报访问 | 用户ID、文档ID、操作类型 | 365天 | 异常批量下载告警 |
| 数据导出 | 导出内容摘要、目标 | 365天 | 每次导出需审批 |
| 系统配置变更 | 变更人、变更前后值 | 永久 | 关键配置变更即时通知 |

## 7. 安全威胁清单（STRIDE 简表）

> 基于 STRIDE 模型，列出主要安全威胁与应对方案。

| 类型 | 威胁 | 风险等级 | 应对措施 | 负责人 |
|------|------|----------|----------|--------|
| S | 伪造用户身份访问研报 | **高** | 强认证 + Token 校验 | 安全负责人 |
| T | 研报内容在传输中被篡改 | 中 | TLS + 文件哈希校验 | 后端负责人 |
| I | 内部数据通过AI模型泄露 | **高** | 本地部署 + 输出过滤 | 安全负责人 |
| D | 大量请求导致服务不可用 | 中 | 速率限制（**生产建议**，教学版无限制）+ 降级策略（LLM 三级降级应对模型不可用，见 `07` 1.2） | 运维负责人 |
| E | 越权访问其他用户数据 | **高** | 数据隔离 + 权限校验 | 后端负责人 |
| R | 研报文件被未授权下载 | **高** | 文件访问鉴权 + 水印 | 安全负责人 |
| E |  elevation of privilege（权限提升） | 中 | 角色权限严格分离，禁止横向越权 | 后端负责人 |
| S | 会话劫持（Session Hijacking） | 中 | Token 绑定 IP + User-Agent，异常登录检测 | 安全负责人 |
| T | 提示词注入导致信息泄露 | **高** | 输入过滤 + 输出审计，禁止包含敏感指令 | 安全负责人 |
| D | 资源耗尽攻击（大文件上传） | 中 | 文件大小限制 50MB（`07` 5.2），批量限制 10 个 | 后端负责人 |

## 8. 部署安全

> 定义部署架构中的安全要求。

- [ ] 生产/开发环境**严格隔离**，数据不互通
- [ ] 数据库访问仅限应用服务器 IP 白名单
- [ ] 敏感配置（API Key、密码）通过环境变量注入，不入代码仓库（`07` 4.1）
- [ ] 依赖包定期安全扫描（`pip audit` / `npm audit`）
- [ ] 容器运行非 root 用户，最小权限原则
- [ ] 研报文件存储目录禁止执行权限（`07` 4.1）
- [ ] 定期漏洞扫描（依赖包 + 容器镜像）
- [ ] 备份数据加密存储，异地容灾
- [ ] 网络出口审计，禁止非授权外联（`07` 4.1）
- [ ] LLM 调用流量监控，异常数据外传检测
- [ ] 日志集中收集，保留 180 天以上（`07` 4.1 审计需求）
- [ ] 文件上传目录与执行目录分离，防止 webshell
- [ ] 向量索引文件访问控制，禁止直接下载
- [ ] API 网关层统一认证鉴权，防止绕过
