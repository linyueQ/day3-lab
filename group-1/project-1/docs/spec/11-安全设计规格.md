## 1. 安全目标

| 安全目标 | 说明 |
|----------|------|
| 防 zip-slip / 炸弹包 | 上传 ZIP 不得突破 `./data/bundles/<skill_id>/`，且必须受大小与文件数硬上限保护 |
| 防 XSS | `skill.md` 渲染后不得执行脚本、事件属性或 `javascript:` URL |
| 防 LLM 失控输出 | 草稿生成与智能搜索必须有超时、断路器、字段白名单和 fallback |
| 防滥用 | 上传、创建、LLM、互动接口必须限速 |
| 互动幂等 | 同一 `visitor_id` 的点赞 / 收藏只能计算一次；评分为覆盖 |
| 最小暴露 | 错误响应仅返回 `code / message / traceId / details`，不得向前端泄露堆栈 |

## 2. 身份与信任边界

### 2.1 身份模型

| 项目 | 规格 |
|------|------|
| 登录体系 | **无**（v0.2 匿名站点） |
| 访客标识 | cookie `hub_visitor`，后端生成随机 opaque ID |
| 用途 | 点赞 / 收藏 / 评分幂等、限速桶 key、收藏列表归属 |
| 敏感度 | 低；不承载认证授权，不存业务秘密 |

### 2.2 角色与权限矩阵

| 角色 | 浏览 | 创建 | 上传 ZIP | 点赞/收藏/评分 | 查看收藏 | 运维写盘 |
|------|------|------|----------|----------------|----------|----------|
| 匿名访客 | ✓ | ✓ | ✓ | ✓ | ✓（仅自己） | ✗ |
| 本地运维 / 开发者 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |

### 2.3 信任边界

```text
浏览器（不可信输入）
   ↓ HTTP / Cookie / Multipart
Flask Backend（校验、限速、消毒、审计）
   ├─ JSON / bundles 文件系统
   └─ LLM 上游（半可信，必须超时 + fallback）
```

## 3. 输入与接口安全

### 3.1 服务端输入校验

| 输入 | 规则 | 失败响应 |
|------|------|----------|
| `name` | 1–80 字符 | 400 `EMPTY_FIELD` |
| `category` | 白名单 8 类 | 400 `INVALID_CATEGORY` |
| `tags[]` | 0–6 项；单项 1–20 | 400 `INVALID_TAG` |
| `description` | 1–200 字符 | 400 `EMPTY_FIELD` |
| `skill_md` | 1–10000 字符 | 400 `EMPTY_FIELD` |
| `intent` | 10–200 字符 | 400 `INVALID_QUERY` |
| `query` / `q` | 1–200 字符 | 400 `INVALID_QUERY` |
| `score` | 1–5 整数 | 400 `INVALID_RATING` |
| `page_size` | 1–50 | 400 `INVALID_PAGE` |
| ZIP 文件 | `<= 10MB` | 413 `FILE_TOO_LARGE` |

### 3.2 限速

| 端点 | 限制 | 触发后 |
|------|------|--------|
| `POST /skills/upload` | 每 visitor 每分钟 `<= 3` 次 | 429 `RATE_LIMITED` |
| `POST /skills` | 每 visitor 每分钟 `<= 5` 次 | 429 |
| `POST /skills/draft` | 每 visitor 每 `5` 秒 `<= 1` 次 | 429 |
| `POST /skills/smart-search` | 每 visitor 每 `3` 秒 `<= 1` 次 | 429 |
| `POST/DELETE /skills/<id>/like|favorite` | 每 visitor 每秒 `<= 5` 次 | 429 |

### 3.3 错误响应要求

```json
{
  "error": {
    "code": "UNSAFE_ZIP",
    "message": "压缩包包含不安全路径",
    "details": { "entry": "../etc/passwd" },
    "traceId": "tr_xxx"
  }
}
```

要求：

- 不返回 Python 堆栈、绝对磁盘路径、上游 LLM 原始报错
- 所有异常路径都带 `traceId`
- `details` 只放调试所需最小字段

## 4. ZIP 上传安全

### 4.1 检查顺序

```text
1. 校验 Content-Length <= 10MB
2. 保存临时 ZIP 到 ./tmp
3. inspect：
   - 禁绝对路径
   - 禁 `..`
   - 禁符号链接 / 特殊文件
   - 统计总解压大小 <= 50MB
   - 文件数 <= 200
   - 必须存在根目录 skill.md
4. 全部通过后才允许解压
5. 解压失败或校验失败：删除临时 ZIP 与目标目录
```

### 4.2 关键防护

| 风险 | 防护 |
|------|------|
| zip-slip | `normpath` + 目标绝对路径前缀校验 |
| zip bomb | 原始大小上限 + 解压总大小上限 + 文件数上限 |
| 覆盖现有目录 | 每次新建独立 `bundles/<skill_id>/`，禁止覆盖其他 skill |
| 缺失 `skill.md` | inspect 阶段直接 400 `MISSING_SKILL_MD` |
| 残留脏文件 | 任何失败路径都执行目录清理 |

## 5. 内容渲染安全

### 5.1 Markdown 渲染链

```text
skill_md (Markdown)
   ↓
[后端] marked -> HTML
   ↓
[后端] bleach.clean(whitelist)
   ↓
存 skill_md_html
   ↓
[前端] DOMPurify
   ↓
dangerouslySetInnerHTML
```

### 5.2 白名单与禁止项

| 类别 | 内容 |
|------|------|
| 允许标签 | `p, h1-h6, ul, ol, li, code, pre, blockquote, strong, em, a, img, table, thead, tbody, tr, td, th` |
| 允许属性 | `href, src, alt, title` 及表格基础属性 |
| 禁止标签 | `script, iframe, object, embed, form, input, style` |
| 禁止属性 | 所有 `on*` 事件属性 |
| URL 约束 | 仅允许 `http(s)` 或相对路径；禁止 `javascript:` |

## 6. LLM 安全

### 6.1 风险面

| 风险 | 说明 | 防护 |
|------|------|------|
| Prompt 注入 | 用户意图试图改写系统指令 | 用户输入只作为变量插槽，系统 Prompt 固定 |
| 越权输出 | LLM 输出 HTML / 脚本 / 非预期 JSON | 输出结构校验 + 白名单字段抽取 |
| 上游超时 | 草稿 / 搜索卡住主链路 | 8 秒超时 + fallback |
| 连续故障 | 上游连续失败拖垮体验 | 断路器 60 秒直接降级 |
| 敏感信息泄露 | 将本地路径、密钥送给上游 | Prompt 中不得拼接 secrets / 文件系统路径 / trace 内容 |

### 6.2 草稿生成安全要求

| 项 | 要求 |
|----|------|
| 输入 | `intent` 仅文本，不接受富文本/HTML |
| 输出 | 必须经过 H1 / code fence 结构校验 |
| 出站消毒 | 返回前剥离 `<script>` 片段和危险 URL |
| 异常策略 | 失败统一返回 200 + `fallback=true` + 模板草稿 |

### 6.3 智能搜索安全要求

| 项 | 要求 |
|----|------|
| 输入 | `query` 1–200 字符 |
| LLM 输出 | 仅允许 `keywords[] / categories[] / match_reason` |
| 不可信结果 | LLM 生成的内容只影响召回和说明，不直接执行 |
| 失败策略 | 任一步骤失败都必须降级为关键词搜索，禁止 5xx |

## 7. Cookie、CORS 与 CSRF 取舍

| 项 | 当前设计 | 说明 |
|----|----------|------|
| Cookie 名 | `hub_visitor` | 仅用于匿名幂等 |
| Cookie 属性 | `Path=/`, `SameSite=Lax`, 长期有效 | 教学环境基线 |
| HttpOnly | 可选 | 若前端无需读 cookie，生产建议开启 |
| CORS | 教学环境可开放；生产改为域名白名单 | 详见 `07` |
| CSRF | v0.2 不引入登录态；因 cookie 仅承载访客标识，风险可接受 | 生产若引入账号体系必须加 CSRF Token |

## 8. 审计与日志

| 事件 | 字段 | 保留期 |
|------|------|--------|
| 请求日志 | method / path / status / duration_ms / ip / visitor_id / traceId | 30 天 |
| 上传审计 | skill_id / size / file_count / traceId | 90 天 |
| LLM 调用 | endpoint / upstream_status / upstream_latency_ms / fallback_used / traceId | 30 天 |
| 互动写入 | skill_id / visitor_id / action / new_value / traceId | 90 天 |
| 异常 | error.code / exc_type / traceId | 30 天 |

## 9. 威胁清单（STRIDE）

| 类型 | 威胁 | 严重度 | 应对 |
|------|------|--------|------|
| **S** | 伪造 visitor_id | 低 | 仅影响匿名互动；不承载授权 |
| **T** | ZIP 篡改文件系统 | **高** | 先 inspect 后解压；目录前缀校验 |
| **R** | 否认点赞/评分 | 低 | visitor_id + traceId + 审计日志 |
| **I** | `skill_md` 执行脚本 | **高** | 双层消毒 |
| **D** | LLM / 上传被刷爆 | 中 | 限速 + 断路器 + fallback |
| **E** | LLM 输出越权内容 | 中 | 输出 schema 校验 + 白名单抽取 |
