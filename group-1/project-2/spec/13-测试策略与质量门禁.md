## 一、测试分层

| 层级 | 工具 | 覆盖对象 | 说明 |
|------|------|----------|------|
| **L1 Unit** | pytest | 单个函数/类的逻辑（资产计算、单价计算） | 不依赖网络，最快 |
| **L2 Integration** | pytest + Flask test_client | API 端点 + 数据库 | 模拟 HTTP 请求 |
| **L3 Contract** | pytest + schema 断言 | 响应体结构与 `09` 一致 | 验字段名、类型 |
| **L4 E2E** | Playwright / Cypress | 完整用户旅程 | 模拟浏览器操作 |

## 二、用例拆分原则

> **核心理念：数"可独立失败的点"** — 每条 TC 只回答一个问题。

| 来源 | 计数方法 | 示例 |
|------|----------|------|
| 正常路径 | 每个端点的 happy path = 1 条 | `POST /gold-records → 201` = TC-001 |
| 错误码 | 每个 distinct error code = 1 条 | weight 为 0 → 40001 = TC-002 |
| 分支路径 | 每个 if/else 分支 = 1 条 | 汇总模式 vs 明细模式 = TC-003/004 |
| 类型约束 | Spec `09` 约定的字段类型 = 1 条 | `weight` 必须 DECIMAL = TC-005 |
