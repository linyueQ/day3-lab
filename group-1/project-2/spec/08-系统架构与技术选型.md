## 1. 架构总览

### 1.1 系统上下文

```
┌─────────────────────────────────┐
│    React 前端应用                 │
│    React + Vite                  │
│    (数据本地存储)                  │
└──────────────┬──────────────────┘
               │ HTTPS  /api/v1/*
               ▼
┌─────────────────────────────────┐
│     Flask Backend (WSGI)        │
│  ┌────────────────────────────┐ │
│  │  Routes — API 端点         │ │
│  │    ↓          ↓         ↓  │ │
│  │ GoldService MarketSvc OCR  │ │
│  └────┬──────────┬────────┬───┘ │
│       ▼          ▼        ▼     │
│  ┌────────┐ ┌────────┐ ┌─────┐ │
│  │ 本地JSON│ │金价 API │ │OCR  │ │
│  │ 文件    │ │第三方   │ │服务  │ │
│  └────────┘ └────────┘ └─────┘ │
└─────────────────────────────────┘
```

### 1.2 架构风格

| 决策 | 选型 | 理由 |
|------|------|------|
| **前后端分离** | React 前端 + Flask REST API | 现代前端框架，开发效率高，生态丰富 |
| **单体部署** | 后端单进程部署 | MVP 阶段不引入微服务复杂度 |
| **分层架构** | Route → Service → Storage | 业务逻辑与数据访问解耦 |
| **数据存储** | 前端 LocalStorage + 后端 JSON 文件 | 无需数据库，简化部署，数据本地化管理 |
| **外部数据代理** | 后端代理金价 API | 避免前端直连第三方，统一缓存策略 |

## 2. 后端分层结构

| 层 | 文件 | 允许做 | 禁止做 |
|----|------|--------|--------|
| **Route** | `gold_bp.py` / `ledger_bp.py` / `market_bp.py` | 参数校验、HTTP 状态码、响应封装 | 直接操作数据库、调用第三方 API |
| **Service** | `gold_service.py` / `market_service.py` | 业务逻辑编排、数据计算 | 感知 HTTP 请求/响应对象 |
| **Provider** | `market_service.py` / `ocr_service.py` | 调用第三方 API（金价/OCR） | 操作本地数据库 |
| **Storage** | `models/` + ORM | 数据库 CRUD、数据校验 | 调用外部 API |

## 3. 前端架构

| 项 | 选型 | 理由 |
|----|------|------|
| 前端框架 | React 18 | 组件化开发，生态成熟，性能优秀 |
| 构建工具 | Vite | 快速冷启动，热更新，现代开发体验 |
| UI 组件库 | Ant Design / Material-UI | 企业级组件库，开箱即用 |
| 状态管理 | React Hooks (useState/useContext) | 轻量级状态管理，无需引入 Redux |
| HTTP 客户端 | Axios | 功能完善，拦截器支持，浏览器兼容好 |
| 图表库 | ECharts / Recharts | 功能丰富，React 集成友好 |
| 本地存储 | LocalStorage / IndexedDB | 数据持久化存储在浏览器本地 |
| 图片处理 | HTML5 File API | 浏览器原生文件选择和处理 |

## 4. 外部服务集成

### 4.1 金价数据服务

| 项 | 规格 |
|----|------|
| 数据源 | 第三方金价 API（如上海黄金交易所数据） |
| 缓存策略 | Redis 缓存，TTL = 5 分钟 |
| 降级策略 | API 不可用时返回最近一次缓存数据 + `updated_at` 时间戳 |
| 调用频率 | 后端定时拉取（每 5 分钟），前端按需请求后端缓存 |

### 4.2 AI 识图（OCR）服务

| 项 | 规格 |
|----|------|
| 服务提供商 | 百度 OCR / 腾讯云 OCR |
| 调用方式 | 后端代理，小程序上传图片到后端，后端转发 OCR |
| 降级策略 | OCR 失败时返回 `weight: null`，前端提示手动输入 |
| 超时时间 | 5 秒 |

## 5. 数据存储架构

### 5.1 存储架构

| 层级 | 存储方式 | 用途 | 说明 |
|------|----------|------|------|
| **前端存储** | LocalStorage / IndexedDB | 用户数据、账本记录 | 浏览器本地持久化，无需后端数据库 |
| **后端存储** | JSON 文件 | 金价缓存、配置数据 | 文件系统存储，轻量级 |
| **文件存储** | 本地文件系统 | 临时图片文件 | 开发阶段使用，生产环境可迁移至 OSS |

### 5.2 数据存储演进路线

| 阶段 | 存储 | 变更范围 | 说明 |
|------|------|----------|------|
| **P0（当前）** | LocalStorage + JSON 文件 | — | 无数据库，纯本地存储 |
| P1 | LocalStorage + Redis | 新增后端缓存 | 金价数据缓存，提升响应速度 |
| P2 | IndexedDB + Redis + OSS | 升级前端存储 | 支持更大容量，图片迁移至 OSS |

## 6. 部署架构

| 服务 | 端口 | 启动命令 |
|------|------|----------|
| Flask Backend | **5000** | `python -m flask --app app run --port 5000` |
| React Dev Server | **5173** | `npm run dev` |
| Redis（可选） | **6379** | 系统服务，用于金价缓存 |
| Nginx（生产） | **80/443** | 反向代理静态资源和 API |

## 7. ADR（架构决策记录）

| ADR | 决策 | 核心理由 |
|-----|------|----------|
| ADR-001 | React 而非微信小程序 | 跨平台、开发生态更丰富、可扩展性强 |
| ADR-002 | 本地存储而非数据库 | 简化部署、无需数据库运维、适合个人/轻量使用场景 |
| ADR-003 | 后端代理金价 API 而非前端直调 | 统一缓存、避免暴露第三方 API Key、降级可控 |
| ADR-004 | Flask 而非 FastAPI | 团队 Python 基础好，Flask 生态成熟，学习曲线低 |

## 8. 安全架构

| 层 | 当前措施 | 生产改进 |
|----|----------|----------|
| 认证 | 无需登录（本地应用） | 可对接 JWT 或 OAuth |
| CORS | Flask-CORS 配置 | 限制特定域名访问 |
| 密钥 | 环境变量 `.env` | Vault / Secrets Manager |
| 数据传输 | HTTPS 强制 | TLS 1.2+ |
| 数据安全 | 本地存储加密（可选） | 敏感数据前端加密存储 |
