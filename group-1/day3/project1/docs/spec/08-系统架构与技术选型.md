### 1.2 架构风格

| 决策 | 选型 | 理由 |
|------|------|------|
| 前后端分离 | React + Flask | 教学清晰度 |
| 单体部署 | 各 1 进程 | 不引入微服务 |
| 分层 | Blueprint → Service → Storage | 替换存储零改动 |
| 无数据库 | JSON + 内存索引 | v0.2 数据量小 |
| LLM 独立 Service | `llm_service.py` 封装 | 便于 Mock / 降级 |
| 热度写时更新 | Service 层写互动时同步更新 `hot_score` | 避免后台任务 |

## 2. 后端分层

| 层 | 文件 | 允许 | 禁止 |
|----|------|------|------|
| **Route** | `blueprints/*.py` | 参数校验、状态码、traceId | 读写文件、调 LLM |
| **Service** | `services/*.py` | 业务规则、编排 | 感知 HTTP |
| **ZIP Service** | `services/zip_service.py` | 解压、zip-slip 校验、根目录 `skill.md` 校验、frontmatter 提取 | 感知 HTTP |
| **LLM Service** | `services/llm_service.py` | HTTP 调 LLM、超时、断路器、fallback | 操作文件 |
| **Ranking Service** | `services/ranking_service.py` | `hot_score` 计算、归一化 | 感知 HTTP |
| **Markdown** | `services/md_render.py` | marked + bleach 消毒 | 感知 HTTP |
| **Storage** | `storage.py` | 文件锁、原子写、CRUD | 调用外部 |

## 3. ZIP 发布流程

```
POST /skills/upload (multipart)
  1. 校验 Content-Length ≤ 10MB
  2. 保存临时文件 tmp/<uuid>.zip
  3. zip_service.inspect(zip):
       - 遍历条目：禁 `..` / 禁绝对路径 / 禁软链
       - 累计大小 ≤ 50MB / 文件数 ≤ 200
       - 查找 root 下的 skill.md
       - 解析 YAML frontmatter → (name/description/category/tags)
  4. 生成 skill_id = sk_ + ULID
  5. 安全解压到 ./data/bundles/<skill_id>/
  6. 读取 skill.md 内容 → 存 Skill 记录（含 skill_md_raw / skill_md_html）
  7. 返回 201 + skill_id
  失败路径：任何校验失败 → 删除临时文件 & 解压目录 → 返回对应 error.code
```

### 3.1 zip-slip 防护

```python
for member in zip.infolist():
```

## 4. LLM 编排

### 4.1 草稿生成

```
POST /skills/draft { intent }
  → llm_service.generate_draft(intent)
      ├─ 转义用户输入（防 prompt 注入）
      ├─ 组装系统 Prompt（角色 + 输出格式约束）
      ├─ 调 LLM，超时 8s
      ├─ 校验输出：必须含 H1 + 至少 1 个 code fence
      ├─ 通过 → 返回 skill_md_draft
      └─ 失败/超时 → 返回 TEMPLATE_DRAFT (fallback=true)
```

### 4.2 智能搜索

```
POST /skills/smart-search { query }
  → llm_service.smart_search(query)
      ├─ Step1: LLM 生成 keywords[] + categories[]（JSON 输出）
      ├─ Step2: 本地召回 = union(
      │          skills where category in categories,
      │          skills where name/desc/tags matches any keyword
      │       ) 按关键字命中数降序
      ├─ Step3: 若召回 <5 → 用 description 相似度补足到 5
      ├─ Step4: LLM 为 Top-N 每条生成 match_reason（可选，超时跳过）
      ├─ Step5: 返回 items[] (≥5)
      └─ 任一步骤失败 → 回退 keyword search（`GET /skills?q=query`）并补足到 5，`fallback=true`
```

### 4.3 断路器

| 状态 | 条件 | 行为 |
|------|------|------|
| Closed | 失败率 < 50% | 正常调用 |
| Open | 连续 5 次失败或超时 | 60 秒内直接 fallback |
| Half-Open | 熔断 60 秒后 | 放 1 个请求探活 |

## 5. 前端架构

| 项 | 选型 | 理由 |
|----|------|------|
| UI | React ≥ 18 | 通用 |
| 构建 | Vite ≥ 5 | 零配置 |
| 路由 | React Router 6 | 4 个页面 |
| 状态 | `useState` + `useReducer` + URL query + Context（Theme、VisitorId） | 轻量 |
| Markdown | `marked` + `DOMPurify` | 双层消毒 |
| 代码高亮 | `highlight.js` | 生态成熟 |
| 上传 | `fetch` + `FormData` + `XHR onprogress`（纯 fetch 无进度，需 XHR） | 显示进度 |
| 主题 | CSS 变量 + `data-theme` + `localStorage` | 无刷新切换 |

## 6. 热度排名 Service

```python
def compute_hot_score(skill, stats):
    d = norm(skill.download_count, stats.max_download)
    r = norm(skill.rating_avg * min(skill.rating_count, 50) / 50, 5.0)
    i = norm(skill.like_count + skill.favorite_count, stats.max_interaction)
    return 0.5 * d + 0.3 * r + 0.2 * i
```

- 写时：每次互动变更 → 重算该 skill 的 `hot_score`；批量写时仅重算受影响 skill
- `stats.max_*` 由 `compute_global_stats()` 缓存 60 秒

## 8. ADR

| ADR | 决策 | 理由 |
|-----|------|------|
| ADR-001 | Flask 而非 FastAPI | 学习曲线 |
| ADR-002 | JSON 而非 SQLite | 数据量小 + 教学友好 |
| ADR-003 | 热度写时更新 | 避免后台任务复杂度 |
| ADR-004 | LLM 独立 Service + 断路器 + fallback | 保证主链路可用 |
| ADR-005 | 解压前先 inspect 再落盘 | zip-slip / 炸弹包防护 |
| ADR-006 | 主题使用 CSS 变量 + `data-theme` | 无刷新切换 + 易维护 |
| ADR-007 | visitor_id 基于 cookie | v0.2 无用户体系，保证互动幂等 |
| ADR-008 | 智能搜索两阶段 + keyword 兜底 | 保证 ≥5 条硬要求 |
