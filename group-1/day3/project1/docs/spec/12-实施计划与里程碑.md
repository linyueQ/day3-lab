## 5. 依赖与关键路径

```text
S1 基础骨架
  ↓
S2 创建 / 上传 / 详情
  ↓
S3 LLM / 标签 / 分类 / 排序 / 智能搜索
  ↓
S4 点赞 / 收藏 / 评分 / 下载 / 热度闭环
  ↓
S5 测试收口与发布
```

关键依赖：

- `W2 Storage` 先于 `W6 Ranking / Interaction`
- `W3 ZIP Service` 先于 `REQ-M2HUB-001` 全链路验收
- `W5 LLM Service` 先于 `REQ-M2HUB-002a/007`
- `W4 Markdown` 先于 `REQ-M2HUB-003` 与安全门禁

## 6. 主要风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| ZIP 安全实现不完整 | 可能写穿文件系统 | 先 inspect 后解压；S2 即补齐安全测试 |
| LLM 上游不稳定 | 草稿/智能搜索不可用 | fallback + 断路器，S3 先做 Mock 契约 |
| 排序与热度公式实现偏差 | 列表顺序不可信 | W6 单元测试先行，独立校验公式 |
| 互动聚合与幂等冲突 | 计数错误 | interactions 明细与 skills 聚合同轮写入 |
| 文档 / 接口 / 测试编号漂移 | 交付验收混乱 | S5 统一复查 `05/09/13/14` 映射 |
