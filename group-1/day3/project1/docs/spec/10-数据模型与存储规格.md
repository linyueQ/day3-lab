## 1. 存储边界

| 文件 / 目录 | 作用 | 写入来源 |
|-------------|------|----------|
| `./data/skills.json` | Skill 主记录、聚合计数、排序字段 | 在线创建 / ZIP 上传 / 互动写入 |
| `./data/interactions.json` | 点赞 / 收藏 / 评分明细 | like / favorite / rate |
| `./data/bundles/<skill_id>/` | ZIP 解压后的技能包目录；至少包含 `skill.md` | ZIP 上传 / 在线创建 |
| `./data/skills.json.bak` / `interactions.json.bak` | 原子写前的最近一次备份 | Storage 层 |
| `./tmp/<uuid>.zip` | 上传 ZIP 的临时文件 | Upload Route / ZIP Service |

## 2. 概念模型

```text
Category（白名单）
   1 ──────────────── * Skill
                         │
                         │ 1
                         ├────────────── 0..* LikeRecord
                         │
                         ├────────────── 0..* FavoriteRecord
                         │
                         └────────────── 0..* RatingRecord

Visitor（无用户表，仅 cookie hub_visitor 标识）
   1 ──────────────── * Like / Favorite / Rating
```

> `Tag` 不建独立表，内聚在 `Skill.tags[]`；`Category` 只做白名单枚举，不做运行时写表。

## 3. 实体定义 — Skill

### 3.1 Skill 字段

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `skill_id` | `string` | 是 | **PK**，`sk_` + ULID | 技能唯一 ID |
| `name` | `string` | 是 | 1–80 字符 | 展示名称 |
| `description` | `string` | 是 | 1–200 字符 | 摘要 |
| `category` | `string` | 是 | 白名单枚举 | 见 3.2 |
| `tags` | `array<string>` | 否 | 0–6 项；去重；单项 1–20 字符 | 标签 |
| `skill_md` | `string` | 是 | 1–10000 字符 | 原始 Markdown |
| `skill_md_html` | `string` | 是 | 已消毒 HTML | 详情页直出字段 |
| `has_bundle` | `boolean` | 是 | 默认 `false` | 是否存在 ZIP 下载包 |
| `bundle_path` | `string \| null` | 否 | `./data/bundles/<skill_id>/` | 相对存储路径 |
| `bundle_size` | `integer \| null` | 否 | `>= 0` | ZIP 原始字节数 |
| `file_count` | `integer \| null` | 否 | `>= 0` | bundle 内文件数 |
| `rating_avg` | `float` | 是 | `0.00–5.00`，保留 2 位小数 | 聚合评分 |
| `rating_count` | `integer` | 是 | `>= 0` | 已评分 visitor 数 |
| `view_count` | `integer` | 是 | `>= 0` | 详情浏览计数 |
| `download_count` | `integer` | 是 | `>= 0` | ZIP 下载计数 |
| `like_count` | `integer` | 是 | `>= 0` | 当前点赞数 |
| `favorite_count` | `integer` | 是 | `>= 0` | 当前收藏数 |
| `hot_score` | `float` | 是 | `>= 0` | 热度排序字段 |
| `created_at` | `string` | 是 | ISO-8601 UTC | 创建时间 |
| `updated_at` | `string` | 是 | ISO-8601 UTC | 最近更新时间 |

### 3.2 Category 白名单

| key | label |
|-----|-------|
| `frontend` | 前端 |
| `backend` | 后端 |
| `security` | 安全 |
| `bigdata` | 大数据 |
| `coding` | 写代码 |
| `ppt` | PPT |
| `writing` | 写作 |
| `other` | 其它 |

### 3.3 Skill 初始值

| 写入来源 | 字段初始化 |
|----------|------------|
| 在线创建 `POST /skills` | `has_bundle=false`，`bundle_path=./data/bundles/<skill_id>/`，`bundle_size=null`，`file_count=1`（仅 `skill.md`），所有计数 `= 0` |
| ZIP 上传 `POST /skills/upload` | `has_bundle=true`，`bundle_path/bundle_size/file_count` 取 ZIP 结果，所有计数 `= 0` |

## 4. 实体定义 — Interaction

> `interactions.json` 存明细，`skills.json` 存聚合结果；写入时两者一起更新，保持同轮原子写。

### 4.1 LikeRecord

| 字段 | 类型 | 必填 | 约束 |
|------|------|------|------|
| `skill_id` | `string` | 是 | FK → `Skill.skill_id` |
| `visitor_id` | `string` | 是 | 1–64 字符，cookie 值 |
| `created_at` | `string` | 是 | ISO-8601 UTC |

唯一约束：`(skill_id, visitor_id)`。

### 4.2 FavoriteRecord

| 字段 | 类型 | 必填 | 约束 |
|------|------|------|------|
| `skill_id` | `string` | 是 | FK |
| `visitor_id` | `string` | 是 | 1–64 字符 |
| `created_at` | `string` | 是 | ISO-8601 UTC |

唯一约束：`(skill_id, visitor_id)`；`GET /me/favorites` 以 `created_at DESC` 返回。

### 4.3 RatingRecord

| 字段 | 类型 | 必填 | 约束 |
|------|------|------|------|
| `skill_id` | `string` | 是 | FK |
| `visitor_id` | `string` | 是 | 1–64 字符 |
| `score` | `integer` | 是 | 1–5 |
| `updated_at` | `string` | 是 | ISO-8601 UTC |

唯一约束：`(skill_id, visitor_id)`；重复评分为**覆盖**，不是新增。

### 4.4 物理存储建议形态

```json
{
  "likes": [
    { "skill_id": "sk_01", "visitor_id": "v_a1", "created_at": "2026-04-15T08:00:00Z" }
  ],
  "favorites": [
    { "skill_id": "sk_01", "visitor_id": "v_a1", "created_at": "2026-04-15T08:10:00Z" }
  ],
  "ratings": [
    { "skill_id": "sk_01", "visitor_id": "v_a1", "score": 5, "updated_at": "2026-04-15T08:20:00Z" }
  ]
}
```

## 5. Storage 方法清单

### 5.1 Skill 读写

| 方法 | 签名 | 行为 | 关联 API |
|------|------|------|---------|
| `list_skills` | `(filters, sort, page, page_size) -> (items, total)` | 列表、筛选、排序、分页 | `GET /skills` |
| `get_skill` | `(skill_id) -> dict | None` | 详情读取 | `GET /skills/<id>` |
| `create_skill` | `(payload) -> dict` | 新建 Skill 记录 | `POST /skills` / `POST /skills/upload` |
| `update_skill` | `(skill_id, patch) -> dict` | 局部更新 | 互动 / 渲染补写 |
| `increment_view` | `(skill_id) -> int` | `view_count +1` | `GET /skills/<id>` |
| `increment_download` | `(skill_id) -> int` | `download_count +1` | `GET /skills/<id>/download` |
| `list_tags` | `(limit=20) -> list` | 聚合标签频次 | `GET /skills/tags` |
| `list_categories` | `() -> list` | 返回白名单 + 动态 count | `GET /categories` |

### 5.2 Interaction 读写

| 方法 | 签名 | 行为 | 关联 API |
|------|------|------|---------|
| `set_like` | `(skill_id, visitor_id, liked) -> dict` | 幂等写点赞状态 | `POST/DELETE /skills/<id>/like` |
| `set_favorite` | `(skill_id, visitor_id, favorited) -> dict` | 幂等写收藏状态 | `POST/DELETE /skills/<id>/favorite` |
| `set_rating` | `(skill_id, visitor_id, score) -> dict` | 覆盖写评分 | `POST /skills/<id>/rate` |
| `list_favorites_by_visitor` | `(visitor_id) -> list` | 返回当前 visitor 收藏列表 | `GET /me/favorites` |
| `recompute_skill_metrics` | `(skill_id) -> dict` | 基于 interactions 重算 like/favorite/rating/hot | 互动写后 |

### 5.3 公共方法

| 方法 | 签名 | 行为 |
|------|------|------|
| `_read_json` | `(path) -> dict | list` | 加锁读取；主文件损坏时回退 `.bak` |
| `_write_json` | `(path, data) -> None` | 临时文件写入后 `os.replace` 原子替换 |
| `_with_lock` | `(path, fn) -> Any` | 文件级排它锁包装 |

## 6. 派生与聚合规则

| 场景 | 写入规则 | 结果 |
|------|----------|------|
| 在线创建成功 | 保存 `skill_md`；同步生成 `skill_md_html` | 详情可直接渲染 |
| ZIP 上传成功 | frontmatter 优先提取 `name / description / category / tags`；正文写入 `skill_md` | `has_bundle=true` |
| 详情访问 | `view_count +1` 后返回最新值 | 影响 hot 排序 |
| 下载 ZIP | `download_count +1` 后返回二进制流 | 影响 hot 排序 |
| 点赞/取消点赞 | 以 `(skill_id, visitor_id)` 是否存在为准幂等写入 | `like_count` 可增可减，但不得 < 0 |
| 收藏/取消收藏 | 同上 | `favorite_count` 可增可减，但不得 < 0 |
| 评分 | 同 visitor 覆盖旧分；`rating_count` 仅首次评分 +1 | `rating_avg` 重新求平均 |
| 任意聚合变更 | 立即调用 `recompute_skill_metrics` | `hot_score` 写时更新 |

### 6.1 热度计算

| sort | 表达式 |
|------|--------|
| `hot` | `0.5 × norm(download_count) + 0.3 × norm(rating_avg × min(rating_count,50)/50) + 0.2 × norm(like_count + favorite_count)` |
| `downloads` | `download_count DESC` |
| `rating` | `rating_avg × min(rating_count,50)/50 DESC` |
| `latest` | `updated_at DESC` |

> `norm(x) = (x - min) / (max - min + 1e-9)`；当分母接近 0 时结果视为 0，禁止出现 `NaN`。

## 7. 索引与检索策略

| 类型 | 实现 |
|------|------|
| 主键索引 | 进程内 `dict[skill_id] -> skill` |
| 分类筛选 | 线性扫描 `skills.json` |
| 标签过滤 | `set(tags)` AND 包含 |
| 收藏列表 | `interactions.favorites` 按 `visitor_id` 过滤 + 时间倒序 |
| 智能搜索召回 | `name / description / tags` 关键词匹配 + category 过滤 |

> P0 目标数据量 `< 500` skills、`< 50,000` interaction records，线性扫描可接受；P1 再迁移 SQLite / FTS5。

## 8. 种子数据与恢复要求

| 项 | 要求 |
|----|------|
| Skill 数量 | `>= 10` 条 |
| 分类覆盖 | 8 个白名单分类至少各 1 条 |
| ZIP 数据 | `>= 3` 条 `has_bundle=true` |
| 互动数据 | 至少覆盖点赞、收藏、评分 3 类记录 |
| smart-search 兜底 | 数据总量不足 5 条时返回全部现有记录 |
| 故障恢复 | 启动时若主 JSON 损坏且 `.bak` 可读，则自动回退 `.bak` 并记录告警 |
