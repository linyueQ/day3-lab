## 1. 背景与问题

| 痛点 | 来源 |
|------|------|
| 技能包含多文件（`skill.md` + 资源脚本 + 截图），单字段表单无法承载 | S-01 会议 |
| 新手写不出规范的 `skill.md`，发布门槛高 | S-03 访谈 |
| 关键词搜索理解力有限，用户用自然语言描述需求时命中率低 | S-03 访谈 |
| 缺少互动信号，用户无法快速识别"好 skill" | S-01 会议 |
| 长时间浏览缺少深色模式，夜间使用体验差 | S-03 访谈 |

## 2. 提案概要

在 Web 端交付一个聚焦"**Skill 管理 / 展示 / 检索 / 互动**"的技能库：
支持 **ZIP 压缩包发布** 与 **在线创建（LLM 协助生成 skill.md）**；详情页渲染 `skill.md` 并展示 **描述/标签/评分/浏览/收藏/下载**；提供 **深色/浅色主题切换**；检索支持 **分类 + 标签 + 多维度排序 + LLM 语义搜索（至少 5 条匹配）**；互动支持 **点赞 / 收藏 / 热度排名**。首版 v0.2 只做这 10 条能力，其余延后。

## 3. 范围边界

| 做（范围内） | 不做（范围外） |
|-------------|---------------|
| ZIP 压缩包上传发布 | 版本管理 / 增量更新 |
| 在线表单创建 skill + LLM 生成 `skill.md` 草稿 | 真机运行 skill / 沙箱测试 |
| 详情页展示 `skill.md`（Markdown 渲染）+ 元数据卡片 | 评论系统 |
| 深色 / 浅色主题切换（记忆偏好） | 多语言 i18n |
| 标签筛选 + 分类筛选 + 按下载/评分/热度排序 | 个性化推荐 |
| LLM 自然语言智能搜索（返回 ≥ 5 条） | 向量数据库生产化 |
| 点赞 / 收藏 / 评分 / 热度计算 | 用户账号系统（v0.2 匿名，`visitor_id` 来自 cookie） |

## 4. 成功标准（可度量）

| 标准 | 度量方式 |
|------|----------|
| ZIP 发布可用 | 用户上传 ≤10MB zip，**30 秒内** 完成解析入库并跳转详情页 |
| LLM 创建可用 | 输入一句话（≥10 字），**10 秒内** 返回一份可编辑的 `skill.md` 草稿 |
| 智能搜索命中 | 对 10 条自然语言测试 query，每条均返回 **≥5 条** 结果，Top-3 人工相关性 ≥ 80% |
| 主题切换可记忆 | 切换主题后刷新页面状态保持，`localStorage` 持久化 |
| 热度排序可演示 | 种子数据下，点赞 / 下载 / 评分发生变化后，列表 `sort=hot` 排序 ≤1 秒内重算生效 |
| 点赞/收藏幂等 | 同一 `visitor_id` 重复点赞/收藏，总计数 **仅 +1** |

## 5. 风险（简表）

| 风险 | 影响 | 应对 |
|------|------|------|
| ZIP 解压安全（zip-slip、炸弹包） | 写穿系统目录 / 磁盘耗尽 | 白名单目录 + 文件名规范化 + 解压前后大小上限 |
| LLM 调用不可用 / 超时 | 创建与智能搜索功能不可用 | 创建：返回模板草稿兜底；搜索：降级到关键词匹配 |
| LLM Prompt 注入 | 输出被污染 | 输入转义 + 系统 Prompt 隔离 + 输出字段白名单校验 |
| JSON 并发写冲突（点赞/收藏高频写） | 计数丢失 | 文件锁 + 原子写；互动数据按 skill_id 分片 |
| Markdown XSS | 详情页脚本执行 | 双层消毒（后端 bleach + 前端 DOMPurify） |

## 6. 立项前置条件（Go / No-Go）

- ☐ LLM 服务（百炼 / OpenAI 兼容）密钥已配置或提供 Mock
- ☐ 后端与前端可独立启动（5000 / 5173）
- ☐ 种子技能包 ≥ 10 个 zip 已准备
- ☐ `data/bundles/` 目录可写
