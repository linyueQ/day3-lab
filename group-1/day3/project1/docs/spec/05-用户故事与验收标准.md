## US-001（P0）ZIP 压缩包发布

| 字段 | 内容 |
|------|------|
| **需求编号** | REQ-M2HUB-001 |
| **故事** | 作为**技能作者**，我希望**上传整个 skill 文件夹的 zip 压缩包**，以便**一次发布包含 `skill.md` + 资源脚本 + 截图的完整技能包**。 |
| **优先级** | P0 |
| **来源** | ← 03 §3 做 / 04 SC-01 / R-01~03 |
| **前置条件** | 本地已有合法 zip 文件 |
| **功能对齐** | → 06 §3 上传页 · 07 §4 上传约束 |
| **设计对齐** | → 08 §3 ZIP Service · 09 `POST /skills/upload` · 10 §3 Skill · 11 §5 zip-slip |
| **验证对齐** | → 13 TC-M02-001~006 · 14 REQ-001 |

| 验收标准 | 描述 |
|----------|------|
| AC-001-01 | 上传 ≤10MB 的 zip，后端识别到根目录 `skill.md` 后返回 201 + skill_id，30 秒内跳转详情页 |
| AC-001-02 | zip 中无 `skill.md` 或 `skill.md` 位于非根目录 → 400 `MISSING_SKILL_MD` |
| AC-001-03 | zip > 10MB → 413 `FILE_TOO_LARGE` |
| AC-001-04 | zip 解压后总大小 > 50MB 或文件数 > 200 → 400 `BUNDLE_LIMIT_EXCEEDED` |
| AC-001-05 | zip 包含 `..` 或绝对路径条目 → 400 `UNSAFE_ZIP`，**禁止任何文件落盘** |
| AC-001-06 | 成功发布后，详情页必须展示从 `skill.md` frontmatter 自动提取的 `name / description / category / tags`（若缺失则用缺省值） |

## US-002a（P0）LLM 协助生成 skill.md 草稿

| 字段 | 内容 |
|------|------|
| **需求编号** | REQ-M2HUB-002a |
| **故事** | 作为**不熟悉 skill.md 规范的作者**，我希望**输入一句话描述意图，系统调用大模型生成 skill.md 草稿**，以便**在草稿基础上快速完成创建**。 |
| **优先级** | P0 |
| **来源** | ← 04 SC-02 / R-05 |
| **前置条件** | 已进入创建页 |
| **功能对齐** | → 06 §4.2 LLM 辅助 |
| **设计对齐** | → 08 §4 LLM 编排 · 09 `POST /skills/draft` · 11 §6 LLM 安全 |
| **验证对齐** | → 13 TC-M02-020~024 · 14 REQ-002a |

| 验收标准 | 描述 |
|----------|------|
| AC-002a-01 | 用户输入 ≥10 字符的自然语言意图 → 10 秒内返回 `skill_md_draft`（长度 ≥100 字符） |
| AC-002a-02 | 返回的草稿必须包含一级标题、使用说明、至少 1 个代码块 |
| AC-002a-03 | LLM 失败/超时 → 返回 200 + `fallback=true` + 本地模板草稿（不得返回 5xx） |
| AC-002a-04 | 同一 `visitor_id` 连续调用间隔 < 5 秒 → 429 `RATE_LIMITED` |
| AC-002a-05 | 草稿内容经过出站消毒，不得包含 `<script>` 片段 |

| **优先级** | P0 |

| 验收标准 | 描述 |

## US-007（P0）LLM 智能搜索

| 字段 | 内容 |
|------|------|
| **需求编号** | REQ-M2HUB-007 |
| **故事** | 作为**不知道准确关键词的用户**，我希望**用自然语言描述需求，系统用大模型匹配至少 5 条合适的 skill**，以便**即使模糊描述也能找到可用技能**。 |
| **优先级** | P0 |
| **来源** | ← 04 SC-06 / R-06 / R-07 |
| **前置条件** | 列表页切换至"智能搜索"模式 |
| **功能对齐** | → 06 §4 智能搜索 |
| **设计对齐** | → 08 §4 LLM 编排 · 09 `POST /skills/smart-search` · 11 §6 |
| **验证对齐** | → 13 TC-M02-070~074 · 14 REQ-007 |

| 验收标准 | 描述 |
|----------|------|
| AC-007-01 | 输入 `query` 1–200 字符，响应 `items.length >= 5`（不足时用关键词兜底补齐到 5） |
| AC-007-02 | 响应包含 `items[].match_reason`，由 LLM 给出 1 句相关性说明 |
| AC-007-03 | LLM 超时（>8s）或不可用 → 响应 `fallback=true`，仍返回 ≥5 条（关键词兜底） |
| AC-007-04 | `query` 超长（>200）→ 400 `INVALID_QUERY` |
| AC-007-05 | 总响应时间 ≤10 秒 |

| 验收标准 | 描述 |
