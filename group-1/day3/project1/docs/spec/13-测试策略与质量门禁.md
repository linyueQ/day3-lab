## 1. 测试分层

| 层级 | 工具 | 覆盖对象 | 说明 |
|------|------|----------|------|
| **L1 Unit** | pytest | ZIP Service / LLM Service / Ranking / Markdown / Storage | 无网络，最快 |
| **L2 Integration** | pytest + Flask `test_client` | 全部 API 端点 | 模拟 HTTP + JSON / multipart |
| **L3 Contract** | pytest + schema | `09` 中 DTO、错误码、字段名 | 防止契约漂移 |
| **L4 Security** | pytest | zip-slip / XSS / 限速 / fallback | 高风险单独门禁 |
| **L5 E2E** | Playwright | 关键用户旅程 | 浏览器级联调 |

## 2. 用例拆分原则

| 来源 | 计数方法 | 示例 |
|------|----------|------|
| Happy path | 每个核心能力至少 1 条 | ZIP 成功上传 |
| 错误码 | 每个 distinct `error.code` 至少 1 条 | `MISSING_SKILL_MD` |
| 幂等 / 覆盖 | 每个状态转换 1 条 | 点赞二次 POST 不重复计数 |
| 公式 / 聚合 | 每个关键派生规则 1 条 | `hot_score` 不得出现 `NaN` |
| 降级路径 | 每个 LLM fallback 1 条 | smart-search 上游失败转 200 |

## 4. 质量门禁

| Gate-ID | 条件 | 阻塞合并 |
|---------|------|----------|
| **G-LINT-BE** | `black --check` + `flake8` 通过 | 是 |
| **G-LINT-FE** | `eslint` 通过 | 是 |
| **G-UNIT** | L1 Unit 全绿 | 是 |
| **G-INT** | L2 Integration 全绿 | 是 |
| **G-CONTRACT** | L3 Contract 全绿 | 是 |
| **G-SEC** | ZIP / XSS / 限速 / fallback 用例全绿 | 是 |
| **G-COV** | 后端覆盖率 `>= 80%` | 是 |
| **G-E2E** | P0 E2E 全绿 | 是 |
| **G-PERF** | `GET /skills` P95 < 300ms | 否（warning） |

## 5. TDD 建议

| 场景 | 建议 |
|------|------|
| ZIP inspect / 解压 | **必须先写失败用例**，再补实现 |
| LLM fallback / 断路器 | 先写 Mock 上游失败用例 |
| hot_score 公式 | 先写纯函数单测，再接入列表排序 |
| 点赞 / 收藏 / 评分幂等 | 先写状态转换表测试 |
| 主题切换 | 以前端 E2E 为主，单测补 reducer / hook |
