# Prompt Engineering 高级技巧

## 概述

系统化的 Prompt 设计方法论。掌握思维链（CoT）、Few-shot 学习、Self-consistency、ReAct 框架等高级技巧，让 AI 输出更精准、更可控。

## 核心框架

### 结构化 System Prompt

```markdown
# 角色
你是一位资深的代码审查专家，专注于 Python 和 TypeScript。

# 能力
- 识别代码中的安全漏洞和性能瓶颈
- 提供符合 SOLID 原则的重构建议
- 评估代码的可测试性和可维护性

# 约束
- 每个问题提供不超过 3 个改进建议
- 按严重程度排序：🔴 严重 → 🟡 警告 → 🟢 建议
- 给出修复前后的代码对比

# 输出格式
## 审查摘要
[一句话总结]

## 问题列表
1. [严重程度] 问题描述
   - 当前代码: `...`
   - 建议修改: `...`
   - 原因: ...
```

### Chain-of-Thought（思维链）

```
问题: 一个商店有 15 个苹果，卖出了 7 个，又进了 12 个，
      然后给员工分了 3 个。现在有多少苹果？

让我们一步一步思考：
1. 初始: 15 个苹果
2. 卖出 7 个: 15 - 7 = 8
3. 进货 12 个: 8 + 12 = 20
4. 分给员工 3 个: 20 - 3 = 17

答案: 17 个苹果
```

### ReAct 框架

```
Thought: 我需要查找最新的 Python 版本号
Action: search("latest Python release version 2024")
Observation: Python 3.12.3 was released on April 9, 2024
Thought: 我找到了答案
Answer: 最新的 Python 稳定版是 3.12.3
```

## 高级技巧

| 技巧 | 适用场景 | 效果 |
|------|----------|------|
| Few-shot | 格式化输出 | 准确率 +30% |
| CoT | 数学/逻辑推理 | 准确率 +40% |
| Self-consistency | 高可靠性需求 | 减少幻觉 |
| Tree of Thought | 复杂决策 | 更优解 |

## 常见陷阱

1. **模糊指令**: "写得好一点" → 应具体说明"好"的标准
2. **上下文过载**: 一次给太多信息 → 分步骤提供
3. **忽略输出格式**: 导致解析困难 → 明确 JSON/Markdown 格式
4. **缺乏示例**: 新任务不给例子 → 至少提供 2-3 个示例
