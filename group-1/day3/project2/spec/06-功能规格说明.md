## 1. 攒金主页（账本列表页）

> 对齐 `05` US-002（资产总览与记录列表）、US-004（多账本管理）。

### 1.1 页面布局

```
┌──────────────────────────────────────┐
│  [默认账本 ▾]         [🔍 搜索] [···] │
├──────────────────────────────────────┤
│  ┌────────────────────────────────┐  │
│  │     总重量(克) 👁 ⚙           │  │
│  │        0.00                    │  │
│  │  购入总价(元)  预估价值(元)  预估收益(元)│
│  │    0.00        0.00       0.00 │  │
│  │ [攒金日历] [添加黄金] [赠卖记录]│  │
│  └────────────────────────────────┘  │
│  [筛选 ▾] [排序 ▾]                   │
│                                      │
│  ┌────── 记录列表 / 空状态 ────────┐ │
│  │  (空状态) 🏆 空空如也~          │ │
│  │         [添加黄金]              │ │
│  └────────────────────────────────┘  │
│                                      │
├──────────────────────────────────────┤
│  [动态] [攒金] [+] [记账] [我的]     │
└──────────────────────────────────────┘
```

> **说明**：本应用为 React Web 应用，通过浏览器访问。底部导航为应用内导航组件。

### 1.2 账本切换

| 功能 | 触发 | 行为 |
|------|------|------|
| 切换账本 | 点击左上角"默认账本 ▾" | 弹出账本列表，选择后刷新页面数据 |
| 新建账本 | 账本列表中点击"新建" | 弹窗输入名称（1~20字符）→ 创建成功自动切换 |

### 1.3 资产总览卡片

| 元素 | 数据来源 | 行为 |
|------|----------|------|
| 总重量(克) | `GET /summary` → `total_weight` | 实时展示，支持 👁 隐藏/显示 |
| 购入总价(元) | `GET /summary` → `total_cost` | 实时展示 |
| 预估价值(元) | `GET /summary` → `estimated_value` | = 总重量 × 当前金价 |
| 预估收益(元) | `GET /summary` → `estimated_profit` | 正值绿色，负值红色 |

### 1.4 操作按钮

| 按钮 | 触发 | API 调用 | 行为 |
|------|------|----------|------|
| 攒金日历 | 点击 | — | 跳转日历视图页（P1） |
| 添加黄金 | 点击 | — | 跳转录入页 |
| 赠卖记录 | 点击 | — | 跳转赠卖记录列表页（P1） |

### 1.5 记录列表

| 状态 | 条件 | 显示内容 |
|------|------|----------|
| **空状态** | 当前账本无记录 | 金元宝图标 + "空空如也~" + "添加黄金"按钮 |
| **有记录** | 记录数 ≥ 1 | 记录卡片列表，每条显示重量、总价、渠道、日期 |

### 1.6 筛选与排序

| 功能 | 触发 | API 参数 | 行为 |
|------|------|----------|------|
| 筛选 | 点击"筛选 ▾" | `channel=xxx` | 按购买渠道筛选 |
| 排序 | 点击"排序 ▾" | `sort_by=xxx&sort_order=xxx` | 按日期/重量/价格 升序/降序 |

## 2. 录入页（攒金）

> 对齐 `05` US-001（添加黄金记录）、US-005（AI 识图攒金）。

### 2.1 页面布局

```
┌──────────────────────────────────────┐
│  [×]           攒金           [···]  │
├──────────────────────────────────────┤
│  重量（克）              [AI 识图攒金]│
│       0.00                           │
│                                      │
│  模式 ⓘ    [汇总模式] [明细模式]     │
│                                      │
│  [成交总价]                          │
│  [购买渠道]                          │
│  [备注]                              │
│  [今天                          ▾]   │
│                                      │
│  ┌──────────────────────────────┐    │
│  │        📷 上传照片            │    │
│  └──────────────────────────────┘    │
│                                      │
│  [========== 保存 ===========]       │
└──────────────────────────────────────┘
```

### 2.2 字段规格

| 字段 | 类型 | 约束 | 行为 |
|------|------|------|------|
| 重量（克） | 数字输入 | 0.01~99999.99 | 大字体展示，支持 AI 识图自动填入 |
| 模式切换 | 切换按钮 | 汇总/明细 | 汇总：输入总价；明细：输入单价，自动计算总价 |
| 成交总价 | 数字输入 | > 0 | 汇总模式手动输入，明细模式自动计算 |
| 购买渠道 | 文本输入 | ≤50 字符 | 可选，常见渠道可下拉选择 |
| 备注 | 文本输入 | ≤200 字符 | 可选 |
| 日期 | 日期选择器 | 默认"今天" | 点击展开日历选择 |
| 上传照片 | 图片选择 | 最多 3 张，单张 ≤2MB | 调用 HTML5 File API |

### 2.3 AI 识图攒金（P1）

| 功能 | 触发 | API 调用 | 行为 |
|------|------|----------|------|
| 拍照识别 | 点击"AI 识图攒金" | `POST /ocr/recognize` | 选择文件 → 上传 → 识别重量 → 自动填入 |
| 识别失败 | OCR 返回 null | — | 提示"识别失败，请手动输入" |

### 2.4 保存逻辑

| 步骤 | 行为 |
|------|------|
| 1. 前端校验 | 重量和总价非空且在范围内 |
| 2. 调用 API | `POST /gold-records` |
| 3. 成功 | Toast "保存成功" → 返回账本列表页 → 刷新数据 |
| 4. 失败 | 展示错误信息，保留表单数据 |

> 对齐 `05` US-003（查看实时金价行情）、US-007（金价订阅）。

### 3.1 页面布局

```
┌──────────────────────────────────────┐
│                            [···]     │
│          黄金价格                     │
│        1060.33 ¥                     │
│       [🔔 订阅金价]                  │
│                                      │
│   [实时] [近一月] [近三月]            │
│  ┌──────────────────────────────┐    │
│  │                              │    │
│  │      📈 K 线图区域            │    │
│  │                              │    │
│  └──────────────────────────────┘    │
│                                      │
├──────────────────────────────────────┤
│  [动态] [攒金] [+] [记账] [我的]     │
└──────────────────────────────────────┘
```

### 3.2 金价展示

| 元素 | 数据来源 | 行为 |
|------|----------|------|
| 黄金价格 | `GET /market/price` → `price` | 大字体居中展示，单位 ¥/克 |
| K 线图 | `GET /market/history?range=xxx` | 切换 Tab 刷新图表 |
| 订阅金价 | `POST /market/subscribe` | 设置金价阈值，达到时推送通知（P1） |

### 3.3 时间范围切换

| Tab | API 参数 | 说明 |
|-----|----------|------|
| 实时 | `range=realtime` | 当日分时数据 |
| 近一月 | `range=1month` | 近 30 天日线 |
| 近三月 | `range=3month` | 近 90 天日线 |

## 6. 错误处理

| error.code | HTTP | 前端展示 |
|------------|------|----------|
| `40001` | 400 | 错误提示：具体字段校验失败信息 |
| `40002` | 400 | 错误提示："账本不存在" |
| `40003` | 400 | 错误提示："记录不存在" |
| `40004` | 400 | 错误提示："默认账本不可删除" |
| `50001` | 500 | Toast："金价数据获取失败，显示缓存数据" |
| `50002` | 500 | Toast："识别失败，请手动输入" |
