## 1. 概念模型

```
Ledger（账本）1 ── * GoldRecord（黄金记录）
   │                      │
   │  id (PK)             │  id (PK)
   │  name                │  ledger_id (FK → Ledger)
   │  is_default          │  weight / total_price / unit_price
   │  created_at          │  mode / channel / note
   └  updated_at          │  purchase_date / photos
                          └  created_at / updated_at

Ledger（账本）1 ── * GiftSellRecord（赠卖记录）
                          │
                          │  id (PK)
                          │  ledger_id (FK → Ledger)
                          │  type (gift/sell)
                          │  weight / amount
                          └  date / note / created_at

GoldPriceCache（金价缓存）
   │  id (PK)
   │  price
   │  currency
   └  updated_at
```

> **核心关系**：一个 Ledger 可包含零到多条 GoldRecord 和 GiftSellRecord。外键通过 `ledger_id` 字段关联。

## 2. 存储架构

| 层级 | 存储方式 | 用途 | 说明 |
|------|----------|------|------|
| **前端存储** | LocalStorage / IndexedDB | 账本、黄金记录、赠卖记录 | 浏览器本地持久化，数据不上传服务器 |
| **后端存储** | JSON 文件 | 金价缓存、OCR 配置 | 文件系统存储，轻量级 |
| **文件存储** | 本地文件系统 / 内存 | 临时图片文件 | 开发阶段使用，生产可迁移至 OSS |

### 存储引擎对比

| 引擎 | 适用场景 | 本项目使用 |
|------|----------|------------|
| LocalStorage | 小数据量（< 5MB），字符串存储 | 账本列表、用户配置 |
| IndexedDB | 大数据量，结构化数据存储 | 黄金记录、赠卖记录 |
| JSON 文件 | 后端配置、缓存数据 | 金价缓存 |

## 3. 实体定义 — Ledger（账本）

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `id` | `string` | 是 | UUID | 账本唯一标识 |
| `name` | `string` | 是 | 1~20 字符 | 账本名称 |
| `is_default` | `boolean` | 是 | 默认 false | 是否为默认账本，系统保留一个 |
| `created_at` | `string` | 是 | ISO-8601 | 创建时间 |
| `updated_at` | `string` | 是 | ISO-8601 | 最后更新时间 |

## 4. 实体定义 — GoldRecord（黄金记录）

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `id` | `string` | 是 | UUID | 记录唯一标识 |
| `ledger_id` | `string` | 是 | 关联 Ledger | 所属账本 |
| `weight` | `number` | 是 | 0.01~99999.99 | 重量（克），JS 中注意浮点精度 |
| `total_price` | `number` | 是 | > 0 | 成交总价（元），JS 中注意浮点精度 |
| `unit_price` | `number` | 是 | 自动计算或手动 | 单价（元/克），JS 中注意浮点精度 |
| `mode` | `string` | 是 | 枚举：summary \| detail | 录入模式 |
| `channel` | `string` | 否 | ≤50 字符 | 购买渠道 |
| `note` | `string` | 否 | ≤200 字符 | 备注 |
| `purchase_date` | `string` | 是 | ISO-8601 日期 | 购买日期 |
| `photos` | `string[]` | 否 | 最多 3 条 URL | 照片 URL 列表 |
| `created_at` | `string` | 是 | ISO-8601 | 创建时间 |
| `updated_at` | `string` | 是 | ISO-8601 | 最后更新时间 |

## 5. 实体定义 — GiftSellRecord（赠卖记录）

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `id` | `string` | 是 | UUID | 记录唯一标识 |
| `ledger_id` | `string` | 是 | 关联 Ledger | 所属账本 |
| `type` | `string` | 是 | 枚举：gift \| sell | 类型 |
| `weight` | `number` | 是 | > 0 | 重量（克），JS 中注意浮点精度 |
| `amount` | `number` | 是 | ≥ 0 | 金额（元），gift 为 0，JS 中注意浮点精度 |
| `date` | `string` | 是 | ISO-8601 日期 | 日期 |
| `note` | `string` | 否 | ≤200 字符 | 备注 |
| `created_at` | `string` | 是 | ISO-8601 | 创建时间 |

## 6. 实体定义 — GoldPriceCache（金价缓存）

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `id` | `number` | 是 | 自增 | 主键 |
| `price` | `number` | 是 | > 0 | 金价（元/克），JS 中注意浮点精度 |
| `currency` | `string` | 是 | 固定 "CNY" | 币种 |
| `updated_at` | `string` | 是 | ISO-8601 | 数据更新时间 |

## 8. 关键业务逻辑

| 逻辑 | 触发条件 | 行为 |
|------|----------|------|
| 汇总模式单价计算 | mode = summary | `unit_price = total_price / weight` |
| 明细模式总价计算 | mode = detail | `total_price = unit_price × weight` |
| 级联删除 | 删除 Ledger | 同步删除所有关联 GoldRecord 和 GiftSellRecord |
| 默认账本保护 | 删除 is_default=true 的账本 | 拒绝删除，返回 40004 |
| 资产总览计算 | 查询 summary | total_weight = Σweight（扣除赠卖），estimated_value = total_weight × current_price |

## 9. API 响应 DTO 概要

| API 端点 | 核心 DTO 字段 | 说明 |
|----------|---------------|------|
| `POST /gold-records` | id, ledger_id, weight, total_price, unit_price, mode, channel, purchase_date | 创建记录 |
| `GET /gold-records` | records[], total | 记录列表 |
| `GET /summary` | total_weight, total_cost, estimated_value, estimated_profit, current_price | 资产总览 |
| `POST /ledgers` | id, name, is_default, created_at | 创建账本 |
| `GET /market/price` | price, currency, updated_at | 实时金价 |
