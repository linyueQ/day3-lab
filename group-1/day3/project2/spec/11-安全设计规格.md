## 1. 安全目标与合规要求

| 安全目标 | 说明 | 对应合规 |
|----------|------|----------|
| 用户资产数据不泄露 | 用户黄金持仓信息仅存储在后端数据库，不传输到第三方 | 个人信息保护法 |
| 最小权限原则 | 每个组件只拥有完成任务所需的最低权限 | 安全设计最佳实践 |
| 传输加密 | 所有 API 通信强制 HTTPS | 微信小程序平台要求 |

## 2. 认证与授权

### 2.1 认证机制

| 项目 | 规格 |
|------|------|
| 认证方式 | 无需认证（本地应用） |
| Session 管理 | 不涉及（本地应用，无需登录态） |
| 密码策略 | 不涉及（微信授权登录，无密码） |
| 多因素认证 | 不涉及（依赖微信账号体系） |

### 2.2 角色与权限矩阵

| 角色 | 创建账本 | 添加记录 | 查看记录 | 删除记录 | 查看金价 |
|------|----------|----------|----------|----------|----------|
| 普通用户 | ✓ | ✓ | ✓（仅自己） | ✓（仅自己） | ✓ |
| 游客（未登录） | ✗ | ✗ | ✗ | ✗ | ✓ |

## 3. 数据安全

### 3.1 数据分类与分级

| 数据类型 | 安全等级 | 示例 | 保护措施 |
|----------|----------|------|----------|
| 用户资产数据 | **内部** | 黄金重量、购入价格、持仓价值 | 数据库存储，按用户隔离 |
| 照片文件 | **内部** | 黄金购买凭证照片 | 服务器存储，访问鉴权 |
| 用户标识 | **内部** | 微信 openid | 不明文展示，仅后端使用 |
| 金价数据 | 一般 | 实时金价、历史K线 | 公开数据，缓存即可 |

### 3.2 加密策略

| 场景 | 加密方式 | 说明 |
|------|----------|------|
| 传输加密 | TLS 1.2+ | 浏览器强制 HTTPS |
| 存储加密 | MySQL 透明数据加密（TDE） | 生产环境启用 |
| 密钥管理 | 环境变量 `.env` | 密钥不硬编码，生产用 Secrets Manager |

## 4. 接口安全

| 防护措施 | 实现方式 | 说明 |
|----------|----------|------|
| 输入校验 | 服务端白名单校验 | 所有用户输入进行类型、长度、格式校验 |
| SQL 注入防护 | SQLAlchemy ORM 参数化查询 | 禁止字符串拼接 SQL |
| XSS 防护 | React 自动转义 | React 框架自带防护 |
| 速率限制 | 100 req/min/user | 防止接口滥用 |
| 文件上传校验 | 类型白名单 + 大小限制 | 仅允许 jpg/png，单文件 ≤ 2MB |

## 5. 第三方服务安全

| 风险点 | 防护措施 | 验收标准 |
|--------|----------|----------|
| 金价 API Key 泄露 | 后端代理调用，Key 存储在环境变量 | 前端代码不含任何 API Key |
| OCR 服务传输用户照片 | 仅传输照片，不传输用户身份信息 | OCR 请求不携带 openid |
| 第三方服务不可用 | 降级策略（缓存/手动输入） | 降级后核心功能可用 |

## 6. 审计与日志

| 事件类型 | 记录内容 | 保留期限 | 告警 |
|----------|----------|----------|------|
| 用户登录 | openid（脱敏）、时间、IP | 90 天 | 异常登录告警 |
| 数据增删改 | 用户 ID、操作类型、记录 ID | 180 天 | 批量删除告警 |
| API 调用异常 | 端点、错误码、响应时间 | 30 天 | 错误率 > 5% 告警 |
| 第三方服务调用 | 服务名、调用结果、耗时 | 30 天 | 连续失败 ≥ 3 次告警 |

## 7. 安全威胁清单（STRIDE 简表）

| 类型 | 威胁 | 风险等级 | 应对措施 |
|------|------|----------|----------|
| S（伪造） | 伪造 openid 访问他人数据 | **高** | 后端校验 openid 有效性 |
| T（篡改） | 金额/重量数据在传输中被篡改 | 中 | HTTPS 传输 + 服务端校验 |
| I（泄露） | 用户资产数据泄露 | **高** | 数据按用户隔离 + 存储加密 |
| D（拒绝服务） | 大量请求导致服务不可用 | 中 | 速率限制 + 降级策略 |
| E（提权） | 普通用户访问管理接口 | 低 | 无管理接口，单角色模型 |

## 8. 部署安全

- [x] 所有 API 通信强制 HTTPS（浏览器安全要求）
- [ ] 数据库访问仅限应用服务器 IP 白名单
- [ ] 敏感配置（API Key）通过环境变量注入，不入代码仓库
- [ ] 依赖包定期安全扫描（`pip audit`）
- [ ] `.gitignore` 包含 `.env`、数据库凭证文件
