## 1. 愿景与目标

> 构建一款基于 React 的 Web 应用黄金记账本，帮助个人投资者在 1 分钟内完成黄金资产录入，随时随地查看持仓总览和实时收益。数据存储在浏览器本地，无需后端数据库。

| 03 痛点 | 04 目标 |
|---------|---------|
| 黄金资产分散记录，无法统一查看 | 多账本统一管理，资产总览一目了然 |
| 金价波动频繁，无法实时了解资产价值 | 接入实时金价 API，自动计算预估价值和收益 |
| 手动记账繁琐，容易遗漏出错 | 汇总/明细双模式录入 + AI 识图辅助 |
| 购买渠道多样，记录散乱 | 每条记录标记购买渠道，支持筛选和排序 |

## 2. 目标用户

| 角色 | 使用目标 | 频率 |
|------|----------|------|
| 黄金投资爱好者 | 记录每次黄金购买，跟踪资产价值和收益 | 每周 1~3 次 |
| 攒金新手 | 记录小额积存金，培养攒金习惯 | 每月 2~4 次 |
| 黄金饰品收藏者 | 记录饰品重量和购入价格，了解保值情况 | 每月 1~2 次 |

## 3. 核心业务场景

| 编号 | 场景名 | 用户路径 |
|------|--------|----------|
| SC-01 | 添加黄金记录 | 用户点击"添加黄金" → 输入重量/成交总价/购买渠道 → 选择日期 → 可选上传照片 → 保存 → 账本列表自动更新 |
| SC-02 | 查看资产总览 | 用户打开攒金主页 → 查看总重量/购入总价/预估价值/预估收益 → 浏览黄金记录列表 → 可筛选/排序 |
| SC-03 | 查看实时金价 | 用户切换到动态页 → 查看当前黄金价格 → 切换实时/近一月/近三月K线图 → 可订阅金价提醒 |
| SC-04 | AI 识图攒金 | 用户点击"识图攒金" → 拍照或选择照片 → AI 自动识别重量 → 填入重量字段 → 继续录入其他信息 |
| SC-05 | 管理赠卖记录 | 用户点击"赠卖记录" → 选择黄金记录 → 录入赠送/出售信息 → 更新持仓数据 |

## 4. 产品规则

| 编号 | 规则 | 来源 |
|------|------|------|
| R-01 | 每条黄金记录 MUST 绑定 `ledger_id`（账本 ID） | 03 §3 "多账本管理" |
| R-02 | `weight` 单位为克，精度 2 位小数，范围 0.01～99999.99 | 03 §4 "资产自动计算" |
| R-03 | `total_price` 单位为元，精度 2 位小数，MUST > 0 | Demo 录入页 |
| R-04 | 录入模式分为汇总模式和明细模式：汇总模式录入总价，明细模式录入单价×重量 | Demo 录入页 |
| R-05 | 预估价值 = 总重量 × 实时金价；预估收益 = 预估价值 - 购入总价 | 03 §4 "资产自动计算" |
| R-06 | 金价数据 MUST 展示数据获取时间戳，延迟不超过 15 分钟 | 03 §4 "金价实时性" |
| R-07 | 每条记录 MAY 关联照片（最多 3 张），单张不超过 2MB | Demo 录入页 |

## 5. 能力与优先级

| 子能力 | 优先级 | 对应场景 |
|--------|--------|----------|
| 黄金记录录入（汇总/明细模式） | **P0** | SC-01 |
| 账本列表与资产总览 | **P0** | SC-02 |
| 实时金价行情展示 | **P0** | SC-03 |
| 多账本管理（创建/切换/删除） | **P0** | SC-02 |
| 筛选与排序 | **P0** | SC-02 |
| AI 识图攒金 | **P1** | SC-04 |
| 赠卖记录管理 | **P1** | SC-05 |
| 攒金日历视图 | **P1** | SC-02 |
| 金价订阅提醒 | **P1** | SC-03 |

## 6. 验收与成功标准（PRD 层）

| 标准 | 度量 |
|------|------|
| 可演示 | 种子数据 ≥ 5 条下，"添加 → 列表 → 金价"主路径 **2 分钟内**演示完毕 |
| 资产计算准确 | 添加记录后总重量/购入总价在 1 秒内更新，预估价值/预估收益与实时金价联动 |
| 离线可用 | 金价 API 不可用时，展示最近一次缓存金价 + "数据更新于 XX:XX"提示 |
| 多账本隔离 | 不同账本的黄金记录互不干扰，资产总览仅统计当前账本 |
