## 1. 背景与问题

> 随着黄金投资热度持续升温，越来越多个人投资者通过银行、金店、线上平台等多渠道购买实物黄金或积存金。但目前缺乏一款简单易用的移动端工具来统一管理黄金资产。

| 痛点 | 来源 |
|------|------|
| 黄金资产分散记录，无法统一查看总持仓重量和收益情况 | S-01 竞品调研 · S-02 用户访谈 |
| 金价实时波动，用户无法随时了解自身黄金资产的当前价值 | S-02 用户访谈 |
| 手动记账繁琐（重量、价格、渠道、日期），容易遗漏或出错 | S-02 用户访谈 |
| 购买渠道多样（银行/金店/线上），记录散乱难以汇总 | S-01 竞品调研 |

## 2. 提案概要

在微信小程序内交付一款**黄金记账本**工具，帮助个人黄金投资者统一记录和管理所有渠道的黄金购买信息。核心能力包括：黄金记录的录入与管理（支持汇总/明细两种模式）、多账本管理、实时金价行情查看、资产总览（总重量/购入成本/预估价值/预估收益自动计算）、AI 识图攒金辅助录入。产品定位为轻量级个人工具，以"记录简单、查看方便"为设计原则。

## 3. 范围边界

| 做（范围内） | 不做（范围外） |
|-------------|---------------|
| 黄金记录 CRUD（汇总模式 + 明细模式） | 黄金交易/买卖撮合功能 |
| 实时金价行情展示（实时/近一月/近三月） | 金价预测或投资建议 |
| 资产总览自动计算（重量/成本/价值/收益） | 多用户社交/分享功能 |
| AI 识图识别黄金重量 | 复杂金融产品管理（基金/期货） |
| 多账本管理 | 多币种支持（仅人民币） |
| 赠卖记录管理 | 税务计算 |
| 攒金日历视图 | 数据导出/报表生成 |

## 4. 成功标准（可度量）

| 标准 | 度量方式 |
|------|----------|
| 主路径可走通 | 用户在 2 分钟内完成"添加黄金 → 查看账本列表 → 查看实时金价"全链路 |
| 资产自动计算 | 添加记录后，总重量/购入总价/预估价值/预估收益在 1 秒内自动更新 |
| 金价实时性 | 金价数据延迟不超过 15 分钟 |
| AI 识图可用 | 拍照上传后 3 秒内返回识别结果（P1，不阻塞首版） |

## 5. 风险（简表）

| 风险 | 影响 | 应对 |
|------|------|------|
| 第三方金价 API 不可用或延迟 | 行情页无法展示实时金价 | 本地缓存最近一次金价 + 展示数据时间戳 |
| AI 识图服务不稳定 | 拍照录入功能不可用 | 降级为手动输入模式，AI 识图为 P1 非阻塞 |
| 微信小程序审核周期不确定 | 上线延期 | 预留 1 周审核缓冲期 |
