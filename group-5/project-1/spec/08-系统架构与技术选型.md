### 1.2 架构风格

> 选择架构风格时思考：教学场景需要什么样的复杂度？分层带来什么好处？

| 决策 | 选型 | 理由 |
|------|------|------|
| **前后端分离** | React SPA + Flask REST API | 教学清晰度：前端独立开发服务器，前后端可独立迭代 |
| **单体部署** | 前后端各一个进程 | 不引入微服务复杂度，适合教学场景快速启动 |
| **分层架构** | Route → Agent → Provider | 替换 LLM Provider 时上层零改动，便于后期升级 |
| **客户端状态管理** | React State（非持久化） | 对话数据不落库，简化架构，无存储层依赖 |
| **API Key 后端管理** | `.env` 环境变量 | 密钥不暴露给前端，安全可控 |

## 2. 后端分层结构

> 每层写清"允许做什么、禁止做什么"。

| 层 | 文件 | 允许做 | 禁止做 |
|----|------|--------|--------|
| **Route** | `api_bp.py` | 参数校验、HTTP 状态码、traceId | 直接调用 LLM、操作文件 |
| **Agent** | `report_agent.py` | 降级编排、结果组装、文件解析调度 | 感知 HTTP 请求/响应 |
| **LLM Manager** | `llm_manager.py` | 管理多 Provider 配置、返回可用列表、切换 Provider | 暴露 API Key |
| **Provider** | `bailian_qa.py` / `deepseek_qa.py` | HTTP 调用外部 LLM API | 操作本地存储 |
| **Skill Client** | `skill_client.py` | 调用 Rabyte Skill API、处理异步结果 | 操作本地存储 |
| **File Parser** | `pdf_parser.py` | PDF/Word 解析、进度回调 | 调用外部 API |

## 3. 前端架构

| 项 | 选型 | 理由 |
|----|------|------|
| UI 框架 | React（Hooks）≥ 18 | 教学通用性最高 |
| 构建工具 | Vite ≥ 5 | 极快 HMR，零配置启动 |
| 状态管理 | `useState` + `useEffect` + `useContext` | 会话/问答/模型选择等状态，非持久化，页面刷新后清空 |
| HTTP 客户端 | `fetch` API | 浏览器原生，不引入 axios |
| 样式方案 | CSS Modules + 基础 UI 组件库 | 样式隔离，避免全局污染 |

## 5. 数据存储架构

### 5.1 存储策略

| 数据类型 | 存储位置 | 说明 |
|----------|----------|------|
| 会话/问答记录 | 前端 React State | 不做持久化，页面刷新后清空 |
| 上传文件（临时） | 后端 `uploads/` 目录 | 文件解析后保留，用于 Skill 分析 |
| API Key 配置 | 后端 `.env` 环境变量 | 安全存储，不暴露给前端 |
| 分析报告 | 前端 React State | 不做持久化 |

### 5.2 存储演进路线

| 阶段 | 存储 | 变更范围 | 说明 |
|------|------|----------|------|
| **P0（当前）** | 前端内存 + 后端文件系统 | — | 零持久化成本，适合教学 |
| P1 | + LocalStorage | 前端 | 支持页面刷新后恢复 |
| P2 | + SQLite / PostgreSQL | 后端 | 支持持久化和多用户 |

## 7. ADR（架构决策记录）

> 每条 ADR 记录一个"为什么选 A 而不选 B"的决策。至少写 2 条。

**示例 — ADR-001**：为什么选 Flask 而非 FastAPI？Flask 学习曲线低，无需 async/await，学员 Python 基础即可上手。

| ADR | 决策 | 核心理由 |
|-----|------|----------|
| ADR-001 | Flask 而非 FastAPI | 学习曲线低，教学更适合 |
| ADR-002 | React State 而非 LocalStorage/数据库 | 不需要持久化，简化架构，零存储层依赖 |
| ADR-003 | API Key 后端管理而非前端输入 | 安全性更高，前端仅做模型切换 |
| ADR-004 | 前端不拆组件库 | 教学场景单文件更易理解，后期可提取公共组件 |

## 8. 安全架构

> 对齐 `07` 4 安全需求。当前教学版与生产需求的差距。

| 层 | 当前措施 | 生产改进 |
|----|----------|----------|
| 认证 | 无（开放访问） | JWT |
| CORS | 全开放 | 白名单 |
| 密钥 | 环境变量 `.env` | Vault / Secrets Manager |
| 文件上传 | 大小限制 100MB | 病毒扫描 + 内容审计 |
| 数据传输 | HTTP（本地开发） | HTTPS/TLS 1.2+ |
| 数据存储 | 明文 JSON | AES-256 加密 |
| 日志审计 | 本地文件记录 | 集中式日志收集 + 审计分析 |
| 访问控制 | 无 | RBAC 角色权限控制 |
