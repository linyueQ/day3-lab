## 5. 依赖与风险

**依赖链**：

**S1 会话管理** → **S2 文件上传**（必须先有会话才能上传文件）→ **S3 问答核心**（必须先有文件才能提问）→ **S4 大模型切换**（扩展问答引擎选择）→ **S5 深度分析**（Skill API 集成）→ **S6 健康检查**（系统整体可用性验证）

| 风险 | 影响 | 应对 |
|------|------|------|
| 上游 LLM 不可用 | 问答降级到 Demo 模式 | Demo 模式兜底，确保基础功能可用 |
| 文件解析库依赖问题 | PDF/Word 解析失败 | 准备备选解析方案，异常时友好提示 |
| Rabyte Skill API 不可用 | 深度分析不可用 | 降级提示，不影响阶段一问答 |
| 前端构建环境问题 | 开发效率降低 | 统一使用 Vite，提供 Docker 开发环境 |
