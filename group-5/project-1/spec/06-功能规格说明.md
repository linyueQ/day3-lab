## 3. Sidebar 会话管理

> 对齐 `05` US-001。描述会话列表的增删查交互。

| 功能 | 触发 | API 调用 | 行为 |
|------|------|----------|------|
| 加载列表 | 页面渲染 | `GET /sessions` | 渲染会话列表 + 默认选中第一个 |

## 4. Main 内容区（三态渲染）

> 对齐 `05` US-002（问答提交）、US-003（历史记录）。描述三种条件下显示什么。

| 状态 | 条件 | 显示内容 |
|------|------|----------|
| **A 空状态** | `currentSession === null` | "请创建或选择一个会话开始" |
| **B 常见问题** | 有会话 && 无问答记录 | 显示常见问题网格："这份研报的核心观点是什么？"、"对比同行业评级"等，点击后自动填入问题框 |
| **C 对话历史** | 有问答记录 | 按时间倒序显示问答卡片，包含：问题、回答、来源标签、时间戳 |

### 4.3 问答卡片元素

> 对齐 `05` US-003 AC-003-02。

| 元素 | 说明 |
|------|------|
| 问题文本 | 用户提问原文，最大显示3行，超出折叠 |
| 回答内容 | AI回答，支持Markdown渲染，包含结构化数据 |
| 来源标签 | 根据 `answer_source` 显示 CoPaw/百炼/离线演示 |
| 时间戳 | 格式：HH:mm，鼠标悬停显示完整时间 |
| 关联文件 | 如有上传文件，显示文件名链接 |

### 4.1 记录卡片中的来源标签逻辑

> 根据 `answer_source` 和 `llm_used` 组合显示不同标签。

| `answer_source` | `llm_used` | 显示文本 | 样式 |
|-----------------|------------|----------|------|
| `copaw` | `true` | CoPaw | 绿色标签 |
| `bailian` | `true` | 百炼 | 蓝色标签 |
| `demo` | `false` | 离线演示 | 灰色标签 |

### 4.2 解析进度显示

| 状态 | 显示 |
|------|------|
| `pending` | "等待解析..." |
| `parsing` | 进度条 + "解析中 {progress}%" |
| `completed` | "解析完成" + 解析结果摘要 |
| `failed` | "解析失败" + 错误信息 |

## 5. 输入区域

> 对齐 `05` US-002 AC（文件上传 + 问题提交）。

| 元素 | 类型 | 行为 |
|------|------|------|
| file-upload | `<input type="file">` | 支持 PDF/Word，≤100MB，显示文件名和大小 |
| parse-progress | `<progress>` | 实时显示解析进度 |
| query-input | `<textarea>` | 3 行，placeholder "请输入您的问题..."，1-500字符 |

| clear-btn | `<button>` | 清空输入框 |

## 6. 错误处理

| 后端 `error.code` | HTTP | 前端展示 |
|--------------------|------|----------|
| `EMPTY_QUERY` | 400 | 错误提示："请输入问题" |
| `INVALID_QUERY` | 400 | 错误提示："问题过长，最多500字符" |
| `FILE_TOO_LARGE` | 400 | 错误提示："文件大小超过100MB限制" |
| `FILE_TYPE_INVALID` | 400 | 错误提示："仅支持 PDF/Word 格式" |
| `PARSE_FAILED` | 500 | 错误提示："文件解析失败，请重试" |
| `UPSTREAM_ERROR` | 500 | 错误提示："服务暂不可用，已切换至演示模式" |

## 7. 深度分析模块

> 对齐 `05` US-005。描述研报分析两阶段工作流的前端交互。

### 7.1 两阶段工作流设计

```
┌─────────────────────────────────────────────────────────────────┐
│  [阶段一：问答分析]              [阶段二：深度分析]              │
│                                                                 │
│  ┌─────────────────────┐        ┌─────────────────────────────┐│
│  │ 用户提问             │        │  [🔍 深度分析] 按钮          ││
│  │ ↓                   │        │  （用户手动触发）            ││
│  │ LLM 结构化回答       │        │  ↓                          ││
│  │ （实时响应）         │        │  Skill API 分析             ││
│  │ ↓                   │        │  （异步执行）                ││
│  │ 多轮对话继续         │        │  ↓                          ││

### 7.3 阶段二：深度分析入口

│  ┌─────────────────────────────────────────────────┐│

│  │    DeepSeek (deepseek-chat)                     ││
│  │    ─── 不可用 ───                               ││
│  │    OpenAI GPT (未配置)                           ││
│  └─────────────────────────────────────────────────┘│
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 9.3 数据来源

| 步骤 | 操作 |
|------|------|
| 页面加载 | 调用 `GET /api/llm/providers` 获取后端已配置的可用模型列表 |
| 渲染列表 | 按 available=true/false 分组显示，不可用模型置灰 |
| 用户选择 | 更新 React State 中的 `currentProvider`，后续对话使用新模型 |
| 偏好保持 | 用户选择的模型保存在 React State，页面刷新后恢复默认 |

## 10. Skill 集成模块

> 对齐 `05` US-007。描述开源研报分析 Skill 集成的功能设计。

### 10.1 Skill 调用流程

```
用户上传文件
     │
     ▼
┌─────────────────┐
│ 调用 Rabyte API │─── 成功 ───▶ 返回结构化解析结果
└─────────────────┘
     │
     失败
     ▼
┌─────────────────┐
│ 本地 PDF 解析   │─── 成功 ───▶ 返回基础文本结果
└─────────────────┘
     │
     失败
     ▼
┌─────────────────┐
│ 基础文本提取    │─── 成功 ───▶ 返回纯文本
└─────────────────┘
     │
     失败
     ▼
  错误提示
```

### 10.2 Rabyte Skill API 配置

| 配置项 | 值 |
|--------|-----|
| API URL | `https://open-api.rabyte.cn/alpha/open-api/v1/file` |
| 认证方式 | Bearer Token |
| 超时时间 | 30s |
| 重试次数 | 2 |
