## 1. 性能需求

### 1.1 延迟 SLO

| 指标 | 目标值 | 度量方式 | 来源 |
|------|--------|----------|------|
| **问答端点 P95 延迟** | < **5000ms** | `response_time_ms` 字段 | `03` §4 成功标准 |
| 会话管理端点延迟 | < 500ms | 服务端计时 | `03` §4 成功标准 |
| 文件解析延迟 | < 5000ms | 单文件 ≤100MB | `03` §4 成功标准 |
| 会话列表加载 | < 1000ms | 服务端计时 | `05` AC-001-02 |
| 页面首屏加载 | < 3000ms | Lighthouse 性能评分 | `05` AC-001-01 |
| 文件上传速度 | > 1MB/s | 上传进度监控 | `03` §4 成功标准 |
| JSON 文件读写 | < 50ms | Storage 层计时 | `10` §2 存储引擎 |
| LLM Provider 列表加载 | < 500ms | 前端 API 调用计时 | `04` §5.2 大模型切换 |
| Skill 深度分析 | < 60000ms | 异步执行计时 | `04` §5.4 阶段二 |

### 1.2 降级策略

**三级降级链**：

```
ReportAgent.ask(query, session_id, file_content)
  ├─[1] CoPaw 桥接 → answer_source='copaw', llm_used=True
  ├─[2] 百炼 DashScope → answer_source='bailian', llm_used=True
  └─[3] 离线演示兜底 → answer_source='demo', llm_used=False
```

| 约束 | 说明 |
|------|------|
| 三级降级**不可跳级** | 必须按 CoPaw → 百炼 → Demo 顺序尝试 |
| 降级**静默执行** | 不抛异常，通过 `answer_source` 标识 |
| Demo 模式**始终可用** | 无外部依赖，纯字符串拼接 |

## 2. 可用性需求

### 2.1 可用性指标

| 指标 | 目标值 | 说明 | 来源 |
|------|--------|------|------|
| 服务可用性 | **99%**（教学环境） | 单实例 Flask，不设 HA | `03` §4 成功标准 |
| MTTR | < 5 分钟 | 重启 Flask 进程即可恢复 | `03` §5 风险 |
| 数据持久化 | N/A（前端内存） | 对话数据保存在 React State，不做持久化，页面刷新后清空 | `04` §5.5 数据存储策略 |

### 2.2 故障模式与恢复方式

| 故障模式 | 影响 | 检测方式 | 恢复方式 | 来源 |
|----------|------|----------|----------|------|
| Flask 进程崩溃 | 服务不可用 | 健康检查接口超时 | 自动重启进程 | `03` §5 风险 |
| LLM 服务不可用 | 问答降级到 Demo 模式 | 调用超时/异常 | 三级降级自动切换 | `03` §5 风险 |
| 磁盘空间不足 | 无法保存上传文件 | 写入异常 | 清理旧文件/扩容磁盘 | `03` §5 风险 |
| 文件解析超时 | 单文件解析失败 | 解析超过 30s | 标记失败，允许重试 | `05` AC-002-01 |
| Skill API 不可用 | 深度分析功能不可用 | API 调用超时/异常 | 降级提示"深度分析暂不可用"，不影响阶段一 | `04` §5.4 P2-08 |

## 3. 可观测性需求

### 3.1 链路追踪

| 项 | 规格 |
|----|------|
| traceId 格式 | `tr_{uuid.hex}`（32 位十六进制） |
| 注入位置 | 所有成功响应顶层 + 错误响应 `error.traceId` |
| 上游透传 | 优先复用请求头 `X-Trace-Id`，缺省时本地生成 |

### 3.2 错误码体系

| HTTP | `error.code` | 触发条件 |
|------|--------------|----------|
| 400 | `EMPTY_QUERY` | query 为空/空白/null |
| 400 | `INVALID_QUERY` | query >500 字符 或 缺少 session_id |
| 400 | `FILE_TOO_LARGE` | file_size > 104857600 (100MB) |
| 400 | `FILE_TYPE_INVALID` | file_type 非 pdf/doc/docx |
| 400 | `PARSE_FAILED` | 文件解析异常 |
| 500 | `UPSTREAM_ERROR` | Agent 编排内部异常 |

## 4. 安全需求

| 项 | 当前实现 | 说明 |
|----|----------|------|
| API 认证 | 无（开放访问） | Workshop 教学版 |
| CORS | 全开放 | 生产需白名单 |
| 速率限制 | 无 | 教学场景不限速 |

### 4.1 敏感数据保护

| 数据类型 | 保护方式 | 来源 |
|----------|----------|------|
| LLM API Key | 后端环境变量存储（`.env`），禁止硬编码，不通过接口暴露给前端 | `04` §5.2 大模型切换 |
| 用户查询 | 仅存储在前端 React State，不做持久化 | `04` §5.5 数据存储策略 |
| 研报文件 | 本地存储，不上传至外部服务器 | `03` §3 范围边界 |
| 会话历史 | 按会话隔离，无跨会话访问 | `10` §3 Session 实体 |
| 解析结果 | 不落第三方存储，仅本地缓存 | `03` §3 范围边界 |
| 错误日志 | 脱敏处理，不包含原始查询内容 | `11` §6 审计与日志 |

## 5. 数据约束

### 5.1 输入约束

| 约束 | 规格 | 来源 |
|------|------|------|
| `query` 最小长度 | 1 字符 | `04` R-02 |
| `query` 最大长度 | 500 字符 | `04` R-02 |
| `session_id` 必填 | 非空字符串 | `04` R-01 |
| 会话标题默认值 | "新会话" | `10` Session 实体 |
| 文件大小限制 | ≤ 104857600 字节 (100MB) | `03` §3 范围边界 |
| 文件类型限制 | pdf / doc / docx | `03` §3 范围边界 |

### 5.2 类型约束

| 字段 | 类型 | 约束 | 来源 |
|------|------|------|------|
| `llm_used` | `boolean` | 必须为 True/False，不得为字符串 | `04` R-04 回答字段 |
| `answer_source` | `string` | 枚举：copaw \| bailian \| demo | `04` R-06 降级策略 |
| `parse_status` | `string` | 枚举：pending \| parsing \| completed \| failed | `10` §5 ReportFile 实体 |
| `file_type` | `string` | 枚举：pdf \| doc \| docx | `03` §3 范围边界 |
| `session_id` | `string` (UUID) | 必须符合 UUID v4 格式 | `10` §3 Session 实体 |
| `file_id` | `string` (UUID) | 必须符合 UUID v4 格式 | `10` §5 ReportFile 实体 |
| `query_count` | `integer` | ≥ 0，最大 9999 | `10` §3 Session 实体 |
| `response_time_ms` | `integer` | ≥ 0，单位毫秒 | `05` AC-002-02 |
| `parse_progress` | `integer` | 0-100，表示百分比 | `10` §5 ReportFile 实体 |
| `created_at` / `updated_at` | `string` | ISO-8601 格式，UTC 时区 | `10` §3/4/5 实体定义 |
