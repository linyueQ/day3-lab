## 1. 背景与问题

> 描述用户遇到的问题，至少写 2~3 条痛点。参考：信息获取、历史记录、可追溯性、系统可用性。

| 痛点 | 来源 |
|------|------|
| 信息分散：研报、新闻、数据分布在多个系统，无统一问答入口 | 用户访谈（S-01, S-03） |
| 快速提取关键信息困难：研报内容繁杂，人工阅读效率低 | 用户访谈（S-01, S-02） |
| 自动对比分析缺失：难以快速对比多家研报观点差异 | 用户访谈（S-02, S-03） |
| 信用资质分析无标准：投资控股型平台公司分析维度多，不同分析师标准不一 | 用户访谈（S-02, S-03） |
| 大模型选择受限：不同分析场景适合不同 LLM，无法灵活切换 | 邮件沟通（S-02） |
| 本地部署需求：数据安全要求高，需支持私有化部署 | 用户访谈（S-03） |

## 2. 提案概要

> 用 3~5 句话概括要做什么。参考格式："在 XXX 应用内交付 YYY 能力"。要素：会话式问答 · 能力降级 · 可追溯

为解决以上痛点，打造一款聚焦统一入口、快速提取、自动对比、标准化信用分析、灵活模型切换的专业投研助手工具。在投研工作台应用内交付研报智能分析能力，支持用户通过会话式交互上传研报文件，系统基于预设 Prompt 模板和 LLM 进行结构化问答（阶段一），并可通过"深度分析"按钮触发开源 Skill 工作流获取全面分析报告（阶段二）。所有分析过程可追溯、可复盘，数据存储于客户端前端状态，保障隐私安全。

## 3. 范围边界

> 明确"做什么"和"不做什么"，避免范围蔓延。各写 3~4 条。

| 做（范围内） | 不做（范围外） |
|-------------|---------------|
| 会话管理：创建、列表、删除 | 多租户/行级数据权限 |
| 资源整合：支持 1-2 家研报平台、财经数据源接入 | 未授权/收费资源不接入 |
| 研报信息对比自动化：PDF 标题、评级、核心观点对比 | 超大文件（>100MB）不支持 |
| 问答交互、结果自定义、可视化呈现 | 特殊格式不支持，不适配小众浏览器 |
| 结果追溯：支持历史分析过程查看 | 非标准格式识别准确率低、提取困难 |
| 投资控股型平台公司信用资质深度分析（三层穿透、八维分析） | 非投资控股类主体的信用分析 |
| 前端大模型切换（API Key 后端统一管理） | 前端输入/管理 API Key |
| Rabyte 开源 Skill 集成（表格提取、图表识别） | 自研 OCR / 表格提取引擎 |
| 两阶段分析工作流（Prompt 问答 + Skill 深度分析） | 全自动无人工介入的分析流程 |
| 客户端前端状态存储（React State，非持久化） | 数据库 / LocalStorage 持久化 |

## 4. 成功标准（可度量）

> 写出 3~4 条可量化的验收标准。句式：用户在 X 分钟内完成 XXX 全链路 / 未配置外部模型时返回 XXX

| 标准 | 度量方式 |
|------|----------|
| 主路径可走通 | 用户在 3 分钟内完成"新建会话 → 上传研报 → 提问 → 查看历史"全链路 |
| 数据提取对比准确率 | 准确率 ≥ 95%，单页响应时间 ≤ 1s |
| 效率提升 | 检索准确率 ≥ 95%，无操作卡顿 |
| 数据检索成功率 | 关键信息提取成功率 ≥ 90% |
| 信用分析结构化输出 | 阶段一回答覆盖八维分析框架，结构完整度 ≥ 90% |
| 深度分析可触发 | 用户点击"深度分析"按钮后 10s 内返回分析状态 |
| 大模型切换生效 | 切换模型后，下一轮对话使用新模型 |

## 5. 风险（简表）

> 列出 2~3 个主要风险。常见方向：外部依赖不可用、数据存储并发、模块间契约延期。

| 风险 | 影响 | 应对 |
|------|------|------|
| 外部 LLM 不可用 | 问答功能降级 | 三级降级：CoPaw → 百炼 → Demo 模式兜底 |
| 主流研报平台接口变更/无法获取数据 | 整合失败 | 设置资源池，遇失败切换其它平台 |
| 大模型信息提取准确率低 | 影响核心功能不达标 | 人工复核校验 |
| Rabyte Skill API 不可用 | 阶段二深度分析不可用 | 降级提示，不影响阶段一问答功能 |
| 前端状态丢失（页面刷新） | 用户历史对话丢失 | 明确告知用户，后续可升级持久化方案 |

## 6. 立项前置条件（Go / No-Go）

- ☑ 后端与前端可独立启动
- ☑ 演示数据目录可读写
- ☐ 至少 1 家研报平台 API 接入测试通过
- ☑ 大模型提取准确率预测试通过
- ☑ Rabyte Skill API 连通性验证
- ☑ 后端 LLM Provider 配置验证（至少 1 个 Provider 可用）
