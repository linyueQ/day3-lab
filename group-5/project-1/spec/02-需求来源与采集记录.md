## S-02 邮件沟通 — 关于投研助手的需求建议

| 项 | 内容 |
|---|---|
| 发件人 | 李总监 <li.d@company.com> |
| 收件人 | 王经理 <wang.m@company.com> |
| 时间 | 2026-04-10 09:30 |

**问题 1 — 研报阅读效率低：** 每天需阅读 10+ 份研报，平均每份 30-50 页，耗时 2-3 小时。希望能快速提取关键信息（评级、目标价、核心观点）。

**问题 2 — 研报对比困难：** 对比不同券商对同一公司研报需手动复制粘贴，效率很低，希望能自动生成对比表。

**问题 3 — 信用资质分析无标准：** 投资控股型平台公司的信用分析维度多、层次深，不同分析师标准不一，希望有统一分析框架（三层穿透 + 八维分析）。

**问题 4 — 大模型选择受限：** 不同分析场景适合不同 LLM，希望能灵活切换，API Key 由 IT 部统一在后端管理。

## S-03 用户访谈 — 高级分析师

| 项 | 内容 |
|---|---|
| 访谈对象 | 陈研究员（高级分析师） |
| 访谈人 | 张产品 |
| 时间 | 2026-04-12 10:00-11:00 |
| 方式 | 线上访谈 |

| 问题 | 回答要点 |
|------|----------|
| Q1：平时工作中最耗时的环节？ | 读研报最耗时。每天扫一遍行业研报找重点，电子版反而不知从哪看起。 |
| Q2：最希望在研报中获取什么信息？ | ①评级和目标价 → ②核心观点（分析师主要逻辑）→ ③数据预测（营收增速、PE 等） |
| Q3：同时看多份研报如何对比？ | 各家格式不同，需手动整理到 Excel。希望能自动生成对比表。 |
| Q4：对 AI 的了解或期待？ | ChatGPT 用得多，但涉及内部数据不敢传。**必须内部部署，数据安全第一。** |
| Q5：信用分析时最大的痛点？ | 投资控股型平台公司的分析维度多，容易遗漏。希望有标准化模板引导，按八个维度逐一分析。 |
| Q6：深度分析结果如何呈现？ | 希望分两步：先回答我的具体问题，再给一个全面的分析报告。两者独立但互补。 |

**总结：** ①快速提取+自动对比为核心诉求 ②必须本地部署，数据不出域 ③AI 接受度高 ④信用分析需标准化框架 ⑤分析分两阶段：问答+深度报告
