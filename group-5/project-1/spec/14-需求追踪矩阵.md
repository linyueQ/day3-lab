> 变更需求时，沿链路定位受影响的用户故事、API、测试用例。

## 1. 需求编号体系

| 编号格式 | 含义 | 示例 |
|----------|------|------|
| `REQ-M5RA-{NNN}` | 功能需求 | REQ-M5RA-001 |
| `RULE-{NNN}` | 数据/类型约束规则 | RULE-001 |

| 级别 | 含义 | 门禁影响 |
|------|------|----------|
| **P0** | 必须实现，阻塞发布 | 关联测试用例全绿方可合并 |
| **P1** | 应当实现，不阻塞首版 | CI warning |
| **P2** | 可选增强 | 不纳入门禁 |

### 关键风险

| 风险项 | 严重度 | 说明 |
|--------|--------|------|
| 文件解析超时处理 | 中 | 需补充 TC 验证解析超时后的状态恢复 |
| 并发写入冲突 | 低 | JSON 文件存储不支持并发，需确保单进程访问 |
| Rabyte Skill API 可用性 | 中 | 第三方服务不可控，需依赖降级策略保障体验 |
| 前端状态丢失 | 低 | 非持久化存储方案，页面刷新后数据清空，需明确告知用户 |
