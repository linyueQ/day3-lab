## 1. 安全目标与合规要求

> 明确本模块需遵循的安全目标和外部合规标准。投研场景尤其关注数据隐私与内部信息防泄露。

| 安全目标 | 说明 | 对应合规 |
|----------|------|----------|
| 数据不出域 | 所有用户数据、研报内容、问答记录禁止传输至外部服务 | 内部信息安全规范 |
| 最小权限原则 | 每个组件只拥有完成任务所需的最低权限 | 安全设计最佳实践 |
| 本地部署优先 | 支持私有化部署，核心数据不落第三方 | 数据安全合规要求 |

## 3. 数据安全

> 定义数据分类分级、加密策略、存储安全规范。**投研场景核心关注点**。

### 3.1 数据分类与分级

| 数据类型 | 安全等级 | 示例 | 保护措施 |
|----------|----------|------|----------|
| 研报原文 | **机密** | PDF/Word 研报文件 | 后端本地存储，不上传云端 |
| 问答记录 | **内部** | 用户提问与AI回答 | 前端 React State，不做持久化，不上传服务端 |
| LLM API Key | **机密** | 各 Provider 密钥 | 后端 `.env` 环境变量，不暴露给前端，不落日志 |
| 用户信息 | **内部** | 用户名、角色、操作日志 | 本地存储，最小采集 |
| 系统配置 | 一般 | 模型参数、界面配置 | 环境变量存储 |

### 3.2 加密策略

| 场景 | 加密方式 | 说明 |
|------|----------|------|
| 传输加密 | 无（本地开发） | 生产环境需启用 HTTPS/TLS 1.2+ |
| 存储加密 | 无（本地文件） | 生产环境需 AES-256 加密 |
| 密钥管理 | 后端环境变量 `.env` | 密钥不硬编码，不入代码仓库，`GET /llm/providers` 接口不返回 API Key |

## 4. 接口安全

> 与 `09` API 规格配合，定义 API 层面的安全防护措施。

| 防护措施 | 实现方式 | 说明 |
|----------|----------|------|
| 输入校验 | 服务端白名单校验 | 所有用户输入进行类型、长度、格式校验 |
| SQL 注入防护 | 不适用 | 使用 JSON 文件存储，无 SQL 注入风险 |
| XSS 防护 | 输出转义 | 研报内容渲染时防止脚本注入 |
| CSRF 防护 | 无 | 教学场景暂不启用 |
| 速率限制 | 无 | 教学场景不限速 |
| 文件上传校验 | 类型白名单 + 大小限制 | 仅允许 PDF/Word，限制单文件 100MB |

## 5. LLM / AI 安全

> 针对大模型集成的专项安全措施。投研场景数据敏感，需严格管控数据流向。

| 风险点 | 防护措施 | 验收标准 |
|--------|----------|----------|
| 数据外泄至第三方模型 | 本地部署模型 / 内网百炼服务，禁止调用外网 API | 网络出口审计零外传记录；所有 LLM 调用仅通过内网网关；answer_source 不为空，可追溯数据来源 |
| Prompt 注入攻击 | 系统 Prompt 隔离，用户输入预处理过滤 | 过滤危险字符（如 `<script>`、`eval()`）；禁止执行指令模式；用户输入长度限制 500 字符 |
| 模型幻觉 / 错误信息 | 回答标注来源引用，标识置信度 | 答案必须携带 answer_source 标识；结构化回答包含置信度字段；不确定内容标注"仅供参考" |
| 敏感信息泄露（模型回复） | 输出过滤层，正则匹配敏感信息并脱敏 | 身份证号、手机号、银行卡号自动脱敏（如 138****8888）；企业敏感数据正则匹配过滤；输出日志脱敏处理 |

## 7. 安全威胁清单（STRIDE 简表）

> 基于 STRIDE 模型，列出主要安全威胁与应对方案。

| 类型 | 威胁 | 风险等级 | 应对措施 | 负责人 |
|------|------|----------|----------|--------|
| S | 伪造用户身份访问研报 | 中 | 教学版暂不启用认证，生产版需 JWT | 开发负责人 |
| T | 研报内容在传输中被篡改 | 低 | 本地部署，无网络传输风险 | 开发负责人 |
| I | 内部数据通过AI模型泄露 | **高** | 本地部署 + 输出过滤 + 网络隔离 | 安全负责人 |
| D | 大量请求导致服务不可用 | 中 | 速率限制 + 降级策略 | 开发负责人 |
| E | 错误信息泄露系统信息 | 低 | 统一错误码，不暴露内部堆栈 | 开发负责人 |
| R | 权限提升访问未授权数据 | 中 | 会话隔离，数据权限校验 | 开发负责人 |

## 8. 部署安全

> 定义部署架构中的安全要求。

- [ ] 生产/开发环境**严格隔离**，数据不互通
- [ ] 敏感配置（API Key、密码）通过环境变量注入，不入代码仓库
- [ ] 依赖包定期安全扫描（`pip audit` / `npm audit`）
- [ ] 本地部署优先，核心数据不出域
- [ ] 定期备份本地数据文件
- [ ] 服务器防火墙仅开放必要端口（5000/5173）
- [ ] 禁用 root 账户直接登录，使用普通用户 + sudo
- [ ] 操作系统和依赖库定期更新安全补丁
- [ ] 数据目录设置适当权限（chmod 700），禁止其他用户访问
- [ ] 启用进程监控（如 systemd），服务异常自动重启
