## 1. 产品概述

### 1.1 产品定位

面向命理测算爱好者与普通用户的**多功能命理工具集合Web应用**，聚焦三类核心使用场景，为用户提供从日常参考到长期规划的个性化命理测算服务。

### 1.2 目标用户

| 用户类型 | 特征描述 | 核心需求 |
|---------|---------|---------|
| 日常参考用户 | 25-45岁，上班族，对命理有兴趣但非专业 | 每日快速查看运势，指导日常决策 |
| 事件决策用户 | 30-55岁，商务人士，面临重要决策 | 针对特定事件获取成功率参考 |
| 深度研究用户 | 28-60岁，命理爱好者，希望深入了解 | 解析自己和他人命格，了解人生基调 |

### 1.3 核心价值

- **个性化**: 区别于通用黄历，基于用户专属八字生成内容
- **场景化**: 覆盖日常参考、事件决策、长期规划三大场景
- **专业化**: 自研命理算法，支持奇门遁甲、大小六壬等传统起卦方式
- **便捷化**: Web端响应式设计，PC/移动端随时访问

### 2.3 功能详细说明

#### 2.3.1 用户注册与登录

**功能描述**: 用户通过手机号或邮箱注册账号，设置密码后登录系统。

**业务流程**:
1. 用户访问首页，点击"注册"按钮
2. 输入手机号/邮箱、验证码、密码
3. 提交注册，系统发送验证码（手机号短信/邮箱邮件）
4. 验证通过后创建账号，引导填写基础信息
5. 登录时输入账号密码，系统验证后发放JWT Token

**验收标准**:
- [ ] 支持手机号和邮箱两种注册方式
- [ ] 密码强度校验（至少8位，包含字母和数字）
- [ ] 登录失败显示明确错误提示
- [ ] Token有效期24小时，过期需重新登录

#### 2.3.2 个人档案管理

**功能描述**: 用户录入个人出生信息，系统自动计算八字并保存。

**业务流程**:
1. 用户进入"我的档案"页面
2. 输入姓名、性别、出生日期（公历）
3. 选择出生时辰（支持"不清楚时辰"选项）
4. 系统自动计算八字（年柱、月柱、日柱、时柱）
5. 用户确认并保存档案

**验收标准**:
- [ ] 支持公历日期输入，自动转农历
- [ ] 时辰选择提供12时辰选项 + "不清楚"选项
- [ ] 八字计算准确率100%
- [ ] 档案可随时修改更新

#### 2.3.3 个性化黄历日历

**功能描述**: 基于用户八字生成当日专属黄历内容。

**输出内容**:
- **适宜事项**: 3-5条当日适合开展的活动（如：签约、出行、面试）
- **不适宜事项**: 2-3条当日应避免的活动（如：动土、借贷）
- **幸运颜色**: 1-2种当日有利衣物颜色（如：红色、黄色）
- **有利方向**: 1-2个当日有利方位（如：正东、东南）

**业务流程**:
1. 用户登录后的首页默认展示今日黄历
2. 系统根据用户八字 + 当日日期计算个性化内容
3. 以卡片形式展示适宜/不适宜事项、幸运提示
4. 支持左右滑动或点击按钮查看前后日期

**验收标准**:
- [ ] 首页加载即显示今日黄历，无需额外操作
- [ ] 内容基于用户八字个性化生成，非通用黄历
- [ ] 响应式设计，移动端单列显示，PC端可多列布局
- [ ] 支持查看前后7天的黄历

#### 2.3.4 奇门遁甲起卦

**功能描述**: 依托用户当前时间+空间信息自动起卦，针对特定事项给出建议。

**业务流程**:
1. 用户进入"运势问询"功能页
2. 选择"奇门遁甲"起卦方式
3. 输入要测算的具体事项（如：今天下午的会议是否顺利？）
4. 系统自动获取当前时间和用户位置（需授权）
5. 起卦计算，显示卦象和解析结果
6. 输出"是否适合开展"和"成功率预估"

**输出内容**:
- **卦象信息**: 局数、值符、值使、九宫排盘
- **事项建议**: 明确给出"适合"或"不适合"的建议
- **成功率**: 百分比形式（如：75%）
- **详细解析**: 200-500字的解析说明

**验收标准**:
- [ ] 需要用户授权地理位置信息
- [ ] 同一用户同日仅限起卦1次（限制防滥用）
- [ ] 起卦计算时间<3秒
- [ ] 结果包含卦象可视化展示

#### 2.3.5 大小六壬起卦

**功能描述**: 用户手动输入3个数字起卦，支持单日多次测算。

**业务流程**:
1. 用户进入"运势问询"功能页
2. 选择"大小六壬"起卦方式
3. 输入要测算的具体事项
4. 手动输入3个数字（范围：1-12）
5. 系统根据数字起卦计算
6. 显示卦象和解析结果

**输出内容**: 与奇门遁甲相同

**验收标准**:
- [ ] 3个数字输入范围限制1-12
- [ ] 支持单日多次起卦（最多5次/日）
- [ ] 不同数字组合产生不同卦象结果
- [ ] 起卦计算时间<2秒

#### 2.3.6 静态命格解析

**功能描述**: 长期静态命理分析，解析个人整体人生基调。

**业务流程**:
1. 用户进入"命格解析"功能页
2. 选择"解析本人"或"解析他人"
3. 输入姓名、出生日期、出生时辰（可选）
4. 系统计算命格，生成关键词和深度解析
5. 展示解析结果

**输出内容**:
- **命格关键词**: 1-3个概括性关键词（如："高楼望月"、"龙腾四海"）
- **五行分析**: 金木水火土五行强弱分布
- **性格特征**: 200-300字性格描述
- **事业财运**: 200-300字事业和财运分析
- **感情婚姻**: 200-300字感情婚姻分析
- **健康提示**: 100-200字健康注意事项
- **运势建议**: 100-200字人生规划建议

**验收标准**:
- [ ] 支持本人和他人命格解析
- [ ] 无时辰情况下仍可解析，但标注"精度降低"
- [ ] 解析内容总字数800-1500字
- [ ] 解析结果支持保存和查看历史

## 3. UI/UX设计规范

### 3.1 设计风格

- **设计语言**: Bitcoin DeFi美学 - "数字黄金"风格
- **风格定位**: 精密科技感、加密信任感、数字奢华体验
- **设计原则**: 
  - **发光能量** - 光线从交互元素发出，比特币橙色光晕、金色高光
  - **数学 precision** - 严格几何规则，超细1px边框，等宽字体显示数据
  - **分层深度** - 玻璃态效果、彩色发光阴影、背景模糊创造Z轴空间
  - **纹理虚空** - 微妙网格图案、径向渐变模糊、噪点纹理让背景呼吸
  - **设计即信任** - 高对比度、清晰层次传达安全可靠性

### 3.2 色彩系统

**主色调**:
- Primary: #F7931A - 比特币橙（主要按钮、链接、活动状态、信任指示器）
- Secondary: #EA580C - 焦橙（渐变、次要元素、视觉深度）
- Tertiary: #FFD600 - 数字金（渐变搭配、高亮、成功状态）

**状态色**:
- Success: #FFD600 - 成功状态、确认操作（数字金）
- Warning: #F7931A - 警告提示（比特币橙）
- Error: #EF4444 - 错误状态、危险操作
- Info: #3B82F6 - 信息提示、帮助说明

**中性色**:
- 页面背景: #030304 - 真虚空（最深的宇宙空间）
- 卡片背景: #0F1115 - 暗物质（卡片、面板表面）

### 3.9 动画规范

**过渡时长**:
- Fast: 150ms - hover、focus状态变化
- Normal: 300ms - 颜色、背景过渡
- Slow: 500ms - 展开、收起、弹窗动画
- Dramatic: 800ms - 页面过渡、重要揭示

**缓动函数**:
- `ease-in-out` - 标准过渡
- `ease-out` - 进入动画
- `ease-in` - 离开动画
- `cubic-bezier(0.4, 0, 0.2, 1)` - 优雅展示

**动画原则**:
- 所有按钮包含 `transition-all` 响应微交互
- Hover时光晕效果增强
- 卡片hover时轻微提升 (-translate-y-1)
- 避免过度动画导致性能问题

**移动端优先策略**

**适配规则**:
- 小屏(<768px): 单列布局，隐藏侧边栏，底部Tab导航，简化装饰
- 中屏(768px-1024px): 双列布局，侧边栏可折叠
- 大屏(>1024px): 多列布局，完整功能展示，完整视觉效果

**触摸优化**:
- 最小点击区域: 44x44px
- 按钮高度: ≥44px
- 输入框高度: ≥48px
- 手势支持: 左右滑动切换日期、下拉刷新

**字体响应式**:
- Display: 桌面60px → 平板48px → 移动36px
- H1: 桌面48px → 移动30px
- Body: 保持16px全平台

### 3.11 可访问性(Accessibility)

- 对比度: WCAG AA标准（普通文本≥4.5:1，大文本≥3:1）
  - 比特币橙(#F7931A)在真虚空(#030304)上: 7.8:1 ✓
  - 纯光(#FFFFFF)在真虚空上: 21:1 ✓
- 键盘导航: 所有交互元素可通过Tab键访问
- 屏幕阅读器: 提供ARIA标签和语义化HTML
- 焦点指示: 清晰的focus-visible样式（橙色边框+光晕）
- 色盲友好: 不单独依赖颜色传达信息（配合图标和文字）

- 组件库: shadcn/ui + Tailwind CSS（可定制组件库）
