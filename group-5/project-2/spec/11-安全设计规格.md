## 1. 安全架构概述

### 1.1 安全原则

- **纵深防御**: 多层安全防护,不依赖单一安全措施
- **最小权限**: 用户和系统仅拥有完成功能所需的最小权限
- **零信任**: 不信任任何请求,所有请求都需要验证
- **安全默认**: 默认配置为最安全状态

## 2. 认证与授权

### 2.1 认证机制

**JWT Token认证**:

```typescript
// Token生成
const token = jwt.sign(
  { userId: user.id, email: user.email },
  process.env.JWT_SECRET,
  {
    expiresIn: '24h',
    algorithm: 'HS256'
  }
)

// Token验证
const decoded = jwt.verify(token, process.env.JWT_SECRET)
```

**Token结构**:
```json
{
  "header": {
    "alg": "HS256",
    "typ": "JWT"
  },
  "payload": {
    "userId": "uuid",
    "email": "user@example.com",
    "iat": 1618444800,
    "exp": 1618531200
  },
  "signature": "HMACSHA256(...)"
}
```

**Refresh Token机制**:
```
Access Token:  24小时有效期
Refresh Token: 7天有效期

刷新流程:
1. Access Token过期
2. 客户端使用Refresh Token请求新Token
3. 服务器验证Refresh Token
4. 返回新的Access Token
5. 可选: 轮换Refresh Token
```

### 2.2 密码安全

**密码策略**:
```
- 最小长度: 8位
- 必须包含: 大写字母、小写字母、数字
- 禁止: 常见弱密码、用户名相关密码
- 历史密码: 不可重复使用最近3次密码
```

**密码加密**:
```typescript
import bcrypt from 'bcrypt'

const saltRounds = 10

// 注册时加密
const hashedPassword = await bcrypt.hash(password, saltRounds)

// 登录时验证
const isValid = await bcrypt.compare(password, hashedPassword)
```

**密码重置**:
```
1. 用户请求重置密码
2. 发送重置链接到邮箱(有效期1小时)
3. 用户点击链接,输入新密码
4. 验证Token有效性
5. 更新密码,使旧Token失效
```

### 2.3 会话管理

**安全配置**:
```javascript
// Cookie设置
res.cookie('token', token, {
  httpOnly: true,      // 防止XSS
  secure: true,        // 仅HTTPS
  sameSite: 'strict',  // 防止CSRF
  maxAge: 24 * 60 * 60 * 1000  // 24小时
})
```

**会话控制**:
```
- 单用户最多3个并发会话
- 新登录使最旧会话失效
- 用户可主动注销所有会话
- 24小时无活动自动退出
```

### 2.4 授权模型

**基于角色的访问控制(RBAC)**:

| 角色 | 权限 | 说明 |
|------|------|------|
| user | 基础功能 | 注册用户可以使用的功能 |
| admin | 管理功能 | 管理员功能(未来扩展) |

**API权限控制**:
```typescript
// 中间件验证
function requireAuth(req, res, next) {
  const token = req.cookies.token
  
  if (!token) {
    return res.status(401).json({ error: 'UNAUTHORIZED' })
  }
  
  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET)
    req.userId = decoded.userId
    next()
  } catch (error) {
    return res.status(401).json({ error: 'INVALID_TOKEN' })
  }
}
```

### 3.2 存储加密

**敏感数据加密**:
```typescript
import crypto from 'crypto'

const algorithm = 'aes-256-gcm'
const key = crypto.randomBytes(32)

function encrypt(text: string): string {
  const iv = crypto.randomBytes(16)
  const cipher = crypto.createCipheriv(algorithm, key, iv)
  
  let encrypted = cipher.update(text, 'utf8', 'hex')
  encrypted += cipher.final('hex')
  
  const authTag = cipher.getAuthTag().toString('hex')
  
  return `${iv.toString('hex')}:${encrypted}:${authTag}`
}

function decrypt(encrypted: string): string {
  const [iv, encryptedText, authTag] = encrypted.split(':')
  
  const decipher = crypto.createDecipheriv(
    algorithm,
    key,
    Buffer.from(iv, 'hex')
  )
  
  decipher.setAuthTag(Buffer.from(authTag, 'hex'))
  
  let decrypted = decipher.update(encryptedText, 'hex', 'utf8')
  decrypted += decipher.final('utf8')
  
  return decrypted
}
```

**加密字段**:
| 字段 | 加密方式 | 说明 |
|------|---------|------|
| 手机号 | AES-256-GCM | 存储前加密 |
| 邮箱 | AES-256-GCM | 存储前加密 |
| 密码 | bcrypt | 哈希不可逆 |

## 5. 应用安全

### 5.1 输入验证

**Zod验证**:
```typescript
import { z } from 'zod'

const registerSchema = z.object({
  email: z.string().email().optional(),
  phone: z.string().regex(/^1[3-9]\d{9}$/).optional(),
  password: z.string()
    .min(8)
    .regex(/[A-Z]/, '必须包含大写字母')
    .regex(/[a-z]/, '必须包含小写字母')
    .regex(/\d/, '必须包含数字'),
  nickname: z.string().min(1).max(50)
}).refine(data => data.email || data.phone, {
  message: '邮箱或手机号至少提供一个'
})

// 使用验证
function validateRegister(req, res, next) {
  const result = registerSchema.safeParse(req.body)
  
  if (!result.success) {
    return res.status(400).json({
      error: 'INVALID_REQUEST',
      details: result.error.errors
    })
  }
  
  req.validatedData = result.data
  next()
}
```

### 5.2 SQL注入防护

**Prisma ORM参数化查询**:
```typescript
// 安全查询(自动参数化)

  where: { email: userInput }

// 禁止原始SQL查询

```

### 5.3 XSS防护

**React自动转义**:

## 6. 隐私保护

### 6.1 数据收集告知

**隐私政策**:
```
必须在注册时明确告知:
- 收集的数据类型
- 数据使用目的
- 数据存储期限
- 用户权利(查阅、修改、删除)
```

### 6.2 用户权利

**数据导出**:
```typescript
// 导出用户所有数据
async function exportUserData(userId: string) {
  const user = await prisma.user.findUnique({
    where: { id: userId },
    include: {
      profile: true,
      divinations: true,
      destinyHistory: true
    }
  })
  
  return JSON.stringify(user, null, 2)
}
```

**数据删除**:
```typescript
// 删除用户及所有数据
async function deleteUser(userId: string) {
  await prisma.$transaction([
    prisma.divinationRecord.deleteMany({ where: { userId } }),
    prisma.destinyHistory.deleteMany({ where: { userId } }),
    prisma.dailyCalendar.deleteMany({ where: { userId } }),
    prisma.destinyProfile.delete({ where: { userId } }),
    prisma.user.delete({ where: { id: userId } })
  ])
}
```

### 6.3 数据保留策略

| 数据类型 | 保留期限 | 到期处理 |
