## 2. 可用性需求

### 2.1 SLA目标

| 指标 | 目标值 | 计算方式 |
|------|--------|---------|
| 服务可用性 | ≥99.9% | (总时间-宕机时间)/总时间 |
| 月度宕机时间 | <43分钟 | 每月累计 |
| 故障恢复时间(RTO) | <5分钟 | 故障发生到恢复服务 |
| 数据恢复点(RPO) | <1分钟 | 可接受的数据丢失量 |

### 2.2 容错设计

**服务降级策略**:
```
1. 命理算法服务不可用:
   - 降级方案: 返回缓存结果或提示"系统维护中"
   - 触发条件: 算法服务响应超时>5秒
   
2. 数据库不可用:
   - 降级方案: 使用Redis缓存数据(只读模式)
   - 触发条件: 数据库连接失败
   
3. CDN不可用:
   - 降级方案: 回源到主服务器
   - 触发条件: CDN节点响应失败
```

**熔断机制**:
```javascript
// 使用cockatiel库实现熔断器
const circuitBreaker = new CircuitBreakerPolicy()
  .handleType(Error)
  .circuitBreaker(10000, 0.5) // 10秒窗口, 50%失败率触发
  
async function callDestinyAPI(params) {
  return await circuitBreaker.execute(() => destinyService.calculate(params))
}
```

**重试策略**:
```javascript
// 指数退避重试
const retryPolicy = new RetryPolicy()
  .handleType(Error)
  .retry(3)
  .exponentialBackoff([1000, 2000, 4000]) // 1s, 2s, 4s
  
async function callAPI(params) {
  return await retryPolicy.execute(() => axios.get('/api/xxx', params))
}
```

### 2.3 灾难恢复

**备份策略**:
- 数据库每日全量备份(凌晨3点)
- 数据库每小时增量备份
- 备份保留30天
- 备份文件加密存储(AES-256)

**恢复流程**:
```
1. 发现数据损坏或丢失
2. 评估影响范围和损坏程度
3. 选择最近的可用备份点
4. 执行数据恢复脚本
5. 验证数据完整性
6. 恢复服务
7. 记录事故报告
```

## 3. 安全性需求

### 3.1 认证与授权

**认证方式**:
- JWT Token认证(有效期24小时)
- Refresh Token机制(有效期7天)
- 支持单点登录(SSO)

**密码策略**:
```
- 最小长度: 8位
- 必须包含: 大写字母、小写字母、数字
- 禁止使用: 常见弱密码(前10000个常见密码)
- 密码加密: bcrypt(salt rounds = 10)
- 密码修改: 用户可随时修改密码
```

**会话管理**:
```
- Token存储: HttpOnly Cookie(防XSS)
- Token刷新: 静默刷新(用户无感知)
- 并发登录: 允许3个设备同时在线
- 会话超时: 24小时无活动自动退出
```

### 3.2 数据安全

**传输加密**:
- HTTPS强制(TLS 1.3)
- HSTS头部(强制HTTPS)
- 证书自动更新(Let's Encrypt)

**存储加密**:
```
敏感数据加密字段:
- 用户手机号: AES-256加密
- 用户邮箱: AES-256加密
- 密码: bcrypt哈希(不可逆)
- 出生信息: 明文存储(非敏感)
```

**数据脱敏**:
```javascript
// 日志脱敏
function sanitizeLog(data) {
  return {
    ...data,
    phone: data.phone?.replace(/(\d{3})\d{4}(\d{4})/, '$1****$2'),
    email: data.email?.replace(/(.{2}).*(@.*)/, '$1***$2')
  }
}
```

### 3.3 安全防护

**SQL注入防护**:
- 使用Prisma ORM(参数化查询)
- 禁止拼接SQL字符串
- 定期安全审计

**XSS防护**:
- React自动转义HTML
- Content Security Policy(CSP)头部
- 禁止使用dangerouslySetInnerHTML(除非必要)

**CSRF防护**:
- SameSite Cookie属性
- CSRF Token验证
- 关键操作二次确认

**速率限制(Rate Limiting)**:
```javascript
// API速率限制
const rateLimit = {
  windowMs: 15 * 60 * 1000, // 15分钟
  max: 100, // 最多100次请求
  message: '请求过于频繁,请稍后重试'
}

// 起卦功能额外限制
const divinationLimit = {
  qimen: { max: 1, windowMs: 24 * 60 * 60 * 1000 }, // 1次/天
  liuren: { max: 5, windowMs: 24 * 60 * 60 * 1000 }  // 5次/天
}
```

**DDoS防护**:
- 使用Cloudflare CDN(自带DDoS防护)
- IP白名单(管理后台)
- 异常流量监控和告警
