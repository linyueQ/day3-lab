# 03-立项提案与范围说明

## 1. 背景与问题

| 痛点 | 来源 |
|------|------|
| 国际局势信息分散，获取效率低 | 市场营销部门选品经验 |
| 主题基金筛选困难，人工对比工作量大 | 市场营销部门选品经验 |
| 小规模优质基金发现难 | 市场营销部门选品经验 |
| 局势与基金表现关联缺乏可视化 | 市场营销部门选品经验 |

## 2. 提案概要

在【基金投资分析 Demo 应用】内交付**国际局势追踪**、**主题基金智能筛选**、**小规模优质基金推荐**、**数据可视化关联分析**四大核心能力，帮助市场营销部门在选品时快速识别当前阶段表现优秀的基金产品，基于美伊战争等国际事件动态做出更科学的决策。

## 3. 范围边界

| 做（范围内） | 不做（范围外） |
|-------------|---------------|
| RSS 数据收集：从网页 RSS 获取美伊战争新闻，编写 3 月以来历史数据收集脚本 | 用户登录/注册系统 |
| 主题基金筛选：从原油、黄金、煤炭、电力、天然气主题中挑选表现最优基金（先按基金名称确定） | 实时交易/下单功能 |
| 小规模基金推荐：从广发、易方达、富国、南方、华夏中筛选规模 5 亿以下、业绩优异基金 | 基金购买/赎回操作 |
| 数据可视化：按周展示时间轴，包含局势分值、主题基金业绩、小规模基金表现 | 移动端适配（仅 Web 端） |
| | 多主题扩展（算力、存储、芯片等） |

## 4. 成功标准

| 标准 | 目标 |
|------|------|
| 数据收集完整性 | 成功收集 3 月初至今美伊战争新闻数据 |
| 基金筛选功能 | 主题基金和小规模基金筛选功能正常运行 |
| 页面展示 | 时间轴可视化页面可正常展示数据 |
| 数据更新 | 支持手动或自动更新数据 |

## 5. 风险与应对

| 风险 | 应对措施 |
|------|----------|
| 基金数据获取受限 | 使用本地 JSON 数据作为 fallback |

## 6. 技术栈

- 前端：React + Vite
- 后端：Python（FastAPI/Flask）
- 数据：RSS 抓取 + 本地存储
