# 机器学习基金评级系统

## 项目简介

本项目实现了一个基于机器学习的基金评级系统，采用LightGBM模型预测基金未来超额收益，并将预测结果映射为A/B/C/D/E五档评级。系统覆盖了数据处理、特征工程、模型训练、评级映射、回测验证和可解释性分析的完整流程。

## 项目结构

```
fund_analysis/
├── data/              # 数据目录，存储基金收益和基准收益数据
├── models/            # 模型目录，存储训练好的LightGBM模型
├── results/           # 结果目录，存储评级结果、回测图表和解释卡
├── utils/             # 工具目录，存储通用工具函数
├── config.py          # 配置文件，定义模型参数和评级参数
├── data_processor.py  # 数据处理模块，包括数据加载和特征工程
├── model_trainer.py   # 模型训练模块，包括LightGBM模型和时间滚动训练
├── rating_mapper.py   # 评级映射模块，将模型预测映射为A/B/C/D/E五档
├── backtester.py      # 回测和评估模块，验证评级系统有效性
├── explainer.py       # 可解释性模块，使用SHAP生成基金评分解释卡
├── main.py            # 主脚本，整合所有模块功能
├── run.py             # 运行脚本，支持多种运行模式
├── requirements.txt   # 依赖包文件
└── README.md          # 项目说明文件
```

## 核心功能

1. **数据处理**：加载基金收益数据和基准收益数据，生成基金特征和标签（未来超额收益）。
2. **模型训练**：使用LightGBM模型进行时间滚动训练和预测，避免未来函数偏差。
3. **评级映射**：将预测的超额收益映射为A/B/C/D/E五档评级，并考虑风险调整因素。
4. **回测验证**：执行分层回测、多空回测和信息系数计算，验证评级系统的有效性。
5. **可解释性分析**：使用SHAP生成基金评分解释卡，展示模型预测的关键特征贡献。

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 准备数据

系统默认会在首次运行时生成示例数据。如果需要使用自己的数据，请在`data`目录下创建以下文件：

- `fund_returns.csv`：基金收益数据，格式为日期索引，列名为基金ID。
- `benchmark_returns.csv`：基准收益数据，格式为日期索引，列名为基准名称（默认：hs300）。

### 3. 运行系统

#### 3.1 运行完整流程（推荐）

```bash
python run.py --mode full
```

此命令会执行完整的基金评级流程，包括数据处理、模型训练、评级映射、回测验证。

#### 3.2 仅运行评级流程

```bash
python run.py --mode rating
```

#### 3.3 仅运行回测流程

```bash
python run.py --mode backtest
```

#### 3.4 仅运行可解释性分析

```bash
python run.py --mode explanation --model_id 202512
```

其中，`model_id`为模型文件名的后缀（格式：YYYYMM），表示要分析的模型。

## 配置说明

系统的主要配置参数存储在`config.py`文件中，包括：

- **MODEL_PARAMS**：LightGBM模型参数，如学习率、树深度等。
- **RATING_PARAMS**：评级参数，如分位数阈值、风险调整阈值等。
- **ROLLING_PARAMS**：时间滚动训练参数，如训练窗口长度、测试窗口长度等。
- **DATA_PARAMS**：数据处理参数，如数据目录、特征配置等。
- **BACKTEST_PARAMS**：回测参数，如是否执行多空回测、分层回测等。
- **EXPLAINER_PARAMS**：可解释性参数，如SHAP计算样本大小、显示的top特征数量等。

用户可以根据需要修改这些参数。

## 输出结果

系统运行完成后，会在`results`目录下生成以下文件：

- `fund_ratings.csv`：基金评级结果，包含日期、基金ID、评级、评级描述和预测超额收益。
- `rating_distribution.png`：评级分布图，展示各评级的基金数量分布。
- `backtest_results.png`：回测结果图，展示分层回测、多空回测和信息系数结果。
- `feature_importance.png`：特征重要性图，基于SHAP值展示各特征的重要性。
- `fund_{fund_id}_{date}_explanation.md`：基金评分解释卡（Markdown格式），展示关键特征贡献。
- `fund_{fund_id}_{date}_explanation.png`：基金评分解释瀑布图，直观展示特征贡献。

## 评级标准

系统采用五档评级标准：

| 评级 | 分位数 | 描述 |
|------|--------|------|
| A    | Top 20% | 优秀，高概率超额收益 |
| B    | 60-80% | 中上，略优 |
| C    | 40-60% | 中位，中性 |
| D    | 20-40% | 偏弱，业绩持续性差 |
| E    | Bottom 20% | 较差，未来表现风险高 |

## 技术特点

1. **时间滚动训练**：采用时间序列交叉验证，避免未来函数偏差，提高模型的泛化能力。
2. **风险调整**：考虑波动率、集中度和换手率等风险因素，对评级进行调整，提高评级的稳健性。
3. **可解释性**：使用SHAP生成基金评分解释卡，提高模型的透明度和可理解性。
4. **完整回测**：执行多种回测方法，全面验证评级系统的有效性。
5. **模块化设计**：系统采用模块化设计，各模块功能独立，便于扩展和维护。

## 扩展建议

1. **特征工程扩展**：可以添加更多基金特征，如持仓特征、行业暴露特征、基金经理特征等。
2. **模型集成**：可以集成多种模型，如XGBoost、CatBoost等，提高预测精度。
3. **实时更新**：可以设置定时任务，定期更新评级结果，保持评级的时效性。
4. **可视化界面**：可以开发Web界面，展示评级结果和解释卡，提高用户体验。
5. **多因子模型**：可以构建多因子模型，同时预测收益和风险，进一步提高评级的准确性。

## 注意事项

1. **数据质量**：系统的性能依赖于数据质量，建议使用高质量的基金收益数据。
2. **参数调优**：默认参数可能不适合所有场景，建议根据实际数据进行参数调优。
3. **过拟合风险**：机器学习模型存在过拟合风险，建议使用交叉验证和正则化技术。
4. **市场变化**：市场环境会发生变化，建议定期重新训练模型，适应市场变化。
5. **结果解释**：评级结果仅供参考，不构成投资建议，投资决策请结合其他因素。
